# 构建产物与安装包格式

> Electron 应用的两类产物：可运行目录与分发安装包

---

## 一、两类构建产物

### 1.1 概述

```
┌─────────────────────────────────────────────────────────────────┐
│                    Electron 构建产物                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 可运行目录（unpacked）                                       │
│     ├── 用于开发调试                                            │
│     ├── 可直接运行，无需安装                                     │
│     ├── 方便查看打包结果                                        │
│     └── 命令：electron-builder --dir                           │
│                                                                 │
│  2. 分发安装包（installer）                                      │
│     ├── 用于最终发布                                            │
│     ├── 用户下载安装使用                                        │
│     ├── 包含安装/卸载逻辑                                       │
│     └── 命令：electron-builder                                 │
│                                                                 │
│  构建流程：                                                      │
│  源码 → 构建 → 打包（unpacked）→ 制作安装包（installer）          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 生成命令

```bash
# 只生成可运行目录
npx electron-builder --dir

# 生成安装包
npx electron-builder

# 指定平台
npx electron-builder --win --dir   # Windows 目录
npx electron-builder --mac         # macOS 安装包
npx electron-builder --linux       # Linux 安装包
```

## 二、可运行目录结构

### 2.1 Windows (win-unpacked)

```
dist/win-unpacked/
├── MyApp.exe                     # 主程序（重命名后的 electron.exe）
├── resources/
│   ├── app.asar                  # 应用代码归档
│   ├── app.asar.unpacked/        # 解压的文件（native modules）
│   │   └── node_modules/
│   │       └── native-module/
│   │           └── build/
│   │               └── Release/
│   │                   └── module.node
│   └── electron.asar             # Electron 资源
├── locales/                      # 语言包
│   ├── zh-CN.pak
│   ├── en-US.pak
│   └── ...
├── *.dll                         # Electron 依赖的 DLL
│   ├── d3dcompiler_47.dll
│   ├── ffmpeg.dll
│   ├── libEGL.dll
│   ├── libGLESv2.dll
│   └── ...
├── chrome_100_percent.pak
├── chrome_200_percent.pak
├── icudtl.dat                    # ICU 数据
├── snapshot_blob.bin             # V8 快照
├── v8_context_snapshot.bin
├── vk_swiftshader.dll            # Vulkan 软件渲染
├── vk_swiftshader_icd.json

```

### 2.2 macOS (mac/MyApp.app)

```
dist/mac/MyApp.app/
└── Contents/
    ├── Info.plist                # 应用元信息

    ├── Resources/
```

## 三、安装包格式对比

### 3.1 Windows 格式

| 格式 | 说明 | 特点 | 使用场景 |
|------|------|------|----------|
| **NSIS** | 最常用的安装程序 | 功能丰富，可定制 | 大多数应用 |
| **NSIS Web** | 在线安装程序 | 小体积安装器，运行时下载 | 大型应用 |
| **Portable** | 绿色版 | 无需安装，直接运行 | 便携使用 |
| **MSI** | Windows Installer | 企业部署，组策略支持 | 企业应用 |
| **AppX** | Windows Store 格式 | UWP 沙盒，自动更新 | Store 分发 |
| **Squirrel** | 自动更新框架 | 增量更新，无需管理员 | 特定场景 |

```yaml
# electron-builder.yml 配置
win:
  target:
    - target: nsis
      arch: [x64, ia32]
    - target: portable
      arch: x64
```

### 3.2 macOS 格式

| 格式 | 说明 | 特点 | 使用场景 |
|------|------|------|----------|
| **DMG** | 磁盘映像 | 最常见，拖拽安装 | 官网分发 |
| **PKG** | 安装包 | 向导式安装，支持脚本 | 复杂安装 |
| **ZIP** | 压缩包 | 简单，用于更新 | 自动更新 |
| **MAS** | Mac App Store | 沙盒，审核 | Store 分发 |

```yaml
mac:
  target:
    - target: dmg
      arch: [x64, arm64]
    - target: zip     # 用于自动更新
      arch: [x64, arm64]
```

### 3.3 Linux 格式

| 格式 | 说明 | 特点 | 使用场景 |
|------|------|------|----------|
| **AppImage** | 便携格式 | 无需安装，跨发行版 | 通用分发 |
| **deb** | Debian 包 | Debian/Ubuntu 系统 | Debian 系 |
| **rpm** | RPM 包 | Fedora/RHEL 系统 | Red Hat 系 |
| **snap** | Snap 包 | 沙盒，自动更新 | Ubuntu |
| **flatpak** | Flatpak 包 | 沙盒，跨发行版 | 现代发行版 |

```yaml
linux:
  target:
    - AppImage
    - deb
    - rpm
```

## 四、各格式详细说明

### 4.1 NSIS (Windows)

```
NSIS (Nullsoft Scriptable Install System)
├── 安装界面可定制
├── 支持多语言
├── 可添加自定义脚本
├── 支持增量更新
└── 输出：MyApp-Setup-1.0.0.exe
```

```yaml
nsis:
  oneClick: false
  allowToChangeInstallationDirectory: true
  createDesktopShortcut: true
  include: installer.nsh
```

### 4.2 DMG (macOS)

```
DMG (Disk Image)
├── 可自定义背景图
├── 拖拽到 Applications 安装
├── 可包含许可协议
└── 输出：MyApp-1.0.0.dmg
```

```yaml
dmg:
  background: build/background.png
  contents:
    - x: 130
      y: 220
      type: file
    - x: 410
      y: 220
      type: link
      path: /Applications
```

### 4.3 AppImage (Linux)

```
AppImage
├── 单文件可执行
├── 无需 root 权限
├── 跨发行版兼容
├── 需要 FUSE 支持
└── 输出：MyApp-1.0.0.AppImage
```

```bash
# 运行 AppImage
chmod +x MyApp.AppImage
./MyApp.AppImage

# 或右键 → 属性 → 允许执行
```

## 五、安装包定制化

### 5.1 图标和品牌

```yaml
# 图标配置
win:
  icon: build/icon.ico          # 256x256 或更大

mac:
  icon: build/icon.icns         # 1024x1024

linux:
  icon: build/icons             # 目录，包含多尺寸 PNG

# 目录结构
build/
├── icon.ico                    # Windows
├── icon.icns                   # macOS
└── icons/                      # Linux
    ├── 16x16.png
    ├── 32x32.png
    ├── 48x48.png
    ├── 64x64.png
    ├── 128x128.png
    ├── 256x256.png
    └── 512x512.png
```

### 5.2 产物命名

```yaml
# 自定义产物文件名
artifactName: "${productName}-${version}-${os}-${arch}.${ext}"

# 输出示例：
# MyApp-1.0.0-win-x64.exe
# MyApp-1.0.0-mac-arm64.dmg
# MyApp-1.0.0-linux-x64.AppImage

# 可用变量：
# ${productName} - 产品名称
# ${version}     - 版本号
# ${name}        - package.json name
# ${arch}        - 架构
# ${os}          - 操作系统
# ${platform}    - 同 os
# ${ext}         - 扩展名
```

### 5.3 元数据

```yaml
appId: com.example.myapp
productName: My Application
copyright: Copyright © 2024 My Company

win:
  publisherName: My Company
  legalTrademarks: MyApp is a trademark
  
mac:
  category: public.app-category.productivity
  
linux:
  category: Utility
  synopsis: Short description
  description: Longer description
```

## 六、安装/卸载行为控制

### 6.1 Windows 安装行为

```yaml
nsis:
  oneClick: false                          # 向导式安装
  allowToChangeInstallationDirectory: true # 允许选择目录
  perMachine: false                        # 安装到用户目录
  createDesktopShortcut: true
  createStartMenuShortcut: true
  runAfterFinish: true                     # 安装后启动
  deleteAppDataOnUninstall: false          # 卸载不删用户数据
```

### 6.2 安装目录

```
Windows 安装位置：
├── perMachine: false → C:\Users\xxx\AppData\Local\Programs\MyApp\
└── perMachine: true  → C:\Program Files\MyApp\

macOS 安装位置：
└── /Applications/MyApp.app

Linux 安装位置：
├── AppImage: 用户下载目录（自选）
├── deb/rpm: /opt/myapp 或 /usr/share/myapp
└── snap: /snap/myapp
```

### 6.3 卸载残留

```yaml
# 卸载时清理用户数据
nsis:
  deleteAppDataOnUninstall: true

# 或在 installer.nsh 中自定义
!macro customUnInstall
  MessageBox MB_YESNO "是否删除用户数据？" IDYES remove IDNO keep
  remove:
    RMDir /r "$APPDATA\${PRODUCT_NAME}"
  keep:
!macroend
```
