> Electron 构建过程中的问题诊断与解决方案

## 一、构建问题诊断

### 1.1 启用详细日志

```bash
# 方式一：环境变量
DEBUG=electron-builder npm run dist

# 方式二：更详细的日志
DEBUG=electron-builder,electron-builder:* npm run dist

# 方式三：指定日志级别
ELECTRON_BUILDER_LOG_LEVEL=debug npm run dist

# Windows PowerShell
$env:DEBUG="electron-builder"
npm run dist
```

### 1.2 electron-builder 日志级别

```javascript
// 在 package.json 或 electron-builder.js 中
{
  "build": {
    // 构建时的日志级别
    // info | warn | error | debug
  }
}
```

### 1.3 常见构建错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `Cannot find module` | 依赖缺失或路径错误 | 检查 package.json，重新 npm install |
| `ENOENT: no such file` | 文件路径配置错误 | 检查 files/extraResources 配置 |
| `Exit code: ENOENT` | 外部工具缺失 | 安装对应工具（如 NSIS、Wine） |
| `Error: spawn UNKNOWN` | Windows 权限问题 | 以管理员运行或检查杀毒软件 |
| `Cannot create symlink` | Windows 符号链接权限 | 启用开发者模式或管理员运行 |
| `EPERM: operation not permitted` | 文件被占用 | 关闭相关程序，重试 |

## 二、打包产物检查

### 2.1 检查目录结构

```bash
# 只生成目录，不打包
npx electron-builder --dir

# 查看生成的目录结构
ls -la dist/win-unpacked/
ls -la dist/mac/MyApp.app/Contents/

# Windows 产物结构
dist/win-unpacked/
├── MyApp.exe
├── resources/
│   ├── app.asar
│   └── app.asar.unpacked/
├── locales/
└── *.dll

# macOS 产物结构
dist/mac/MyApp.app/Contents/
├── MacOS/MyApp
├── Resources/
│   ├── app.asar
│   └── electron.icns
├── Frameworks/
└── Info.plist
```

### 2.2 检查 asar 内容

```bash
# 安装 asar 工具
npm install -g @electron/asar

# 列出 asar 内容
asar list dist/win-unpacked/resources/app.asar

# 提取 asar 内容
asar extract dist/win-unpacked/resources/app.asar extracted/

# 查看特定文件
asar extract-file dist/win-unpacked/resources/app.asar package.json

# 验证文件是否在 asar 中
asar list app.asar | grep "main.js"
```

### 2.3 文件完整性检查

```javascript
// scripts/verify-build.js
const fs = require('fs')
const path = require('path')
const asar = require('@electron/asar')

const requiredFiles = [
  'package.json',
  'dist/main/index.js',
  'dist/preload/index.js',
  'dist/renderer/index.html'
]

function verifyAsar(asarPath) {
  console.log('Verifying:', asarPath)
  
  const files = asar.listPackage(asarPath)
  
  for (const required of requiredFiles) {
    if (!files.includes(required)) {
      console.error(`Missing: ${required}`)
      process.exit(1)
    }
  }
  
  console.log('All required files present')
}

verifyAsar('dist/win-unpacked/resources/app.asar')
```

## 三、安装包测试

### 3.1 NSIS 日志模式

```bash
# 方式一：安装时按 Ctrl+Alt+T 显示日志窗口

# 方式二：命令行参数
MyApp-Setup.exe /D=C:\MyApp /LOG=install.log

MyApp-Setup.exe /S /LOG=install.log

# 查看 NSIS 调试输出
# 在 installer.nsh 中添加
!macro customInstall
  DetailPrint "Installing to: $INSTDIR"
  DetailPrint "User: $USERNAME"
  DetailPrint "OS: $%OS%"
!macroend
```

### 3.2 macOS 安装验证

```bash
# 验证签名
codesign --verify --deep --strict --verbose=2 MyApp.app

# 显示签名详情
codesign -dv --verbose=4 MyApp.app

# 检查权限
codesign -d --entitlements :- MyApp.app

# Gatekeeper 评估
spctl -a -vvv -t execute MyApp.app

# 验证公证
stapler validate MyApp.app

# DMG 验证
hdiutil verify MyApp.dmg
```

## 四、常见问题清单

### 4.1 白屏问题

```javascript
// 症状：应用启动后显示白屏

// 原因1：资源路径错误
// 检查 loadFile 或 loadURL 路径
mainWindow.loadFile(path.join(__dirname, '../renderer/index.html'))
// 生产环境路径可能不同

// 原因2：file:// 协议问题
// Vite 等工具的 base 配置
// vite.config.js
export default {
  base: './'  // 相对路径，而非 '/'
}

// 原因3：CSP 阻止
// 检查 Content-Security-Policy

// 调试方法：
mainWindow.webContents.openDevTools()
mainWindow.webContents.on('did-fail-load', (event, code, desc) => {
  console.error('Load failed:', code, desc)
})
```

### 4.2 Native 模块加载失败

```javascript
// 错误信息
// Error: The module was compiled against a different Node.js version

// 解决方案
// 1. 重新编译
npx electron-rebuild -f

// 2. 检查 asarUnpack 配置
asarUnpack:
  - "**/*.node"
  - "**/node_modules/your-native-module/**"

// 3. 验证模块加载
try {
  require('native-module')
} catch (err) {
  console.error('Native module load failed:', err)
  console.log('Module path:', require.resolve('native-module'))
}
```

### 4.3 资源文件找不到

```javascript
// 问题：打包后找不到资源文件

// 检查点：
// 1. 文件是否在 files 配置中包含
// 2. 路径是否正确（开发 vs 生产）

// 统一路径处理
const getResourcePath = (relativePath) => {
  if (app.isPackaged) {
    return path.join(process.resourcesPath, relativePath)
  }
  return path.join(__dirname, '..', relativePath)
}

// 调试
console.log('App path:', app.getAppPath())
console.log('Resources path:', process.resourcesPath)
console.log('__dirname:', __dirname)
```

### 4.4 签名验证失败

```bash
# Windows 签名问题
# 错误：The signature is invalid

# 验证签名
signtool verify /pa MyApp.exe

# macOS 签名问题
# 错误：code signature invalid

# 重新签名
codesign --force --deep --sign "Developer ID Application: ..." MyApp.app

# 检查问题
codesign --verify --verbose MyApp.app 2>&1
```

### 4.5 更新失败

```javascript
// 调试更新
const { autoUpdater } = require('electron-updater')
const log = require('electron-log')

autoUpdater.logger = log
autoUpdater.logger.transports.file.level = 'debug'

// 检查更新源
console.log('Feed URL:', autoUpdater.getFeedURL())

// 手动测试更新检查
autoUpdater.checkForUpdates().then(result => {
  console.log('Update check result:', result)
}).catch(err => {
  console.error('Update check failed:', err)
})
```

### 4.6 权限问题

```javascript
// Windows UAC 问题
// 检查 requestedExecutionLevel
win:
  requestedExecutionLevel: asInvoker  # 不需要管理员
  # 或 requireAdministrator 如果确实需要

// macOS 权限
// 检查 entitlements.plist
// 确保包含必要权限

// Linux 权限
// AppImage 需要可执行权限
chmod +x MyApp.AppImage
```

## 五、日志收集

### 5.1 应用日志

```javascript
// 使用 electron-log
const log = require('electron-log')

// 配置日志路径
log.transports.file.resolvePath = () => {
  return path.join(app.getPath('userData'), 'logs', 'main.log')
}

// 日志级别
log.transports.file.level = 'info'
log.transports.console.level = 'debug'

// 使用
log.info('App started')
log.error('Error occurred:', error)

// 捕获未处理异常
process.on('uncaughtException', (error) => {
  log.error('Uncaught exception:', error)
})
```

### 5.2 构建日志

```bash
# 保存构建日志
npm run dist 2>&1 | tee build.log

# 分析日志
grep -i error build.log
grep -i warning build.log
```

### 5.3 崩溃报告

```javascript
// 启用 Electron 崩溃报告
const { crashReporter } = require('electron')

crashReporter.start({
  productName: 'MyApp',
  companyName: 'MyCompany',
  submitURL: 'https://crashes.example.com/submit',
  uploadToServer: true
})

// 崩溃日志位置
// Windows: %APPDATA%\MyApp\Crashpad
// macOS: ~/Library/Application Support/MyApp/Crashpad
// Linux: ~/.config/MyApp/Crashpad
```

## 六、调试工具

### 6.1 Electron DevTools

```javascript
// 开发环境自动打开
if (!app.isPackaged) {
  mainWindow.webContents.openDevTools()
}

// 生产环境快捷键打开
app.on('ready', () => {
  globalShortcut.register('CommandOrControl+Shift+I', () => {
    const win = BrowserWindow.getFocusedWindow()
    if (win) {
      win.webContents.toggleDevTools()
    }
  })
})
```

### 6.2 远程调试

{
  "type": "node",
  "request": "launch",
  "name": "Electron Main",
  "runtimeExecutable": "${workspaceFolder}/node_modules/.bin/electron",

# 详细日志构建
