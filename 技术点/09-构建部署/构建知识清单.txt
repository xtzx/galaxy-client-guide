# Electron 构建知识体系清单

> 目标：完整、深入、实战导向的 Electron 构建知识体系
> 说明：Web 前端构建部分（Vite/Webpack）仅简要提及

---

## 1) 基础概念

### 1.1 Electron 架构与进程模型
- Main 进程
  - Node.js 运行时环境
  - 窗口管理（BrowserWindow 生命周期）
  - 系统级 API 访问（文件系统、托盘、菜单、对话框）
  - 应用生命周期管理（app 模块）
- Renderer 进程
  - Chromium 渲染引擎
  - 每个窗口一个独立进程
  - 默认受限的 Node.js 访问（安全考量）
- Preload 脚本
  - Main 与 Renderer 之间的安全桥梁
  - contextBridge API 暴露
  - 沙箱模式下的能力边界
- IPC 通信机制
  - ipcMain / ipcRenderer
  - invoke/handle 模式（Promise 化）
  - 单向 vs 双向通信
  - 通信性能与数据序列化

### 1.2 安全模型
- 上下文隔离（contextIsolation）
- 沙箱模式（sandbox）
- nodeIntegration 的风险与使用场景
- webSecurity 与 CSP 配置

### 1.3 两类构建产物
- 可运行目录（unpacked）
  - 开发调试用途
  - 目录结构解析
  - app.asar 与 app.asar.unpacked
- 分发包（installer）
  - Windows: NSIS / MSI / Squirrel / portable
  - macOS: DMG / PKG / ZIP
  - Linux: AppImage / deb / rpm / snap / flatpak

### 1.4 Electron 版本管理
- Electron 与 Node.js / Chromium 版本对应关系
- 版本升级的影响评估
- LTS 策略与版本选择

## 3) package.json scripts 组织

### 3.1 标准脚本命名
- dev / start：开发模式启动
- build:renderer：构建前端资源
- build:main / build:preload：构建主进程代码
- build：完整构建（renderer + main）
- pack：生成可运行目录（不出安装包）
- dist：生成安装包
- publish：打包并发布（CI 专用）
- clean：清理构建产物
- rebuild：重编译 native modules
- lint / typecheck：代码检查

  - 渠道号注入
- 条件构建
  - 是否签名
  - 是否 publish
  - 目标平台/架构

### 3.3 跨平台脚本兼容
- cross-env 使用

## 4) electron-builder 深入

### 4.1 配置文件格式与位置
- package.json 内嵌 build 字段
- electron-builder.yml / .json / .js
- 配置继承（extends）
- 多配置文件策略（不同渠道/环境）

### 4.2 核心配置项详解
- appId：应用唯一标识（影响用户数据路径、协议注册）
- productName：显示名称
- files：打包包含/排除规则（glob 语法）
- extraFiles：复制到安装目录根目录
- extraResources：复制到 resources 目录
- asar / asarUnpack：归档配置
- directories：
  - output：输出目录
  - buildResources：构建资源目录（图标等）
  - app：应用源码目录
- artifactName：产物命名模板（${productName}-${version}-${arch}.${ext}）
- compression：压缩级别（store/normal/maximum）
- npmRebuild：是否自动 rebuild

### 4.3 publish 配置
- provider 类型：
  - generic（HTTP 服务器）
  - github（GitHub Releases）
  - s3 / spaces / gcs（对象存储）
  - custom
- 多渠道发布配置
- private 仓库认证

### 4.4 钩子函数
- beforeBuild：构建前
- afterPack：打包后（安装包生成前）
- afterSign：签名后
- afterAllArtifactBuild：所有产物生成后
- 钩子应用场景：
  - 注入额外文件
  - 修改产物
  - 通知 / 上传

### 4.5 构建缓存与优化
- electron-builder 缓存目录
- node_modules 缓存
- Electron 二进制缓存
- 增量构建策略

## 5) Windows 打包重点（NSIS）

### 5.1 NSIS 配置项全解
- oneClick：一键安装 vs 向导式安装
- allowElevation：是否允许提升权限
- allowToChangeInstallationDirectory：是否允许选择安装目录
- perMachine：系统级安装 vs 用户级安装
- createDesktopShortcut / createStartMenuShortcut
- runAfterFinish：安装完成后启动
- deleteAppDataOnUninstall：卸载时清理用户数据
- installerIcon / uninstallerIcon：安装/卸载图标
- installerHeader / installerHeaderIcon：安装向导头部
- installerSidebar / uninstallerSidebar：侧边栏图片
- license：许可协议文件
- language：安装语言
- unicode：是否使用 Unicode NSIS

### 5.2 installer.nsh 自定义脚本
- NSIS 脚本基础语法
- 常用宏与函数
- 自定义页面
- 安装前检查：
  - 检测是否已运行（互斥锁）
  - 检测旧版本并提示卸载
  - 系统要求检查
- 安装过程控制：
  - 杀死正在运行的进程
  - 备份/迁移用户配置
  - 清理旧版本残留
- 注册表操作
- 环境变量设置
- 文件关联注册
- 协议（URL Scheme）注册
- 安装驱动/证书（高级场景）
- 卸载钩子

### 5.3 win 平台配置
- target 类型：
  - nsis：标准安装包
  - nsis-web：在线安装包
  - portable：绿色版
  - appx：Windows Store
  - msi：企业部署
  - squirrel：自动更新框架
- icon：应用图标
- requestedExecutionLevel：
  - asInvoker：普通权限
  - requireAdministrator：管理员权限
  - highestAvailable：最高可用权限
- publisherName：发布者名称
- legalTrademarks：法律商标
- fileAssociations：文件关联
- protocols：协议注册

### 5.4 单实例控制

- 配合 NSIS 安装检测
- 多实例场景处理

### 5.5 Windows 特殊问题

  - 路径长度问题规避（Windows）
