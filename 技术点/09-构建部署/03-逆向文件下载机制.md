# 逆向文件下载机制详解

> `downloadDevFile.js` 脚本执行逻辑

---

## 一、脚本概述

### 1.1 用途

`scripts/downloadDevFile.js` 用于从内部服务器下载逆向服务相关文件，这些文件是与微信/企微交互的核心组件。

### 1.2 使用方法

```bash
# 下载指定版本
node ./scripts/downloadDevFile.js 5.4.2-release01

# 下载默认版本
node ./scripts/downloadDevFile.js
```

## 二、执行流程

### 2.1 流程图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    downloadDevFile 执行流程                                  │
└─────────────────────────────────────────────────────────────────────────────┘

1️⃣ 解析命令行参数
   │
   │  process.argv.splice(2) → ["5.4.2-release01"] 或 []
   │  currentVersion = "5.4.2-release01" 或 "default"
   │
   ▼
2️⃣ 读取版本配置 (versionConfig.json)
   │
   │  查找对应版本的下载地址
   │  如果版本不存在，使用 "default" 配置
   │
   │  ┌───────────────────────────────────────────────────────────────────┐
   │  │  versionConfig["5.4.2-release01"] 或 versionConfig["default"]:   │
   │  │  {                                                               │
   │  │    "sirius_dll_path": "http://10.252.71.71:9091/.../Release.zip",│
   │  │    "basicService_exe_path": "http://10.252.71.71:9091/...",     │
   │  │    "galaxy64_dll_path": "http://10.252.71.71:9091/..."          │
   │  │  }                                                               │
   │  └───────────────────────────────────────────────────────────────────┘
   │
   ▼
3️⃣ 清空目标目录
   │
   │  targetDir = ./extraResources/Inject/
   │  递归删除该目录下所有文件
   │
   ▼
4️⃣ 循环处理每个下载项
   │
   │  for (const [key, val] of Object.entries(currentVersionConfig))
   │  │
   │  │  ┌───────────────────────────────────────────────────────────┐
   │  │  │  以 sirius_dll_path 为例:                                 │
   │  │  │                                                          │
   │  │  │  a. unzipper.Open.url(request, url)                     │
   │  │  │     从远程 URL 直接打开 zip 文件（流式处理）             │
   │  │  │                                                          │
   │  │  │  b. 过滤 mock 文件                                       │
   │  │  │     files.filter(f => f.path.indexOf("mock") === -1)    │
   │  │  │                                                          │
   │  │  │  c. 并行写入所有文件                                     │
   │  │  │     Promise.all(files.map(file => writeStream(file)))   │
   │  │  └───────────────────────────────────────────────────────────┘
   │  │
   │  │  继续处理下一个 (basicService_exe_path, galaxy64_dll_path...)
   │
   ▼
5️⃣ 完成
   │
   │  extraResources/Inject/ 目录内容:
   │  ├── BasicService.exe    ← 逆向服务主程序
   │  ├── Sirius.dll          ← 微信注入 DLL
   │  ├── PipeCore.dll        ← IPC 通信 DLL
   │  └── Galaxy64.dll        ← 64位版本 DLL
```

### 2.2 核心代码解析

```javascript
// 1. 解析参数
const params = process.argv.splice(2);
const currentVersion = params[0] || 'default';

// 2. 读取版本配置
const currentVersionConfig =
    currentVersion in versionConfig
        ? versionConfig[currentVersion]
        : versionConfig.default;

// 3. 清空目标目录
const targetDir = path.resolve(__dirname, "../extraResources/Inject");
deleteAllFilesInDirectorySync(targetDir);

// 4. 下载并解压
for (const [key, val] of Object.entries(currentVersionConfig)) {
    // 从 URL 直接打开 zip（流式，无需先下载）
    const directory = await unzipper.Open.url(request, val);
    
    // 过滤掉 mock 文件
    const files = directory.files.filter(
        (file) => file.path.indexOf("mock") === -1
    );
    
    // 并行写入所有文件
    const promises = files.map((file) => {
        return new Promise((resolve, reject) => {
            const writeStream = fs.createWriteStream(
                path.resolve(targetDir, file.path)
            );
            file.stream()
                .pipe(writeStream)
                .on("finish", resolve)
                .on("error", reject);
        });
    });
    
    await Promise.all(promises);
}
```

## 三、版本配置详解

### 3.1 versionConfig.json 结构

```json
{
    "5.4.2-release01": {
        "sirius_dll_path": "http://10.252.71.71:9091/Sirius/1039/.../Release.zip",
        "basicService_exe_path": "http://10.252.71.71:9091/BaseService/1012/.../Release.zip",
        "galaxy64_dll_path": "http://10.252.71.71:9091/Galaxy/2.0.1/.../Release.zip"
    },
    "default": {
        "sirius_dll_path": "http://10.252.71.71:9091/Sirius/1039/.../Release.zip",
        "basicService_exe_path": "http://10.252.71.71:9091/BaseService/1012/.../Release.zip",
        "galaxy64_dll_path": "http://10.252.71.71:9091/Galaxy/2.0.1/.../Release.zip"
    }
}
```

### 3.2 下载文件说明

| 配置项 | 下载的文件 | 用途 |
|-------|----------|-----|
| `sirius_dll_path` | Sirius.dll, PipeCore.dll | 微信注入、IPC 通信 |
| `basicService_exe_path` | BasicService.exe | 逆向服务主程序 |
| `galaxy_dll_path` | Galaxy.dll | 32位微信支持 |
| `galaxy64_dll_path` | Galaxy64.dll | 64位微信支持 |

### 3.3 下载地址格式

```
http://10.252.71.71:9091/{项目}/{版本}/{Hash}/Release.zip

示例:
http://10.252.71.71:9091/Sirius/1039/40f449fc10bbf02c5f6ae37c76b44aa0/Release.zip
                        │       │    │
                        项目    版本  构建Hash
```

## 四、注意事项

### 4.1 网络要求

- 下载服务器地址 `10.252.71.71:9091` 是**内网地址**
- 需要连接公司 VPN 或在内网环境才能访问
- 如果下载超时，检查网络连接

### 4.2 目录权限

- 目标目录 `extraResources/Inject/` 需要写入权限
- Windows 下可能需要管理员权限

### 4.3 版本匹配

- 版本号需要与 `versionConfig.json` 中的 key 匹配
- 如果版本不存在，会自动使用 `default` 配置
- 新版本发布时需要更新 `versionConfig.json`

### 4.4 Git 忽略

这些文件被 `.gitignore` 忽略，不会提交到仓库：

```gitignore
/extraResources/Inject/*.exe
/extraResources/Inject/*.dll
/extraResources/Inject/*.pdb
```

## 五、常见问题

### Q1: 下载失败怎么办？

```bash
# 1. 检查网络（需要内网或VPN）
ping 10.252.71.71

# 2. 检查版本号是否存在
cat scripts/versionConfig.json | grep "5.4.2-release01"

# 3. 手动下载测试
curl http://10.252.71.71:9091/Sirius/1039/.../Release.zip -o test.zip
```

### Q2: 新版本如何添加？

1. 获取新版本的下载地址（从逆向团队获取）
2. 编辑 `scripts/versionConfig.json`
3. 添加新版本配置

```json
{
    "5.5.0-release01": {
        "sirius_dll_path": "http://10.252.71.71:9091/Sirius/新版本/新Hash/Release.zip",
        "basicService_exe_path": "...",
        "galaxy64_dll_path": "..."
    }
}
```

### Q3: 为什么过滤 mock 文件？

```javascript
files.filter((file) => file.path.indexOf("mock") === -1)
```

zip 包中可能包含测试用的 mock 文件，生产环境不需要。
