# 安全配置与代码保护

> Electron 应用的安全最佳实践与源码保护策略

---

## 一、Electron 安全模型

### 1.1 安全威胁

```
┌─────────────────────────────────────────────────────────────────┐
│                    Electron 安全威胁                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 远程代码执行 (RCE)                                          │
│     ├── 渲染进程直接访问 Node.js                                │
│     ├── 加载恶意远程内容                                        │
│     └── XSS 攻击升级为系统访问                                  │
│                                                                 │
│  2. 源码泄露                                                    │
│     ├── asar 可以轻易解包                                       │
│     ├── 业务逻辑暴露                                            │
│     └── API 密钥泄露                                            │
│                                                                 │
│  3. 中间人攻击                                                   │
│     ├── HTTP 请求被拦截                                        │
│     ├── 更新包被篡改                                            │
│     └── 证书验证绕过                                            │
│                                                                 │
│  4. 本地权限滥用                                                 │
│     ├── 读写任意文件                                            │
│     ├── 执行系统命令                                            │
│     └── 访问敏感数据                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 安全配置层次

```
应用安全配置层次：

1. BrowserWindow 配置
   └── webPreferences 安全选项

2. Preload 脚本
   └── contextBridge 暴露最小 API

3. 内容安全策略 (CSP)
   └── 限制资源加载

4. 代码签名
   └── 防止篡改

5. 源码保护
   └── 防止逆向
```

## 二、Preload 安全最佳实践

### 2.1 推荐配置

```javascript
// main.js - 安全的 BrowserWindow 配置
const mainWindow = new BrowserWindow({
  width: 1200,
  height: 800,
  webPreferences: {
    // ✓ 必须开启
    contextIsolation: true,      // 隔离上下文
    nodeIntegration: false,      // 禁用 Node.js
    sandbox: true,               // 启用沙箱
    
    // ✓ 推荐配置
    webSecurity: true,           // 启用同源策略
    allowRunningInsecureContent: false,  // 禁止 HTTPS 页面加载 HTTP
    enableRemoteModule: false,   // 禁用 remote 模块（已废弃）
    
    // Preload 脚本
    preload: path.join(__dirname, 'preload.js')
  }
})

// 加载远程内容时的额外保护
if (loadingRemoteContent) {
  mainWindow.webContents.on('will-navigate', (event, url) => {
    if (!isAllowedUrl(url)) {
      event.preventDefault()
    }
  })
  
  mainWindow.webContents.setWindowOpenHandler(({ url }) => {
    if (!isAllowedUrl(url)) {
      return { action: 'deny' }
    }
    return { action: 'allow' }
  })
}
```

### 2.2 安全的 Preload 脚本

```javascript
// preload.js - 最小权限原则
const { contextBridge, ipcRenderer } = require('electron')

// ✓ 好的做法：只暴露必要的、安全的 API
contextBridge.exposeInMainWorld('api', {
  // 获取版本
  getVersion: () => ipcRenderer.invoke('app:getVersion'),
  
  // 打开外部链接（经过主进程验证）
  openExternal: (url) => ipcRenderer.invoke('shell:openExternal', url),
  
  // 读取设置（不直接访问文件系统）
  getSetting: (key) => ipcRenderer.invoke('settings:get', key),
  setSetting: (key, value) => ipcRenderer.invoke('settings:set', key, value)
})

// ✗ 不要这样做！
// contextBridge.exposeInMainWorld('fs', require('fs'))
// contextBridge.exposeInMainWorld('ipc', ipcRenderer)
// contextBridge.exposeInMainWorld('require', require)
```

### 2.3 主进程验证

```javascript
// main.js - 验证渲染进程请求
const { ipcMain, shell } = require('electron')

// 白名单 URL
const ALLOWED_EXTERNAL_URLS = [
  /^https:\/\/example\.com/,
  /^https:\/\/github\.com/
]

ipcMain.handle('shell:openExternal', async (event, url) => {
  // 验证 URL
  const isAllowed = ALLOWED_EXTERNAL_URLS.some(pattern => pattern.test(url))
  
  if (!isAllowed) {
    throw new Error('URL not allowed')
  }
  
  await shell.openExternal(url)
})

ipcMain.handle('settings:get', async (event, key) => {
  // 验证 key 是否合法
  const allowedKeys = ['theme', 'language', 'fontSize']
  
  if (!allowedKeys.includes(key)) {
    throw new Error('Invalid setting key')
  }
  
  return store.get(key)
})
```

## 三、内容安全策略 (CSP)

### 3.1 HTML 中配置

```html
<!DOCTYPE html>
<html>
<head>
  <meta http-equiv="Content-Security-Policy" content="
    default-src 'self';
    script-src 'self';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: https:;
    font-src 'self' data:;
    connect-src 'self' https://api.example.com;
    media-src 'self';
    object-src 'none';
    frame-src 'none';
  ">
</head>
<body>
  ...
</body>
</html>
```

### 3.2 主进程中配置

```javascript
// main.js
const { session } = require('electron')

app.whenReady().then(() => {
  // 为所有请求添加 CSP 头
  session.defaultSession.webRequest.onHeadersReceived((details, callback) => {
    callback({
      responseHeaders: {
        ...details.responseHeaders,
        'Content-Security-Policy': [
          "default-src 'self'; " +
          "script-src 'self'; " +
          "style-src 'self' 'unsafe-inline'; " +
          "img-src 'self' data: https:; " +
          "connect-src 'self' https://api.example.com"
        ]
      }
    })
  })
})
```

### 3.3 CSP 指令说明

| 指令 | 说明 | 推荐值 |
|------|------|--------|
| `default-src` | 默认策略 | `'self'` |
| `script-src` | JavaScript 来源 | `'self'` |
| `style-src` | CSS 来源 | `'self' 'unsafe-inline'` |
| `img-src` | 图片来源 | `'self' data: https:` |
| `connect-src` | XHR/WebSocket | `'self'` + API 域名 |
| `frame-src` | iframe 来源 | `'none'` |
| `object-src` | 插件 | `'none'` |

## 四、代码保护方案

### 4.1 asar 的局限性

```
asar 不是加密！

可以轻易解包：
$ asar extract app.asar ./output

asar 只提供：
✓ 路径长度问题规避
✓ 一定的"障碍"（非技术用户）
✗ 不能防止有意的逆向
✗ 不是加密
```

### 4.2 代码混淆

```javascript
// 使用 terser/uglify 混淆
// vite.config.js
export default {
  build: {
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true
      },
      mangle: {
        toplevel: true,
        properties: {
          regex: /^_/  // 混淆 _ 开头的属性
        }
      },
      format: {
        comments: false
      }
    }
  }
}

// esbuild 混淆
{
  minify: true,
  keepNames: false,
  drop: ['console', 'debugger']
}
```

### 4.3 字节码编译（bytenode）

```bash
# 安装
npm install bytenode

# 编译为 V8 字节码
npx bytenode -c main.js
# 输出 main.jsc
```

```javascript
// 加载字节码
require('bytenode')
require('./main.jsc')
```

**注意事项**：
- 字节码与 Node/Electron 版本绑定
- 仍可被反编译（但难度增加）
- 主要用于主进程代码

### 4.4 asar 加密（第三方）

```javascript
// 使用 asarmor 等工具
// npm install asarmor

// 加密 asar
const asarmor = require('asarmor')

asarmor.encrypt({
  src: 'dist/resources/app.asar',
  dst: 'dist/resources/app.asar',
  key: process.env.ASAR_KEY
})

// 需要在应用启动时解密
// 增加了复杂度，但不是完美保护
```

### 4.5 保护建议

```
┌─────────────────────────────────────────────────────────────────┐
│                    代码保护建议                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  现实情况：                                                      │
│  ├── 没有完美的客户端代码保护                                    │
│  ├── 足够有决心的攻击者总能逆向                                  │
│  └── 保护的目标是提高门槛                                        │
│                                                                 │
│  推荐策略：                                                      │
│  1. 核心逻辑放服务端                                            │
│  2. 使用代码混淆                                                │
│  3. 敏感操作需要服务端验证                                       │
│  4. 不要在客户端存储真正的密钥                                   │
│  5. 接受客户端代码可能被查看的现实                               │
│                                                                 │
│  性价比排序：                                                    │
│  1. 代码混淆（简单有效）                                        │
│  2. 关键逻辑服务端化（根本解决）                                 │
│  3. 字节码编译（增加难度）                                       │
│  4. asar 加密（有限保护）                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 五、敏感信息处理

### 5.1 不要硬编码密钥

```javascript
// ✗ 不要这样做
const API_KEY = 'sk-1234567890abcdef'
const DB_PASSWORD = 'mysecretpassword'

// ✓ 使用环境变量（构建时注入）
const API_URL = process.env.VITE_API_URL

// ✓ 或从服务端获取
async function getApiKey() {
  const response = await fetch('https://api.example.com/auth/key')
  return response.json()
}
```

### 5.2 安全存储

```javascript
// 使用系统密钥链（keytar）
const keytar = require('keytar')

// 存储
await keytar.setPassword('myapp', 'user@example.com', 'secret-token')

// 读取
const token = await keytar.getPassword('myapp', 'user@example.com')

// 删除
await keytar.deletePassword('myapp', 'user@example.com')
```

```javascript
// 或使用 electron-store 加密
const Store = require('electron-store')

const store = new Store({
  encryptionKey: 'your-encryption-key',  // 但这个 key 怎么保护？
  // 更好的做法：从系统密钥链获取加密 key
})

store.set('token', 'secret-value')
```

### 5.3 安全通信

```javascript
// 只使用 HTTPS
const apiUrl = 'https://api.example.com'  // ✓
const apiUrl = 'http://api.example.com'   // ✗

// 验证证书
const https = require('https')

const options = {
  hostname: 'api.example.com',
  rejectUnauthorized: true  // 不要设为 false
}

// 证书固定（高安全场景）
const tls = require('tls')
const trustedCert = fs.readFileSync('trusted-cert.pem')

tls.connect({
  host: 'api.example.com',
  port: 443,
  checkServerIdentity: (host, cert) => {
    if (cert.fingerprint !== 'expected-fingerprint') {
      throw new Error('Certificate mismatch')
    }
  }
})
```

## 六、代码签名

### 6.1 为什么要签名

```
代码签名的作用：

1. 证明来源
   └── 用户知道应用来自可信开发者

2. 完整性验证
   └── 检测是否被篡改

3. 系统信任
   └── 减少安全警告（SmartScreen、Gatekeeper）

4. 自动更新安全
   └── 验证更新包未被篡改
```

### 6.2 签名验证

```javascript
// 运行时验证签名完整性
const { app } = require('electron')
const { execSync } = require('child_process')

function verifySignature() {
  if (process.platform === 'darwin') {
    try {
      execSync(`codesign --verify --deep --strict "${app.getAppPath()}"`)
      return true
    } catch {
      return false
    }
  }
  
  if (process.platform === 'win32') {
    // Windows 签名验证需要其他工具
  }
  
  return true  // Linux 通常不签名
}

if (!verifySignature()) {
  dialog.showErrorBox('安全警告', '应用签名验证失败，可能已被篡改')
  app.quit()
}
```

## 八、完整安全配置示例

```javascript
// main.js - 安全配置示例

const { app, BrowserWindow, session, ipcMain, shell } = require('electron')
const path = require('path')

// 安全的窗口配置
function createSecureWindow() {
  const win = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
      contextIsolation: true,
      nodeIntegration: false,
      sandbox: true,
      webSecurity: true,
      allowRunningInsecureContent: false,
      preload: path.join(__dirname, 'preload.js')
    }
  })
  
  // 导航限制
  win.webContents.on('will-navigate', (event, url) => {
    const allowed = url.startsWith('file://') || 
                    url.startsWith('http://localhost')
    if (!allowed) {
      event.preventDefault()
    }
  })
  
  // 新窗口限制
  win.webContents.setWindowOpenHandler(({ url }) => {
    // 外部链接在浏览器打开
    if (url.startsWith('https://')) {
      shell.openExternal(url)
    }
    return { action: 'deny' }
  })
  
  return win
}

// CSP 配置
app.whenReady().then(() => {
  session.defaultSession.webRequest.onHeadersReceived((details, callback) => {
    callback({
      responseHeaders: {
        ...details.responseHeaders,
        'Content-Security-Policy': [
          "default-src 'self'; " +
          "script-src 'self'; " +
          "style-src 'self' 'unsafe-inline'; " +
          "img-src 'self' data: https:; " +
          "connect-src 'self' https://api.example.com"
        ]
      }
    })
  })
  
  createSecureWindow()
})

// 安全的 IPC 处理
const ALLOWED_EXTERNAL_HOSTS = ['example.com', 'github.com']

ipcMain.handle('shell:openExternal', async (event, url) => {
  try {
    const { hostname } = new URL(url)
    if (!ALLOWED_EXTERNAL_HOSTS.includes(hostname)) {
      throw new Error('Host not allowed')
    }
    await shell.openExternal(url)
  } catch (error) {
    console.error('openExternal failed:', error)
    throw error
  }
})
```

```javascript
// preload.js - 安全的 API 暴露

const { contextBridge, ipcRenderer } = require('electron')

// 只暴露必要的、安全的 API
contextBridge.exposeInMainWorld('api', {
  versions: {
    node: process.versions.node,
    chrome: process.versions.chrome,
    electron: process.versions.electron
  },
  
  openExternal: (url) => {
    // 基本验证
    if (typeof url !== 'string' || !url.startsWith('https://')) {
      throw new Error('Invalid URL')
    }
    return ipcRenderer.invoke('shell:openExternal', url)
  },
  
  settings: {
    get: (key) => ipcRenderer.invoke('settings:get', key),
    set: (key, value) => ipcRenderer.invoke('settings:set', key, value)
  }
})
```
