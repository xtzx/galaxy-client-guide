# 10-安全与权限管理

> 安全使用 SLS 服务

---

## 一、账号体系

### 1.1 账号类型

| 类型 | 说明 | 权限级别 |
|-----|-----|---------|
| 主账号 | 阿里云账号所有者 | 最高权限 |
| RAM 用户 | 子账号 | 按需授权 |
| RAM 角色 | 临时身份 | 临时权限 |

### 1.2 推荐做法

- **不使用主账号**进行日常操作
- 为每个用户/应用创建 **RAM 用户**
- 使用 **RAM 角色**进行临时授权
- 遵循**最小权限原则**

## 二、RAM 用户管理

### 2.1 创建 RAM 用户

1. 登录 RAM 控制台
2. 选择「用户」→「创建用户」
3. 填写用户信息
4. 选择访问方式：
   - 控制台访问：需要登录
   - 编程访问：使用 AccessKey

### 2.2 用户命名规范

```
{项目}-{用途}-{环境}
```

示例：
- `galaxy-client-prod`
- `galaxy-query-dev`
- `sls-readonly-admin`

### 2.3 用户分类

| 用途 | 权限 | 示例用户 |
|-----|-----|---------|
| 日志写入 | 只写 | `galaxy-writer` |
| 日志查询 | 只读 | `galaxy-reader` |
| 管理员 | 读写+配置 | `galaxy-admin` |

## 三、权限策略

### 3.1 系统策略

| 策略 | 权限说明 |
|-----|---------|
| `AliyunLogFullAccess` | SLS 完全控制权限 |
| `AliyunLogReadOnlyAccess` | SLS 只读权限 |

### 3.2 自定义策略

**只读策略示例：**

```json
{
    "Version": "1",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "log:Get*",
                "log:List*",
                "log:Query*"
            ],
            "Resource": [
                "acs:log:*:*:project/your-project/*"
            ]
        }
    ]
}
```

**只写策略示例：**

```json
{
    "Version": "1",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "log:PostLogstoreLogs"
            ],
            "Resource": [
                "acs:log:*:*:project/your-project/logstore/your-logstore"
            ]
        }
    ]
}
```

**特定 Logstore 读写策略：**

```json
{
    "Version": "1",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "log:Get*",
                "log:List*",
                "log:Query*",
                "log:PostLogstoreLogs"
            ],
            "Resource": [
                "acs:log:cn-beijing:*:project/your-project/logstore/gaotu-wxzs-client-node-log"
            ]
        }
    ]
}
```

### 3.3 常用 Action

| Action | 说明 |
|-------|-----|
| `log:GetLogs` | 查询日志 |
| `log:GetHistograms` | 获取日志分布 |
| `log:PostLogstoreLogs` | 写入日志 |
| `log:CreateLogStore` | 创建 Logstore |
| `log:DeleteLogStore` | 删除 Logstore |
| `log:UpdateLogStore` | 更新 Logstore |
| `log:CreateIndex` | 创建索引 |
| `log:UpdateIndex` | 更新索引 |

### 3.4 授权给用户

1. 进入 RAM 控制台
2. 选择用户
3. 点击「添加权限」
4. 选择策略
5. 确认授权

## 四、AccessKey 管理

### 4.1 创建 AccessKey

1. 进入 RAM 用户详情
2. 点击「创建 AccessKey」
3. **立即保存** AccessKey ID 和 Secret

> **重要**：AccessKey Secret 只显示一次！

### 4.2 AccessKey 安全规范

| 规范 | 说明 |
|-----|-----|
| 不要硬编码 | 不要在代码中直接写 AccessKey |
| 使用环境变量 | 通过环境变量传递 |
| 定期轮换 | 定期更换 AccessKey |
| 及时禁用 | 不用时立即禁用 |
| 权限最小化 | 只授予必要权限 |

### 4.3 安全存储方式

**方式一：环境变量**

```bash
export ALIYUN_ACCESS_KEY_ID=xxx
export ALIYUN_ACCESS_KEY_SECRET=xxx
```

```javascript
const client = new SlsLogger({
    accessKey: process.env.ALIYUN_ACCESS_KEY_ID,
    accessSecret: process.env.ALIYUN_ACCESS_KEY_SECRET,
    // ...
});
```

**方式二：配置文件（不提交到 Git）**

```javascript
// config/secrets.js（添加到 .gitignore）
module.exports = {
    accessKeyId: 'xxx',
    accessKeySecret: 'xxx'
};
```

**方式三：配置中心**

从 Apollo、Nacos 等配置中心获取。

**方式四：后端服务获取（Galaxy 项目方式）**

```javascript
// 从后端服务获取 AccessKey
const getSlsAk = async () => {
    const response = await fetch(ossAccessKeyUrl, {
        method: 'POST',
        body: JSON.stringify({ access: access("arms-corp") })
    });
    return response.json();  // { accessKey, secretKey }
};
```

### 4.4 AccessKey 轮换

1. 创建新 AccessKey
2. 更新应用配置使用新 Key
3. 验证新 Key 正常工作
4. 禁用旧 AccessKey
5. 删除旧 AccessKey

### 4.5 禁用/删除 AccessKey

1. 进入 RAM 用户详情
2. 在 AccessKey 列表中
3. 点击「禁用」或「删除」

## 五、STS 临时凭证

### 5.1 为什么使用 STS

- 临时授权，自动过期
- 权限精细控制
- 避免长期凭证泄露风险

### 5.2 使用流程

```
1. 后端服务使用长期 AccessKey 调用 STS
2. STS 返回临时凭证（AccessKey + Token）
3. 前端/客户端使用临时凭证访问 SLS
4. 临时凭证过期后重新获取
```

### 5.3 代码示例

```javascript
const STS = require('@alicloud/sts-sdk');

// 后端服务
async function getSTSToken() {
    const sts = new STS({
        accessKeyId: 'xxx',
        accessKeySecret: 'xxx'
    });

    const result = await sts.assumeRole(
        'acs:ram::123456789:role/sls-write-role',  // 角色 ARN
        'session-name',
        {
            // 可选：进一步限制权限
            Statement: [{
                Effect: 'Allow',
                Action: ['log:PostLogstoreLogs'],
                Resource: ['acs:log:*:*:project/your-project/*']
            }]
        },
        3600  // 有效期（秒）
    );

    return result.Credentials;
    // { AccessKeyId, AccessKeySecret, SecurityToken, Expiration }
}

// 客户端
async function writeLogWithSTS(credentials, message) {
    const client = new SlsLogger({
        accessKey: credentials.AccessKeyId,
        accessSecret: credentials.AccessKeySecret,
        securityToken: credentials.SecurityToken,
        // ...
    });

    await client.info(message);
}
```

## 六、日志脱敏

### 6.1 敏感数据类型

| 类型 | 示例 |
|-----|-----|
| 个人信息 | 手机号、身份证、邮箱 |
| 认证信息 | 密码、Token、AccessKey |
| 业务数据 | 用户 ID、订单号 |

### 6.2 脱敏方式

**方式一：写入前脱敏**

```javascript
function maskSensitive(message) {
    // 手机号脱敏
    message = message.replace(/1[3-9]\d{9}/g, '138****8888');
    // 邮箱脱敏
    message = message.replace(/\w+@\w+\.\w+/g, '***@***.com');
    return message;
}

slsClient.info(maskSensitive(logMessage));
```

**方式二：查询时脱敏**

```sql
* | SELECT
        REGEXP_REPLACE(message, '1[3-9]\d{9}', '138****8888') AS masked_message
```

### 6.3 敏感字段不建索引

对于敏感字段，不建立索引可以：
- 减少存储
- 避免被搜索

## 八、网络访问控制

### 8.1 访问方式

| 方式 | 端点 | 适用场景 |
|-----|-----|---------|
| 公网 | `{region}.log.aliyuncs.com` | 外部访问 |
| 内网 | `{region}-intranet.log.aliyuncs.com` | VPC 内访问 |
| 跨域 | `{region}.log.aliyuncs.com` + 配置 | 浏览器访问 |

### 8.2 推荐配置

- **生产环境**：使用内网端点
- **开发环境**：可使用公网端点
- **客户端**：使用公网端点 + STS

### 8.3 Galaxy 项目配置

```javascript
// 使用内网端点
const END_POINT = "gaotu-new.cn-beijing-intranet.log.aliyuncs.com";
```

## 十、常见安全问题

### 10.1 AccessKey 泄露

**应急处理：**
1. 立即禁用泄露的 AccessKey
2. 创建新 AccessKey
3. 更新所有使用该 Key 的应用
4. 检查是否有异常操作
5. 分析泄露原因并改进

### 10.2 权限过大

**问题**：用户拥有超出需要的权限

**解决**：
1. 审查当前权限
2. 创建自定义策略
3. 只授予必要权限

### 10.3 未脱敏

**问题**：敏感数据直接写入日志

**解决**：
1. 在写入端添加脱敏逻辑
2. 代码审查检查敏感数据
3. 建立日志规范
