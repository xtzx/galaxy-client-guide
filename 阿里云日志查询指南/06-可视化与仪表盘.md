# 06-可视化与仪表盘

> 将日志数据转化为直观的图表和仪表盘

---

## 一、图表类型

### 1.1 可用图表类型

| 图表类型 | 适用场景 | 示例 |
|---------|---------|-----|
| **折线图** | 趋势分析、时间序列 | 日志量随时间变化 |
| **柱状图** | 对比分析、分类统计 | 各级别日志对比 |
| **饼图** | 占比分析 | 错误类型分布 |
| **表格** | 明细展示 | 错误日志列表 |
| **数字大盘** | KPI 展示 | 总日志数、错误率 |
| **面积图** | 累计趋势 | 累计错误数 |
| **散点图** | 相关性分析 | 请求量与错误数关系 |
| **地图** | 地域分布 | 用户地理分布 |

### 1.2 选择图表的原则

| 数据特点 | 推荐图表 |
|---------|---------|
| 随时间变化的数据 | 折线图、面积图 |
| 分类对比 | 柱状图 |
| 占比/比例 | 饼图 |
| 精确数值 | 表格、数字大盘 |
| 多维度数据 | 堆叠柱状图、堆叠面积图 |

## 二、创建图表

### 2.1 基本步骤

1. 在查询页面执行 SQL 查询
2. 查看结果后，点击「图表」标签
3. 选择图表类型
4. 配置 X 轴、Y 轴等
5. 调整图表样式
6. 保存或添加到仪表盘

### 2.2 折线图示例

**查询：按小时统计日志量**

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    ORDER BY hour
```

**图表配置：**
- 图表类型：折线图
- X 轴：`hour`
- Y 轴：`cnt`
- 标题：日志量趋势

### 2.3 柱状图示例

**查询：按级别统计**

```sql
* | SELECT level, COUNT(*) AS cnt
    GROUP BY level
    ORDER BY cnt DESC
```

**图表配置：**
- 图表类型：柱状图
- X 轴：`level`
- Y 轴：`cnt`
- 标题：日志级别分布

### 2.4 饼图示例

**查询：错误类型占比**

```sql
level:error | SELECT
        SUBSTR(message, 1, 50) AS error_type,
        COUNT(*) AS cnt
    GROUP BY SUBSTR(message, 1, 50)
    ORDER BY cnt DESC
    LIMIT 10
```

**图表配置：**
- 图表类型：饼图
- 分类字段：`error_type`
- 数值字段：`cnt`
- 标题：错误类型分布

### 2.5 多系列折线图

**查询：分级别的日志趋势**

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
        SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS errors,
        SUM(CASE WHEN level = 'warn' THEN 1 ELSE 0 END) AS warns,
        SUM(CASE WHEN level = 'info' THEN 1 ELSE 0 END) AS infos
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    ORDER BY hour
```

**图表配置：**
- 图表类型：折线图
- X 轴：`hour`
- Y 轴：`errors`、`warns`、`infos`（多系列）
- 标题：各级别日志趋势

### 2.6 数字大盘示例

**查询：关键指标**

```sql
* | SELECT
        COUNT(*) AS total,
        SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS errors,
        ROUND(SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS error_rate
```

**图表配置：**
- 图表类型：数字大盘
- 显示字段：`total`、`errors`、`error_rate`
- 标题：实时指标

## 四、创建仪表盘

### 4.1 创建步骤

1. 进入 SLS 控制台
2. 选择 Project
3. 点击「仪表盘」→「创建仪表盘」
4. 输入仪表盘名称
5. 点击「确定」

### 4.2 添加图表

1. 打开仪表盘
2. 点击「编辑」
3. 点击「添加」→「添加图表」
4. 选择图表类型
5. 输入查询语句
6. 配置图表
7. 点击「确定」

### 4.3 从查询添加到仪表盘

1. 在查询页面执行查询
2. 配置好图表
3. 点击「添加到仪表盘」
4. 选择目标仪表盘
5. 输入图表名称
6. 点击「确定」

## 五、仪表盘变量

### 5.1 变量的作用

变量可以让仪表盘更加灵活，用户可以通过下拉框选择不同的值来过滤数据。

### 5.2 创建变量

1. 打开仪表盘
2. 点击「编辑」
3. 点击「变量」→「添加变量」
4. 配置变量

### 5.3 变量类型

| 类型 | 说明 | 示例 |
|-----|-----|-----|
| 文本 | 用户输入文本 | 用户名 |
| 枚举 | 预定义选项列表 | error/warn/info |
| 查询 | 从查询结果获取选项 | 动态用户列表 |
| 时间 | 时间选择器 | 开始/结束时间 |

### 5.4 在查询中使用变量

```sql
-- 使用变量 ${username}
${username} | SELECT * ORDER BY __time__ DESC LIMIT 100

-- 在 SQL 中使用
* | SELECT * WHERE message LIKE '%${keyword}%'

-- 级别变量
level:${level} | SELECT COUNT(*) AS cnt
```

### 5.5 变量配置示例

**用户名变量：**
- 名称：`username`
- 类型：文本
- 默认值：`*`

**级别变量：**
- 名称：`level`
- 类型：枚举
- 选项：`*`, `error`, `warn`, `info`
- 默认值：`*`

**动态用户列表变量：**
- 名称：`user`
- 类型：查询
- 查询语句：
```sql
* | SELECT DISTINCT REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1) AS user LIMIT 100
```

## 七、仪表盘功能

### 7.1 时间范围

- 全局时间范围：影响所有图表
- 图表单独时间：可覆盖全局设置

### 7.2 自动刷新

1. 点击时间选择器旁的刷新按钮
2. 选择刷新间隔：15秒、30秒、1分钟、5分钟等
3. 开启自动刷新

### 7.3 全屏展示

点击「全屏」按钮，适合大屏展示。

### 7.4 导出

- 导出图片：截图保存
- 导出 PDF：生成报告
- 导出数据：CSV/Excel

### 7.5 分享

1. 点击「分享」按钮
2. 选择分享方式：
   - 分享链接（需登录）
   - 公开分享（无需登录）
3. 设置有效期

### 7.6 订阅

1. 点击「订阅」按钮
2. 设置接收邮箱
3. 设置发送频率：每天、每周等
4. 设置发送时间

## 九、最佳实践

### 9.1 命名规范

- 仪表盘：`[项目]-[用途]`，如 `Galaxy-运维监控`
- 图表：`[指标]-[图表类型]`，如 `错误率-趋势图`

### 9.2 性能优化

- 避免单个仪表盘图表过多（建议 < 15 个）
- 使用合适的时间范围
- 避免复杂的 SQL 查询
- 使用近似函数（如 `APPROX_DISTINCT`）

### 9.3 可读性

- 添加清晰的标题和描述
- 使用一致的颜色主题
- 合理的布局和分组
- 添加单位说明
