# 02-基础查询语法

> 掌握日常 80% 的日志查询需求

---

## 一、搜索与分析的区别

SLS 查询语句由两部分组成，用 `|` 分隔：

```
搜索条件 | SQL 分析语句
```

| 部分 | 作用 | 是否必须 |
|-----|-----|---------|
| 搜索条件 | 筛选日志（类似 grep） | 必须 |
| SQL 分析语句 | 对结果进行统计分析 | 可选 |

### 示例

```sql
-- 只有搜索条件
error

-- 搜索 + 分析
error | select count(*) as cnt

-- 搜索所有，只做分析
* | select level, count(*) group by level
```

## 二、全文搜索

### 2.1 单词搜索

搜索包含某个词的日志：

```
error
```

```
timeout
```

```
lixiaolu02
```

### 2.2 搜索原理

SLS 会对日志内容进行**分词**，搜索时匹配分词结果。

默认分词符包括：`, '";=()[]{}?@&<>/:\n\t\r`

例如日志内容 `[error] connection timeout` 会被分词为：
- `error`
- `connection`
- `timeout`

## 三、多关键词搜索

### 3.1 AND（与）

两个条件都必须满足：

```
error and timeout
```

```
lixiaolu02 and handleSendFriendToFront
```

等价写法：

```
error AND timeout
error && timeout
```

### 3.2 OR（或）

满足任一条件即可：

```
error or warn
```

```
lixiaolu02 or zhangsan
```

等价写法：

```
error OR warn
error || warn
```

### 3.3 NOT（非）

排除某个关键词：

```
error not debug
```

```
lixiaolu02 not test
```

等价写法：

```
error NOT debug
error !debug
```

### 3.4 括号优先级

使用括号控制运算优先级：

```
(error or warn) and mqtt
```

```
lixiaolu02 and (login or logout)
```

```
(error or exception) and (mqtt or websocket) not debug
```

## 四、字段搜索

### 4.1 语法格式

```
字段名:值
```

### 4.2 精确匹配

```
level:error
```

```
level:warn
```

### 4.3 字段模糊匹配

使用通配符 `*`：

```
message:*timeout*
```

```
message:mqtt*
```

```
message:*error
```

### 4.4 存在性检查

检查字段是否存在：

```
errorKey:*
```

```
level:*
```

### 4.5 字段不等于

```
not level:debug
```

```
level:* not level:info
```

### 4.6 常用字段

| 字段 | 说明 | 示例 |
|-----|-----|-----|
| `message` | 日志内容 | `message:*error*` |
| `level` | 日志级别 | `level:error` |
| `__time__` | 时间戳 | `__time__ > 1706000000` |
| `__source__` | 来源 IP | `__source__:10.0.0.1` |

## 五、短语搜索

### 5.1 精确短语

搜索完整的短语（保持词序）：

```
"connection refused"
```

```
"task timeout"
```

```
"[handleSendFriendToFront]"
```

### 5.2 短语与关键词的区别

| 搜索方式 | 匹配内容 |
|---------|---------|
| `connection refused` | 包含 connection 且包含 refused（不限顺序） |
| `"connection refused"` | 包含完整短语 "connection refused" |

### 5.3 短语中的特殊字符

包含特殊字符时使用引号：

```
"[error]"
```

```
"群总数：0"
```

```
"号数量: 1"
```

## 六、通配符搜索

### 6.1 前缀匹配

```
mqtt*
```

匹配：`mqtt`、`mqttClient`、`mqttHelper` 等

### 6.2 后缀匹配

```
*error
```

匹配：`error`、`codeError`、`parseError` 等

### 6.3 中间匹配

```
*time*
```

匹配：`timeout`、`timestamp`、`datetime` 等

### 6.4 单字符匹配

使用 `?` 匹配单个字符：

```
wxid_?????
```

### 6.5 注意事项

- 通配符搜索性能较低，尽量缩小时间范围
- 不支持以通配符开头的搜索（如 `*abc` 较慢）
- 建议配合其他条件使用

## 七、数值搜索

### 7.1 比较运算

```
status > 400
```

```
status >= 500
```

```
status < 300
```

```
status <= 200
```

```
status = 200
```

### 7.2 范围搜索

```
status in [400 500]
```

包含 400 和 500 之间的所有值（包含边界）

### 7.3 时间戳搜索

```
__time__ > 1706000000
```

```
__time__ >= 1706000000 and __time__ < 1706100000
```

## 八、中文搜索

### 8.1 中文分词

SLS 默认对中文进行分词，搜索时需注意：

```
群总数
```

可能被分词为 `群`、`总数`，搜索 `群总数` 可能匹配不到。

### 8.2 解决方案

**方案一：使用短语搜索**

```
"群总数"
```

**方案二：使用通配符**

```
*群总数*
```

**方案三：在 SQL 中使用 LIKE**

```
* | select * where message like '%群总数%'
```

### 8.3 常见中文关键词搜索

```
"群总数：0"
```

```
"好友总数"
```

```
"无数据需要发送到前端"
```

```
"登录成功"
```

## 九、转义特殊字符

### 9.1 需要转义的字符

以下字符在搜索时有特殊含义，需要转义：

```
" ' ( ) [ ] { } : | \ / * ? > < = ! @ #
```

### 9.2 转义方法

使用反斜杠 `\` 转义：

```
\[error\]
```

或使用引号包裹：

```
"[error]"
```

### 9.3 常见转义示例

| 要搜索的内容 | 正确写法 |
|------------|---------|
| `[error]` | `"[error]"` 或 `\[error\]` |
| `key:value` | `"key:value"` |
| `a/b/c` | `"a/b/c"` |
| `C:\path` | `"C:\\path"` |

## 十、组合查询示例

### 10.1 用户 + 错误

查询某用户的错误日志：

```
lixiaolu02 and (error or Error or ERROR)
```

### 10.2 用户 + 功能

查询某用户的特定功能日志：

```
lixiaolu02 and handleSendFriendToFront
```

### 10.3 用户 + 时间段 + 错误

```
lixiaolu02 and error and __time__ > 1706000000
```

### 10.4 排除调试日志

```
error not debug not test
```

### 10.5 多功能模块

```
lixiaolu02 and (mqtt or websocket or ipc)
```

### 10.6 复杂条件

```
(lixiaolu02 or zhangsan) and (error or warn) and (mqtt or websocket) not debug
```

## 十一、搜索优化技巧

### 11.1 缩小时间范围

时间范围越小，查询越快。

### 11.2 使用精确关键词

```
-- 较慢
*error*

-- 较快
error
```

### 11.3 使用字段搜索

```
-- 较慢（全文搜索）
error

-- 较快（字段搜索）
level:error
```

### 11.4 先粗后细

1. 先用宽泛条件查看大致范围
2. 再逐步添加条件缩小范围

### 11.5 利用日志格式

Galaxy-Client 日志格式固定，可以利用：

```
-- 查询某用户
"[lixiaolu02]"

-- 查询某版本
"[5.4.2-release01]"

-- 查询某功能
"[handlerGetUserList]"
```

### 12.1 查询某用户所有日志

lixiaolu02 and level:error

lixiaolu02 and (error or Error)

```
lixiaolu02 not test not debug
```

### 12.5 查询登录相关

```
lixiaolu02 and (login or 登录)
```

### 12.6 查询 MQTT 相关

```
lixiaolu02 and mqtt
```

### 12.7 查询数据同步

```
lixiaolu02 and (handleSendFriendToFront or friendList or chatroomList)
```
