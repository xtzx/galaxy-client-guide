# 05-聚合统计分析

> 从日志中提取数据洞察和业务指标

---

## 一、基础统计

### 1.1 日志总量

```sql
* | SELECT COUNT(*) AS total
```

### 1.2 按级别统计

```sql
* | SELECT level, COUNT(*) AS cnt
    GROUP BY level
    ORDER BY cnt DESC
```

### 1.3 按来源统计

```sql
* | SELECT __source__, COUNT(*) AS cnt
    GROUP BY __source__
    ORDER BY cnt DESC
    LIMIT 20
```

### 1.4 去重计数

```sql
-- 有多少个不同的来源
* | SELECT COUNT(DISTINCT __source__) AS unique_sources

-- 有多少个不同的用户
* | SELECT COUNT(DISTINCT REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1)) AS unique_users
```

## 二、时间维度分析

### 2.1 按分钟统计

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:%i') AS minute,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:%i')
    ORDER BY minute
```

### 2.2 按小时统计

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    ORDER BY hour
```

### 2.3 按天统计

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d') AS day,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d')
    ORDER BY day
```

### 2.4 按周统计

```sql
* | SELECT
        DATE_TRUNC('week', FROM_UNIXTIME(__time__)) AS week,
        COUNT(*) AS cnt
    GROUP BY DATE_TRUNC('week', FROM_UNIXTIME(__time__))
    ORDER BY week
```

### 2.5 按月统计

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m') AS month,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m')
    ORDER BY month
```

### 2.6 按小时段分布（24小时）

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%H') AS hour_of_day,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%H')
    ORDER BY hour_of_day
```

### 2.7 按星期分布

```sql
* | SELECT
        DAY_OF_WEEK(FROM_UNIXTIME(__time__)) AS day_of_week,
        COUNT(*) AS cnt
    GROUP BY DAY_OF_WEEK(FROM_UNIXTIME(__time__))
    ORDER BY day_of_week
```

## 三、多维度分析

### 3.1 按用户和级别

```sql
* | SELECT
        REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1) AS username,
        level,
        COUNT(*) AS cnt
    GROUP BY REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1), level
    ORDER BY cnt DESC
    LIMIT 50
```

### 3.2 按时间和级别

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
        level,
        COUNT(*) AS cnt
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00'), level
    ORDER BY hour, level
```

### 3.3 按来源和级别

```sql
* | SELECT
        __source__,
        level,
        COUNT(*) AS cnt
    GROUP BY __source__, level
    ORDER BY __source__, cnt DESC
```

### 3.4 透视表（Pivot）

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
        SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS errors,
        SUM(CASE WHEN level = 'warn' THEN 1 ELSE 0 END) AS warns,
        SUM(CASE WHEN level = 'info' THEN 1 ELSE 0 END) AS infos,
        COUNT(*) AS total
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    ORDER BY hour
```

## 四、错误分析

### 4.1 错误率计算

```sql
* | SELECT
        COUNT(*) AS total,
        SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS error_count,
        ROUND(SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS error_rate
```

### 4.2 错误率趋势

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
        COUNT(*) AS total,
        SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS errors,
        ROUND(SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS error_rate
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    ORDER BY hour
```

### 4.3 错误类型分布

```sql
level:error | SELECT
        SUBSTR(message, 1, 100) AS error_type,
        COUNT(*) AS cnt
    GROUP BY SUBSTR(message, 1, 100)
    ORDER BY cnt DESC
    LIMIT 20
```

### 4.4 错误来源分析

```sql
level:error | SELECT
        __source__,
        COUNT(*) AS error_count
    GROUP BY __source__
    ORDER BY error_count DESC
    LIMIT 20
```

### 4.5 成功率计算

```sql
* | SELECT
        COUNT(*) AS total,
        SUM(CASE WHEN level != 'error' THEN 1 ELSE 0 END) AS success_count,
        ROUND(SUM(CASE WHEN level != 'error' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) AS success_rate
```

## 六、环比分析

### 6.1 与前一小时对比

```sql
* | SELECT
        hour,
        cnt,
        LAG(cnt, 1) OVER (ORDER BY hour) AS prev_hour_cnt,
        cnt - LAG(cnt, 1) OVER (ORDER BY hour) AS diff,
        CASE
            WHEN LAG(cnt, 1) OVER (ORDER BY hour) = 0 THEN 0
            ELSE ROUND((cnt - LAG(cnt, 1) OVER (ORDER BY hour)) * 100.0 / LAG(cnt, 1) OVER (ORDER BY hour), 2)
        END AS change_rate_pct
    FROM (
        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
            COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    )
    ORDER BY hour
```

### 6.2 与前一天对比

```sql
* | SELECT
        day,
        cnt,
        LAG(cnt, 1) OVER (ORDER BY day) AS prev_day_cnt,
        cnt - LAG(cnt, 1) OVER (ORDER BY day) AS diff,
        CASE
            WHEN LAG(cnt, 1) OVER (ORDER BY day) = 0 THEN 0
            ELSE ROUND((cnt - LAG(cnt, 1) OVER (ORDER BY day)) * 100.0 / LAG(cnt, 1) OVER (ORDER BY day), 2)
        END AS change_rate_pct
    FROM (
        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d') AS day,
            COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d')
    )
    ORDER BY day
```

## 七、同比分析

### 7.1 与昨日同时段对比

```sql
* | SELECT
        COALESCE(t.hour_of_day, y.hour_of_day) AS hour,
        t.cnt AS today_cnt,
        y.cnt AS yesterday_cnt,
        t.cnt - y.cnt AS diff
    FROM (
        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%H') AS hour_of_day,
            COUNT(*) AS cnt
        FROM log
        WHERE __time__ >= UNIX_TIMESTAMP(CURRENT_DATE)
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%H')
    ) t
    FULL OUTER JOIN (
        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%H') AS hour_of_day,
            COUNT(*) AS cnt
        FROM log
        WHERE __time__ >= UNIX_TIMESTAMP(CURRENT_DATE) - 86400
          AND __time__ < UNIX_TIMESTAMP(CURRENT_DATE)
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%H')
    ) y
    ON t.hour_of_day = y.hour_of_day
    ORDER BY hour
```

### 7.2 与上周同日对比

```sql
* | SELECT
        DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d') AS day,
        COUNT(*) AS this_week,
        0 AS last_week  -- 需要单独查询上周数据并 JOIN
    WHERE __time__ >= UNIX_TIMESTAMP(CURRENT_DATE) - 86400 * 7
    GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d')
    ORDER BY day
```

## 九、漏斗分析

### 9.1 简单漏斗

```sql
* | SELECT
        SUM(CASE WHEN message LIKE '%login%' THEN 1 ELSE 0 END) AS step1_login,
        SUM(CASE WHEN message LIKE '%getuserlist%' THEN 1 ELSE 0 END) AS step2_getlist,
        SUM(CASE WHEN message LIKE '%sendmsg%' THEN 1 ELSE 0 END) AS step3_sendmsg
```

### 9.2 漏斗转化率

```sql
* | SELECT
        step1,
        step2,
        step3,
        ROUND(step2 * 100.0 / step1, 2) AS step1_to_step2_rate,
        ROUND(step3 * 100.0 / step2, 2) AS step2_to_step3_rate,
        ROUND(step3 * 100.0 / step1, 2) AS total_conversion_rate
    FROM (
        SELECT
            SUM(CASE WHEN message LIKE '%login%' THEN 1 ELSE 0 END) AS step1,
            SUM(CASE WHEN message LIKE '%getuserlist%' THEN 1 ELSE 0 END) AS step2,
            SUM(CASE WHEN message LIKE '%sendmsg%' THEN 1 ELSE 0 END) AS step3
        FROM log
    )
```

## 十、分布分析

### 10.1 百分比分布

```sql
* | SELECT
        level,
        cnt,
        ROUND(cnt * 100.0 / SUM(cnt) OVER (), 2) AS percentage
    FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
    ORDER BY cnt DESC
```

### 10.2 区间分布

```sql
* | SELECT
        CASE
            WHEN duration < 100 THEN '0-100ms'
            WHEN duration < 500 THEN '100-500ms'
            WHEN duration < 1000 THEN '500-1000ms'
            WHEN duration < 5000 THEN '1-5s'
            ELSE '>5s'
        END AS duration_range,
        COUNT(*) AS cnt
    GROUP BY
        CASE
            WHEN duration < 100 THEN '0-100ms'
            WHEN duration < 500 THEN '100-500ms'
            WHEN duration < 1000 THEN '500-1000ms'
            WHEN duration < 5000 THEN '1-5s'
            ELSE '>5s'
        END
    ORDER BY
        CASE duration_range
            WHEN '0-100ms' THEN 1
            WHEN '100-500ms' THEN 2
            WHEN '500-1000ms' THEN 3
            WHEN '1-5s' THEN 4
            ELSE 5
        END
```

### 10.3 累计分布

```sql
* | SELECT
        level,
        cnt,
        SUM(cnt) OVER (ORDER BY cnt DESC) AS cumulative_cnt,
        ROUND(SUM(cnt) OVER (ORDER BY cnt DESC) * 100.0 / SUM(cnt) OVER (), 2) AS cumulative_pct
    FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
    ORDER BY cnt DESC
```

## 十一、移动统计

### 11.1 移动平均

```sql
* | SELECT
        hour,
        cnt,
        ROUND(AVG(cnt) OVER (ORDER BY hour ROWS BETWEEN 2 PRECEDING AND CURRENT ROW), 2) AS moving_avg_3
    FROM (
        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
            COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    )
    ORDER BY hour
```

### 11.2 移动求和

```sql
* | SELECT
        hour,
        cnt,
        SUM(cnt) OVER (ORDER BY hour ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS rolling_sum_7
    FROM (
        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
            COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
    )
    ORDER BY hour
```

### 11.3 累计求和

```sql
* | SELECT
        hour,
        cnt,

        SELECT
            DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,

        FROM log
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
```

## 十二、异常检测

### 12.1 基于标准差

```sql
* | SELECT
        hour,
        cnt,
        avg_cnt,
        stddev_cnt,
        CASE
            WHEN cnt > avg_cnt + 2 * stddev_cnt THEN 'HIGH'
            WHEN cnt < avg_cnt - 2 * stddev_cnt THEN 'LOW'
            ELSE 'NORMAL'
        END AS status
    FROM (
        SELECT
            hour,
            cnt,
            AVG(cnt) OVER () AS avg_cnt,
            STDDEV(cnt) OVER () AS stddev_cnt
        FROM (
            SELECT
                DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
                COUNT(*) AS cnt
            FROM log
            GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
        )
    )
    ORDER BY hour
```

### 12.2 基于倍数阈值

```sql
* | SELECT
        hour,
        cnt,
        avg_cnt,
        CASE
            WHEN cnt > avg_cnt * 2 THEN 'SPIKE'
            WHEN cnt < avg_cnt * 0.5 THEN 'DROP'
            ELSE 'NORMAL'
        END AS status
    FROM (
        SELECT
            hour,
            cnt,
            AVG(cnt) OVER () AS avg_cnt
        FROM (
            SELECT
                DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00') AS hour,
                COUNT(*) AS cnt
            FROM log
            GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%Y-%m-%d %H:00')
        )
    )
    WHERE status != 'NORMAL'
    ORDER BY hour
```

## 十三、Galaxy 项目常用统计

### 13.1 用户活跃度

```sql
* | SELECT
        REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1) AS username,
        COUNT(*) AS total_logs,
        SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS errors,
        MIN(__time__) AS first_seen,
        MAX(__time__) AS last_seen
    GROUP BY REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1)
    ORDER BY total_logs DESC
    LIMIT 20
```

### 13.2 版本分布

```sql
* | SELECT
        REGEXP_EXTRACT(message, '\[(\d+\.\d+\.\d+-[^\]]+)\]', 1) AS version,
        COUNT(*) AS cnt
    GROUP BY REGEXP_EXTRACT(message, '\[(\d+\.\d+\.\d+-[^\]]+)\]', 1)
    ORDER BY cnt DESC
```

### 13.3 功能使用统计

```sql
* | SELECT
        CASE
            WHEN message LIKE '%mqtt%' THEN 'MQTT'
            WHEN message LIKE '%websocket%' OR message LIKE '%ws%' THEN 'WebSocket'
            WHEN message LIKE '%ipc%' OR message LIKE '%逆向%' THEN 'IPC'
            WHEN message LIKE '%login%' OR message LIKE '%登录%' THEN 'Login'
            WHEN message LIKE '%sendmsg%' OR message LIKE '%发送%' THEN 'SendMsg'
            ELSE 'Other'
        END AS feature,
        COUNT(*) AS cnt
    GROUP BY
        CASE
            WHEN message LIKE '%mqtt%' THEN 'MQTT'
            WHEN message LIKE '%websocket%' OR message LIKE '%ws%' THEN 'WebSocket'
            WHEN message LIKE '%ipc%' OR message LIKE '%逆向%' THEN 'IPC'
            WHEN message LIKE '%login%' OR message LIKE '%登录%' THEN 'Login'
            WHEN message LIKE '%sendmsg%' OR message LIKE '%发送%' THEN 'SendMsg'
            ELSE 'Other'
        END
    ORDER BY cnt DESC
```
