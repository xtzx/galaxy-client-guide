> 正则表达式、JSON 解析、窗口函数等高级用法

## 一、正则表达式

### 1.1 REGEXP_LIKE - 正则匹配

检查字符串是否匹配正则表达式：

```sql
* | SELECT * WHERE REGEXP_LIKE(message, 'wxid_[a-z0-9]+')
```

```sql
* | SELECT * WHERE REGEXP_LIKE(message, 'error|Error|ERROR')
```

```sql
* | SELECT * WHERE REGEXP_LIKE(message, '^\[error\]')
```

### 1.2 REGEXP_EXTRACT - 正则提取

提取匹配的内容：

```sql
-- 提取整个匹配
* | SELECT REGEXP_EXTRACT(message, 'wxid_[a-z0-9]+') AS wxid
```

```sql
-- 提取分组（第1个括号）
* | SELECT REGEXP_EXTRACT(message, 'wxid_([a-z0-9]+)', 1) AS wxid_suffix
```

```sql
-- 提取用户名
* | SELECT REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1) AS username
```

### 1.3 REGEXP_REPLACE - 正则替换

```sql
* | SELECT REGEXP_REPLACE(message, 'wxid_[a-z0-9]+', '***') AS masked
```

```sql
* | SELECT REGEXP_REPLACE(message, '\d{11}', '***') AS no_phone
```

### 1.4 REGEXP_SPLIT - 正则分割

```sql
* | SELECT REGEXP_SPLIT(message, '\s+') AS parts
```

### 1.5 常用正则模式

| 模式 | 说明 | 示例 |
|-----|-----|-----|
| `wxid_[a-z0-9]+` | 微信 ID | `wxid_abc123` |
| `\d{11}` | 手机号 | `13800138000` |
| `\d{4}-\d{2}-\d{2}` | 日期 | `2026-01-27` |
| `\d+\.\d+\.\d+` | 版本号 | `5.4.2` |
| `[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+` | 邮箱 | `test@example.com` |
| `https?://[^\s]+` | URL | `https://example.com` |
| `\[([^\]]+)\]` | 方括号内容 | `[error]` 中的 `error` |

### 1.6 正则表达式语法

| 符号 | 含义 |
|-----|-----|
| `.` | 匹配任意单个字符 |
| `*` | 匹配前一个字符 0 次或多次 |
| `+` | 匹配前一个字符 1 次或多次 |
| `?` | 匹配前一个字符 0 次或 1 次 |
| `^` | 匹配字符串开头 |
| `$` | 匹配字符串结尾 |
| `[abc]` | 匹配 a、b 或 c |
| `[^abc]` | 匹配除 a、b、c 外的字符 |
| `[a-z]` | 匹配 a 到 z |
| `\d` | 匹配数字 |
| `\w` | 匹配字母、数字、下划线 |
| `\s` | 匹配空白字符 |
| `(...)` | 分组 |
| `a|b` | 匹配 a 或 b |

## 二、JSON 解析

### 2.1 JSON_EXTRACT - 提取 JSON 值

```sql
* | SELECT JSON_EXTRACT(json_field, '$.key') AS value
```

```sql
* | SELECT JSON_EXTRACT(json_field, '$.data.array[0]') AS first_item
```

### 2.2 JSON_EXTRACT_SCALAR - 提取标量值

返回字符串类型：

```sql
* | SELECT JSON_EXTRACT_SCALAR(json_field, '$.name') AS name
```

### 2.3 JSON 路径语法

| 语法 | 说明 | 示例 |
|-----|-----|-----|
| `$.key` | 获取键值 | `$.name` |
| `$.a.b` | 嵌套键 | `$.data.user` |
| `$[0]` | 数组索引 | `$[0]` |
| `$.arr[0]` | 数组中的元素 | `$.users[0]` |
| `$.arr[*]` | 数组所有元素 | `$.users[*].name` |

### 2.4 JSON_ARRAY_LENGTH - 数组长度

```sql
* | SELECT JSON_ARRAY_LENGTH(JSON_EXTRACT(json_field, '$.array')) AS arr_len
```

### 2.5 JSON_FORMAT - 格式化输出

```sql
* | SELECT JSON_FORMAT(JSON_PARSE(json_str)) AS formatted
```

### 2.6 解析日志中的 JSON

如果日志内容包含 JSON：

```sql
* | SELECT
        JSON_EXTRACT_SCALAR(
            REGEXP_EXTRACT(message, '\{.*\}'),
            '$.type'
        ) AS msg_type
```

### 2.7 实际示例

Galaxy-Client 日志中有时包含 JSON 数据：

```sql
-- 提取发送到前端的消息类型
* and "发送前端WS消息" | SELECT
    REGEXP_EXTRACT(message, '"type":"([^"]+)"', 1) AS msg_type,
    COUNT(*) AS cnt
    GROUP BY REGEXP_EXTRACT(message, '"type":"([^"]+)"', 1)
```

## 三、子查询

### 3.1 基本子查询

```sql
* | SELECT * FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
    WHERE cnt > 100
```

### 3.2 IN 子查询

```sql
* | SELECT * WHERE __source__ IN (
        SELECT __source__
        FROM log
        WHERE level = 'error'
        GROUP BY __source__
        HAVING COUNT(*) > 10
    )
```

### 3.3 嵌套聚合

```sql
* | SELECT AVG(cnt) AS avg_errors_per_source FROM (
        SELECT __source__, COUNT(*) AS cnt
        FROM log
        WHERE level = 'error'
        GROUP BY __source__
    )
```

## 四、窗口函数

### 4.1 ROW_NUMBER - 行号

```sql
* | SELECT
        message,
        __time__,
        ROW_NUMBER() OVER (ORDER BY __time__ DESC) AS row_num
    LIMIT 100
```

### 4.2 RANK - 排名

```sql
* | SELECT
        __source__,
        error_count,
        RANK() OVER (ORDER BY error_count DESC) AS rank
    FROM (
        SELECT __source__, COUNT(*) AS error_count
        FROM log
        WHERE level = 'error'
        GROUP BY __source__
    )
```

### 4.3 DENSE_RANK - 密集排名

```sql
* | SELECT
        level,
        cnt,
        DENSE_RANK() OVER (ORDER BY cnt DESC) AS dense_rank
    FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
```

### 4.4 LAG - 前一行值

```sql
* | SELECT
        hour,
        cnt,
        LAG(cnt, 1) OVER (ORDER BY hour) AS prev_cnt
    FROM (
        SELECT DATE_TRUNC('hour', __time__) AS hour, COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
```

### 4.5 LEAD - 后一行值

```sql
* | SELECT
        hour,
        cnt,
        LEAD(cnt, 1) OVER (ORDER BY hour) AS next_cnt
    FROM (
        SELECT DATE_TRUNC('hour', __time__) AS hour, COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
```

### 4.6 FIRST_VALUE / LAST_VALUE

```sql
* | SELECT
        level,
        cnt,
        FIRST_VALUE(cnt) OVER (ORDER BY cnt DESC) AS max_cnt
    FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
```

### 4.7 PARTITION BY - 分区

```sql
* | SELECT
        __source__,
        level,
        cnt,
        ROW_NUMBER() OVER (PARTITION BY __source__ ORDER BY cnt DESC) AS rank_in_source
    FROM (
        SELECT __source__, level, COUNT(*) AS cnt
        FROM log
        GROUP BY __source__, level
    )
```

### 4.8 累计求和

```sql
* | SELECT
        hour,
        cnt,
        SUM(cnt) OVER (ORDER BY hour) AS cumulative_cnt
    FROM (
        SELECT DATE_TRUNC('hour', __time__) AS hour, COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
```

## 五、数组操作

### 5.1 ARRAY_AGG - 聚合为数组

```sql
* | SELECT
        __source__,
        ARRAY_AGG(DISTINCT level) AS levels
    GROUP BY __source__
```

### 5.2 CARDINALITY - 数组长度

```sql
* | SELECT CARDINALITY(ARRAY[1, 2, 3]) AS len
```

### 5.3 CONTAINS - 包含判断

```sql
* | SELECT * WHERE CONTAINS(SPLIT(message, ' '), 'error')
```

### 5.4 ARRAY_JOIN - 数组转字符串

```sql
* | SELECT ARRAY_JOIN(ARRAY['a', 'b', 'c'], ',') AS joined
-- 结果: 'a,b,c'
```

### 5.5 TRANSFORM - 数组变换

```sql
* | SELECT TRANSFORM(ARRAY[1, 2, 3], x -> x * 2) AS doubled
-- 结果: [2, 4, 6]
```

### 5.6 FILTER - 数组过滤

```sql
* | SELECT FILTER(ARRAY[1, 2, 3, 4, 5], x -> x > 3) AS filtered
-- 结果: [4, 5]
```

## 六、Map 操作

### 6.1 MAP_AGG - 聚合为 Map

```sql
* | SELECT MAP_AGG(level, cnt) AS level_map FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
```

### 6.2 MAP_KEYS / MAP_VALUES

```sql
* | SELECT MAP_KEYS(MAP_AGG(level, cnt)) AS keys FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
```

### 6.3 ELEMENT_AT - 获取 Map 元素

```sql
* | SELECT ELEMENT_AT(MAP_AGG(level, cnt), 'error') AS error_cnt FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
```

## 七、UNION 操作

### 7.1 UNION - 合并去重

```sql
* | SELECT level, COUNT(*) AS cnt FROM log WHERE level = 'error' GROUP BY level
    UNION
    SELECT level, COUNT(*) AS cnt FROM log WHERE level = 'warn' GROUP BY level
```

### 7.2 UNION ALL - 合并不去重

```sql
* | SELECT 'error' AS type, COUNT(*) AS cnt FROM log WHERE level = 'error'
    UNION ALL
    SELECT 'warn' AS type, COUNT(*) AS cnt FROM log WHERE level = 'warn'
```

## 八、高级分析技巧

### 8.1 环比分析

与上一时间段对比：

```sql
* | SELECT
        hour,
        cnt,
        LAG(cnt, 1) OVER (ORDER BY hour) AS prev_cnt,
        cnt - LAG(cnt, 1) OVER (ORDER BY hour) AS diff,
        ROUND((cnt - LAG(cnt, 1) OVER (ORDER BY hour)) * 100.0 / LAG(cnt, 1) OVER (ORDER BY hour), 2) AS change_rate
    FROM (
        SELECT DATE_TRUNC('hour', __time__) AS hour, COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
    ORDER BY hour
```

### 8.2 同比分析

与昨日同时段对比：

```sql
* | SELECT
        today.hour,
        today.cnt AS today_cnt,
        yesterday.cnt AS yesterday_cnt,
        today.cnt - yesterday.cnt AS diff
    FROM (
        SELECT DATE_FORMAT(FROM_UNIXTIME(__time__), '%H') AS hour, COUNT(*) AS cnt
        FROM log
        WHERE __time__ >= UNIX_TIMESTAMP(CURRENT_DATE)
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%H')
    ) today
    LEFT JOIN (
        SELECT DATE_FORMAT(FROM_UNIXTIME(__time__), '%H') AS hour, COUNT(*) AS cnt
        FROM log
        WHERE __time__ >= UNIX_TIMESTAMP(DATE_ADD('day', -1, CURRENT_DATE))
          AND __time__ < UNIX_TIMESTAMP(CURRENT_DATE)
        GROUP BY DATE_FORMAT(FROM_UNIXTIME(__time__), '%H')
    ) yesterday
    ON today.hour = yesterday.hour
    ORDER BY today.hour
```

### 8.3 漏斗分析

```sql
* | SELECT
        SUM(CASE WHEN message LIKE '%login%' THEN 1 ELSE 0 END) AS step1_login,
        SUM(CASE WHEN message LIKE '%getuserlist%' THEN 1 ELSE 0 END) AS step2_getlist,
        SUM(CASE WHEN message LIKE '%sendmsg%' THEN 1 ELSE 0 END) AS step3_send
```

### 8.4 百分比分布

```sql
* | SELECT
        level,
        cnt,
        ROUND(cnt * 100.0 / SUM(cnt) OVER (), 2) AS percentage
    FROM (
        SELECT level, COUNT(*) AS cnt
        FROM log
        GROUP BY level
    )
    ORDER BY cnt DESC
```

### 8.5 移动平均

```sql
* | SELECT
        hour,
        cnt,
        AVG(cnt) OVER (ORDER BY hour ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) AS moving_avg_3
    FROM (
        SELECT DATE_TRUNC('hour', __time__) AS hour, COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
```

## 九、性能优化技巧

### 9.1 避免 SELECT *

```sql
-- 较慢
* | SELECT * LIMIT 100

-- 较快
* | SELECT message, __time__ LIMIT 100
```

### 9.2 先过滤再聚合

```sql
-- 较快：先用搜索条件过滤
level:error | SELECT COUNT(*) AS cnt

-- 较慢：在 SQL 中过滤
* | SELECT COUNT(*) AS cnt WHERE level = 'error'
```

### 9.3 使用近似函数

```sql
-- 精确但较慢
* | SELECT COUNT(DISTINCT __source__) AS cnt

-- 近似但较快
* | SELECT APPROX_DISTINCT(__source__) AS cnt
```

### 9.4 限制结果数量

```sql
* | SELECT * ORDER BY __time__ DESC LIMIT 100
```

### 9.5 减少复杂计算

```sql
-- 避免在 WHERE 中使用函数
* | SELECT * WHERE __time__ > 1706000000  -- 好

* | SELECT * WHERE FROM_UNIXTIME(__time__) > '2026-01-27'  -- 避免
```

## 十、实际应用示例

### 10.1 提取并统计用户

```sql
* | SELECT
        REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1) AS username,
        COUNT(*) AS cnt
    GROUP BY REGEXP_EXTRACT(message, '\[([^\]]+)\]', 1)
    ORDER BY cnt DESC
    LIMIT 20
```

### 10.2 分析错误分布

```sql
level:error | SELECT
        SUBSTR(message, 1, 80) AS error_prefix,
        COUNT(*) AS cnt,
        MIN(__time__) AS first_seen,
        MAX(__time__) AS last_seen
    GROUP BY SUBSTR(message, 1, 80)
    ORDER BY cnt DESC
    LIMIT 20
```

### 10.3 计算错误率趋势

```sql
* | SELECT
        hour,
        total,
        errors,
        ROUND(errors * 100.0 / total, 2) AS error_rate
    FROM (
        SELECT
            DATE_TRUNC('hour', __time__) AS hour,
            COUNT(*) AS total,
            SUM(CASE WHEN level = 'error' THEN 1 ELSE 0 END) AS errors
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
    ORDER BY hour
```

### 10.4 找出异常时间段

```sql
* | SELECT
        hour,
        cnt,
        CASE
            WHEN cnt > AVG(cnt) OVER () * 2 THEN 'HIGH'
            WHEN cnt < AVG(cnt) OVER () * 0.5 THEN 'LOW'
            ELSE 'NORMAL'
        END AS status
    FROM (
        SELECT DATE_TRUNC('hour', __time__) AS hour, COUNT(*) AS cnt
        FROM log
        GROUP BY DATE_TRUNC('hour', __time__)
    )
    ORDER BY hour
```
