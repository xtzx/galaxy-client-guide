# 阿里云 SLS 日志查询指南

> 从零开始掌握阿里云日志服务（SLS），实现高效的日志查询与分析

---

## 什么是 SLS？

**SLS（Simple Log Service）** 是阿里云提供的一站式日志管理服务，具备以下核心能力：

| 能力 | 说明 |
|-----|-----|
| 日志采集 | 支持多种方式采集日志（SDK、API、Agent） |
| 日志存储 | 海量日志存储，支持冷热分层 |
| 日志查询 | 毫秒级查询，支持全文搜索和 SQL 分析 |
| 可视化 | 内置图表和仪表盘 |
| 告警监控 | 基于日志的智能告警 |

## 核心概念

### 概念层级图

```
阿里云账号
└── Project（项目）
    ├── Logstore（日志库）
    │   ├── 索引配置
    │   │   ├── 全文索引（对所有内容建索引）
    │   │   └── 字段索引（对特定字段建索引）
    │   └── 日志数据
    │       ├── Log 1: { __time__, __source__, message, level, ... }
    │       ├── Log 2: { __time__, __source__, message, level, ... }
    │       └── ...
    ├── Logstore 2
    └── ...
```

### 概念说明

| 概念 | 说明 | 类比 |
|-----|-----|-----|
| **Project** | 项目容器，管理多个 Logstore | 数据库实例 |
| **Logstore** | 日志库，存储同类日志 | 数据库表 |
| **索引** | 决定哪些字段可被搜索 | 数据库索引 |
| **分词** | 将日志内容拆分为可搜索的词 | 全文检索 |

### 系统字段

| 字段 | 说明 | 示例 |
|-----|-----|-----|
| `__time__` | 日志时间戳（Unix 秒） | `1706000000` |
| `__source__` | 日志来源 IP | `10.0.0.1` |
| `__topic__` | 日志主题（可选） | `app_log` |
| `__tag__:xxx` | 标签字段 | `__tag__:env=prod` |

## 本项目日志配置

Galaxy-Client 项目使用 SLS 进行日志上报：

| 配置项 | 值 |
|-------|---|
| **Logstore** | `gaotu-wxzs-client-node-log` |
| **Endpoint** | `gaotu-new.cn-beijing-intranet.log.aliyuncs.com` |
| **地域** | `cn-beijing` |
| **SDK** | `ali-sls` (Node.js) |

### 日志格式

每条日志的 `message` 字段格式：

```
[用户名] [GID] [sonVersion] [version] [号数量: X] 实际日志内容
```

示例：

```
[lixiaolu02] [YYDwStXQbutWaV2LC0JYJutQE5gBKtEx_win32_win-10-22H2] [sonVersion1] [5.4.2-release01] [号数量: 1] [初始化] [handlerGetUserList] ...
```

### 日志上报代码

```javascript
// src/init/slsLog.js
this.slsClient?.info(
    `[${casUserInfo?.user}] [${this.gid}] [${clientSonVersion}] [${clientVersion}] [号数量: ${registrys.length}] ${message}`
);
```
