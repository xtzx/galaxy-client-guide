# 13-与本地日志对比

> electron-log 与阿里云 SLS 的对比和配合使用

---

## 一、两套日志系统

Galaxy-Client 项目同时使用两套日志系统：

| 系统 | 用途 |
|-----|-----|
| **electron-log** | 本地文件日志 |
| **阿里云 SLS** | 云端日志服务 |

---

## 二、electron-log 介绍

### 2.1 基本信息

| 项目 | 说明 |
|-----|-----|
| 包名 | `electron-log` |
| 作用 | Electron 应用的本地日志记录 |
| 存储 | 本地文件 |

### 2.2 日志文件位置

**Windows:**

```
%USERPROFILE%\AppData\Roaming\galaxy-client\logs\main.log
```

**macOS:**

```
~/Library/Logs/galaxy-client/main.log
```

### 2.3 日志格式

```
[2026-01-27 14:30:45.123] [info] 日志内容
[2026-01-27 14:30:46.456] [error] errorKey 错误信息
```

### 2.4 日志级别

| 级别 | 说明 |
|-----|-----|
| `error` | 错误 |
| `warn` | 警告 |
| `info` | 信息 |
| `verbose` | 详细 |
| `debug` | 调试 |
| `silly` | 最详细 |

### 2.5 代码使用

```javascript
const log = require('electron-log');

log.info('This is info');
log.warn('This is warning');
log.error('errorKey', 'This is error');
```

## 三、SLS 日志上报机制

### 3.1 上报代码

**文件位置**：`src/init/slsLog.js`

```javascript
const SlsLogger = require("ali-sls");

const slsStore = "gaotu-wxzs-client-node-log";
const END_POINT = "gaotu-new.cn-beijing-intranet.log.aliyuncs.com";

const slsLogUtil = {
    slsClient: null,

    async initSlsLog() {
        const { accessKey, secretKey } = await getSlsAk();
        this.slsClient = new SlsLogger({
            endpoint: END_POINT,
            accessKey,
            accessSecret: secretKey,
            logstore: slsStore,
        });
    },

    async customLog(message) {
        if (!this.slsClient) {
            await this.initSlsLog();
        }

        const casUserInfo = store.getUserInfo();
        const registrys = registryList.getRegistryList();

        // 格式化日志
        this.slsClient?.info(
            `[${casUserInfo?.user}] [${this.gid}] [${clientSonVersion}] [${clientVersion}] [号数量: ${registrys.length}] ${message}`
        );
    },
};
```

### 3.2 统一入口

**文件位置**：`src/init/log.js`

```javascript
const log = require('electron-log');
const slsLogUtil = require('./slsLog.js');

const logUtil = {
    customLog(message, options) {
        // 1. 上报到阿里云 SLS
        slsLogUtil.customLog(message);

        // 2. 写入本地文件
        const { level = 'info', errorKey = '' } = options ?? {};
        if (level === 'error') {
            log[level](errorKey, message);
        } else {
            log[level](message);
        }
    },
};

module.exports = logUtil;
```

### 3.3 上报流程

```
业务代码调用 logUtil.customLog()
         │
         ├──► slsLogUtil.customLog() ──► 阿里云 SLS
         │
         └──► electron-log ──► 本地文件
```

## 四、对比分析

### 4.1 功能对比

| 特性 | electron-log | 阿里云 SLS |
|-----|-------------|-----------|
| 存储位置 | 本地文件 | 云端 |
| 查询方式 | 文本搜索/打开文件 | SQL 分析 |
| 实时性 | 实时写入 | 1-3 秒延迟 |
| 持久性 | 随文件系统 | 云端永久保存 |
| 跨设备 | 仅本地 | 任意设备访问 |
| 成本 | 免费 | 按量付费 |
| 聚合分析 | 不支持 | 支持 |
| 告警 | 不支持 | 支持 |
| 可视化 | 不支持 | 支持 |

### 4.2 内容对比

| 项目 | electron-log | 阿里云 SLS |
|-----|-------------|-----------|
| 日志格式 | `[时间] [级别] 内容` | `[用户] [GID] [版本] [号数量] 内容` |
| 用户信息 | 无 | 有（用户名、GID） |
| 版本信息 | 无 | 有（版本号） |
| 业务信息 | 无 | 有（微信号数量） |
| 时区 | 本地时区 | UTC（需转换） |

### 4.3 日志内容差异

**同一条日志：**

**electron-log 格式：**
```
[2026-01-27 14:30:45.123] [info] [初始化] [handlerGetUserList] 逆向返回好友数：747
```

**SLS 格式：**
```
[lixiaolu02] [YYDwStXQ...] [sonVersion1] [5.4.2-release01] [号数量: 1] [初始化] [handlerGetUserList] 逆向返回好友数：747
```

## 五、使用场景

### 5.1 何时使用本地日志

| 场景 | 说明 |
|-----|-----|
| **开发调试** | 本地开发时查看实时日志 |
| **离线排查** | 用户电脑无网络时 |
| **用户现场** | 需要用户提供日志文件 |
| **详细调试** | 需要 verbose/debug 级别 |
| **敏感信息** | 不适合上传到云端的内容 |

### 5.2 何时使用 SLS

| 场景 | 说明 |
|-----|-----|
| **生产问题排查** | 远程排查用户问题 |
| **多用户分析** | 跨用户的问题分析 |
| **统计分析** | 错误率、使用量统计 |
| **告警监控** | 自动化监控 |
| **趋势分析** | 历史数据分析 |
| **问题定位** | 快速定位问题发生时间和用户 |

### 5.3 典型场景对比

| 场景 | 推荐方案 |
|-----|---------|
| 开发时调试代码 | 本地日志 |
| 用户反馈问题排查 | 先 SLS 定位，再要本地日志 |
| 统计错误率 | SLS |
| 分析用户行为 | SLS |
| 现场调试 | 本地日志 |
| 性能分析 | SLS |

## 六、日志级别对应

### 6.1 级别映射

| 业务意图 | electron-log | SLS |
|---------|-------------|-----|
| 错误 | `log.error()` | `level:error` |
| 警告 | `log.warn()` | `level:warn` |
| 信息 | `log.info()` | `level:info` |
| 调试 | `log.debug()` | 通常不上报 |

### 6.2 当前项目实现

```javascript
// src/init/log.js
const logUtil = {
    customLog(message, options) {
        slsLogUtil.customLog(message);  // 所有日志都上报 SLS

        const { level = 'info' } = options ?? {};
        log[level](message);  // 本地根据级别记录
    },
};
```

## 七、配合使用策略

### 7.1 排查流程

```
1. 用户反馈问题
        │
        ▼
2. 在 SLS 中搜索用户日志
   - 获取用户名
   - 确定时间范围
   - 定位问题类型
        │
        ▼
3. 分析 SLS 日志
   - 错误发生时间
   - 错误前后上下文
   - 是否有其他用户遇到
        │
        ▼
4. 如需更详细信息
   - 请用户提供本地日志文件
   - 查看 verbose/debug 级别日志
        │
        ▼
5. 定位问题根因
```

### 7.2 SLS 初步定位

```sql
-- 1. 搜索用户日志
用户名 | SELECT * ORDER BY __time__ DESC LIMIT 100

-- 2. 查找错误
用户名 and error | SELECT * ORDER BY __time__ DESC LIMIT 50

-- 3. 确定时间范围
用户名 and error | SELECT MIN(__time__) AS first, MAX(__time__) AS last

-- 4. 分析错误类型
用户名 and error | SELECT message, COUNT(*) GROUP BY message
```

### 7.3 本地日志补充

当 SLS 信息不足时，请用户提供本地日志：

**指导用户找到日志文件：**

```
Windows:
1. 按 Win + R
2. 输入 %USERPROFILE%\AppData\Roaming\galaxy-client\logs
3. 找到 main.log 文件

macOS:
1. 打开 Finder
2. 按 Cmd + Shift + G
3. 输入 ~/Library/Logs/galaxy-client/
4. 找到 main.log 文件
```

## 八、最佳实践

### 8.1 日志记录原则

1. **错误日志**：一定要上报 SLS
2. **关键流程**：登录、消息发送等关键操作要记录
3. **调试日志**：只记录本地，不上报 SLS
4. **敏感信息**：脱敏或只记录本地

### 8.2 日志内容建议

```javascript
// 好的日志
logUtil.customLog(`[初始化] [handlerGetUserList] [wxid-${wxid}] 好友数：${count}`, { level: 'info' });

// 不好的日志
logUtil.customLog('done');  // 没有上下文
logUtil.customLog(JSON.stringify(largeObject));  // 内容过大
```

### 8.3 本地日志配置

```javascript
// 配置 electron-log
const log = require('electron-log');

// 日志文件大小限制
log.transports.file.maxSize = 10 * 1024 * 1024;  // 10MB

// 日志保留天数
log.transports.file.archiveLog = (file) => {
    file.path = file.path.replace(/\.log$/, `.${Date.now()}.log`);
};
```

## 九、问题排查示例

### 9.1 用户反馈「好友列表空白」

**第一步：SLS 搜索**

```sql
lixiaolu02 and handleSendFriendToFront | SELECT message, __time__
ORDER BY __time__ DESC LIMIT 10
```

**发现**：`无数据需要发送到前端`

**第二步：追溯上游**

```sql
lixiaolu02 and "群总数" | SELECT message, __time__
ORDER BY __time__ DESC LIMIT 10
```

**发现**：`群总数：0 好友总数：0`

**第三步：定位根因**

```sql
lixiaolu02 and "发送前端WS消息" and userinfolist | SELECT message
ORDER BY __time__ DESC LIMIT 5
```

**发现**：逆向返回的数据格式异常

**第四步（如需）**：请用户提供本地日志查看更详细的数据内容

### 9.2 用户反馈「登录失败」

**SLS 搜索：**

```sql
用户名 and (login or 登录) and (fail or 失败 or error) | SELECT *
ORDER BY __time__ DESC LIMIT 20
```

**查看错误详情后，如需本地日志：**

请用户提供 `main.log` 文件，搜索 `login` 或 `MQTT` 关键词。

## 十、总结

### 10.1 选择指南

| 需求 | 推荐 |
|-----|-----|
| 快速定位生产问题 | SLS |
| 本地开发调试 | electron-log |
| 统计分析 | SLS |
| 详细调试信息 | electron-log |
| 用户现场排查 | electron-log |
| 自动化监控 | SLS |

### 10.2 黄金法则

1. **先 SLS，后本地**：先用 SLS 快速定位，再用本地日志补充
2. **远程用 SLS**：远程排查问题首选 SLS
3. **本地用文件**：开发调试和现场排查用本地日志
4. **结合使用**：两者互补，不是替代关系
