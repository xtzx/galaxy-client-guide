# 15-实战踩坑与高效查询技巧

> 基于真实排查经验总结的 SLS 查询踩坑记录，解决"明明日志里有但搜不到"的问题

---

## 一、三种查询写法的区别

### 1.1 `A AND B` vs `"A" AND "B"` vs `A AND B*`

以实际场景为例：搜索 zhouming 的日志中包含 taskId `1266762108771500107` 的记录。

| 查询方式 | 写法 | 含义 | 效果 |
|---------|------|------|------|
| 不加引号 | `zhouming AND 1266762108771500107` | 分词后做**全文索引匹配** | ⚠️ 依赖分词结果，长数字可能被拆分匹配不上 |
| 加双引号 | `"zhouming" AND "1266762108771500107"` | **精确短语匹配**，token 必须连续完整 | ⚠️ 如果数字被分词拆散则匹配失败 |
| 通配符 | `zhouming AND 1266762108771500107*` | 前缀匹配 | ⚠️ 通配符只能在**词尾**使用 |

### 1.2 SLS 分词原理（核心知识）

SLS 在写入日志时会根据**分词符**将内容切割成 token，建立倒排索引。搜索时匹配的是 token，不是原始字符串。

**默认分词符**：

```
, '";=()[]{}?@&<>/:\n\t\r 以及空格
```

**分词示例**：

```
原始日志：taskId= 1266762108771500107 [mqttTaskMapLock]
分词结果：taskId、1266762108771500107、mqttTaskMapLock
（= 和 [ ] 都是分词符，会被去掉）
```

```
原始日志："status":4,"reason":"MM_ERR_SPAM"
分词结果：status、4、reason、MM_ERR_SPAM
（" : , 都是分词符）
```

### 1.3 什么时候加引号，什么时候不加

| 场景 | 推荐写法 | 原因 |
|------|---------|------|
| 简短英文单词 | `zhouming` 不加引号 | 本身就是一个完整 token |
| 含下划线的 wxid | `wxid_c6w6hg2qwkie22` 不加引号 | 下划线不是默认分词符，整体是一个 token |
| 纯数字 taskId | `1266762108771500107` 不加引号 | 通常是完整 token |
| 带空格的短语 | `"MQTT收到消息"` 加引号 | 确保按短语匹配 |
| 含特殊字符的内容 | 拆分为多个关键词 | 特殊字符会被分词拆散 |

### 1.4 通配符使用规则

```
✅ 可以：zhouming*        （前缀匹配，匹配 zhouming, zhouming02 等）
✅ 可以：message:*error*  （字段级模糊匹配）
❌ 不行：*zhouming         （不支持前导通配符）
❌ 不行：zhou*ming         （不支持中间通配符）
```

## 二、搜不到日志的常见原因和解法

### 2.1 长数字 taskId 搜不到

**问题**：`"1266762108771500107"` 搜不到

**原因**：19位数字在某些上下文中可能被分词器截断或合并（如紧挨着其他字符）

**解决方案**：

```bash
# 方案一：不加引号直接搜（最推荐）
zhouming AND 1266762108771500107

# 方案二：搜部分数字 + 通配符
zhouming AND 126676210877150*

# 方案三：搜周围的固定关键字
zhouming AND msgId AND 126676210877*

# 方案四：用 SQL where 子句（最可靠但最慢）
zhouming | WHERE message LIKE '%1266762108771500107%'
```

### 2.2 中文关键字搜不到

**问题**：搜 `"发文本任务失败"` 搜不到

**原因**：中文全角标点（`：` `！` `，`等）是分词符，会把中文短语拆散

**解决方案**：

```bash
# 方案一：去掉引号，让每个词独立匹配
zhouming AND 发文本任务失败

# 方案二：拆分为更短的关键词
zhouming AND 发文本 AND 任务失败

# 方案三：用英文函数名替代（最推荐）
# 代码里的函数名比中文日志更可靠
zhouming AND sendTextmsgResponse    # 而不是搜 "发文本任务"
zhouming AND taskResponseV2          # 而不是搜 "通知任务结果"
zhouming AND msgResNodeMap          # 而不是搜 "三段式"
```

### 2.3 JSON 字段值搜不到

**问题**：搜 `"status":4` 搜不到

**原因**：`"` `:` 都是分词符，`"status":4` 被分词为 `status` 和 `4`，但 `4` 太短且太常见

**解决方案**：

```bash
# 方案一：同时搜两个词
zhouming AND status AND 4 AND 通知任务结果

# 方案二：用 SQL 精确匹配
zhouming AND 通知任务结果 | WHERE message LIKE '%"status":4%'

# 方案三：搜 reason 字段（更有辨识度）
zhouming AND MM_ERR_SPAM
zhouming AND 未匹配到失败原因
```

### 2.4 搜索结果太多，淹没在噪音中

**问题**：搜 `zhouming` 返回几万条日志，找不到想要的

**解决方案**：

```bash
# 缩小时间范围（最有效）
# 把时间精确到 ±30 分钟

# 多加一个有辨识度的关键词
zhouming AND wxid_c6w6hg2qwkie22      # 加上目标 wxid
zhouming AND MQTT收到消息 AND type.*100  # 加上任务类型

# 排除无关日志
zhouming AND wxid_c6w6hg2qwkie22 NOT 心跳 NOT heartbeat

# 用 SQL 只看特定类型
zhouming AND wxid_c6w6hg2qwkie22 | WHERE message LIKE '%MQTT收到消息%'
```

## 三、高效排查的查询模板

### 3.1 群发消息失败排查模板

```bash
# 第一步：搜目标对象所有日志（下载 JSON 全量分析）
用户名 AND 目标wxid

# 第二步：只看任务下发
用户名 AND 目标wxid AND MQTT收到消息

# 第三步：看三段式结果
用户名 AND 目标wxid AND msgResNodeMap

# 第四步：看失败日志
用户名 AND 目标wxid AND (发文本任务失败 OR 发消息任务失败)

# 第五步：看超时
用户名 AND 目标wxid AND 设置任务超时返回

# 第六步：用 taskId 精确查
用户名 AND 子任务taskId
```

### 3.2 邀请入群排查模板

```bash
# 看邀请任务
用户名 AND 群号@chatroom AND addsmallchatmenber

# 看邀请结果
用户名 AND 群号@chatroom AND AddChatRoomMemberResponse
```

### 3.3 好友列表问题排查模板

```bash
# 看好友列表上报
用户名 AND handleSendFriendToFront

# 看好友数量
用户名 AND 好友列表上传*
```

### 3.4 MQTT 连接问题排查模板

```bash
# 看 MQTT 连接状态
用户名 AND (mqttOnError OR mqttOnClose OR mqttOfflie)

# 看重连
用户名 AND handleReconnectMqtt
```

## 四、终极技巧：下载日志 + 脚本分析

当 SLS 在线搜索无法满足需求时（分词问题、需要交叉分析），最有效的方式是：

### 4.1 下载日志

1. 在 SLS 控制台搜索 `用户名 AND 目标关键字`
2. 设定精确时间范围
3. 点击「下载日志」→ 选 JSON 格式

### 4.2 用命令行快速分析

```bash
# 统计各类日志数量
cat logs.json | python3 -c "
import json, sys
lines = sys.stdin.readlines()
keywords = ['MQTT收到消息', '通知任务结果', '发文本任务失败', '超时']
for kw in keywords:
    count = sum(1 for l in lines if kw in l)
    print(f'{kw}: {count}条')
"

# 提取所有 taskId
cat logs.json | python3 -c "
import json, sys, re
lines = sys.stdin.readlines()
seen = set()
for line in lines:
    for m in re.finditer(r'taskId[=: ]+(\d{15,})', line):
        tid = m.group(1)
        if tid not in seen:
            seen.add(tid)
            print(tid)
"

# 看某个 taskId 的所有日志
cat logs.json | python3 -c "
import json, sys
lines = sys.stdin.readlines()
target = '1266762108771500107'
for i, line in enumerate(lines):
    if target in line:
        data = json.loads(line)
        print(f'行{i+1}: {data[\"message\"][:200]}')
"
```

### 4.3 什么时候用在线搜索 vs 下载分析

| 场景 | 推荐方式 |
|------|---------|
| 确认某个任务是否到达客户端 | 在线搜索 |
| 查看错误码和失败原因 | 在线搜索 |
| 分析一个群发任务中 N 条消息各自的状态 | 下载分析 |
| 对比成功和失败的日志差异 | 下载分析 |
| 还原完整事件时间线 | 下载分析 |
| 排查偶发性问题（需要大量日志对比） | 下载分析 |

## 五、Galaxy 项目常用搜索关键字映射

日志是中文的，但有时中文受分词影响搜不到，可以用对应的**英文函数名/变量名**替代：

| 想搜的内容 | 中文关键字（可能搜不到） | 英文替代（推荐） |
|-----------|----------------------|----------------|
| MQTT 收到任务 | MQTT收到消息 | `mqttClientBase` |
| 任务被接收 | 接收mqtt任务 | `mqttTaskMapLock` |
| 发送给逆向 | 发送消息给逆向 | `ReverseSendService` |
| 三段式结果 | 第三条消息 | `msgResNodeMap` |
| 任务结果上报 | 通知任务结果 | `taskResponseV2` 或 `taskResponse` |
| 文本发送失败 | 发文本任务失败 | `sendTextmsgResponse` |
| 图片发送失败 | - | `sendPictureMsgResponse` |
| 任务超时 | 设置任务超时返回 | `GalaxyTaskStatusTimer` |
| MQTT 上行成功 | MQTT发往云端数据成功 | `MqttClientUtil-send` |
| 邀请入群 | - | `addsmallchatmenber` |
| 邀请结果 | - | `AddChatRoomMemberResponse` |
| 好友列表 | - | `handleSendFriendToFront` |
| 改备注 | - | `remarkWx4Response` |
