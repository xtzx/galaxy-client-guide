# 01-快速入门

> 5 分钟完成第一次日志查询

---

## 一、登录阿里云控制台

### 1.1 访问地址

打开浏览器，访问：

```
https://sls.console.aliyun.com/
```

或从阿里云首页导航：

```
阿里云首页 → 控制台 → 产品与服务 → 日志服务 SLS
```

### 1.2 登录方式

| 方式 | 说明 |
|-----|-----|
| 主账号登录 | 使用阿里云主账号 |
| RAM 用户登录 | 使用子账号（推荐） |
| SSO 单点登录 | 企业账号集成 |

> **提示**：首次使用需要联系管理员获取 RAM 账号和权限

---

## 二、定位日志库

### 2.1 选择 Project

登录后，在左侧项目列表中找到对应的 Project。

Galaxy-Client 项目日志位于：

| 配置项 | 值 |
|-------|---|
| 地域 | `华北2（北京）` |
| Project | （联系管理员获取） |
| Logstore | `gaotu-wxzs-client-node-log` |

### 2.2 进入 Logstore

```
Project 列表 → 点击项目名 → 日志库 → gaotu-wxzs-client-node-log → 查询/分析
```

## 四、第一个查询

### 4.1 查看最新日志

在查询框中输入：

```
*
```

点击「查询/分析」按钮，即可看到最新的日志。

`*` 表示匹配所有日志。

### 4.2 搜索特定关键词

搜索包含 `error` 的日志：

```
error
```

### 4.3 搜索特定用户

搜索用户 `lixiaolu02` 的日志：

```
lixiaolu02
```

### 4.4 组合搜索

搜索用户 `lixiaolu02` 的错误日志：

```
lixiaolu02 and error
```

### 4.5 使用 SQL 分析

统计最近的日志数量：

```
* | select count(*) as total
```

> **注意**：`|` 是分隔符，左边是搜索条件，右边是 SQL 分析语句

## 五、时间范围设置

### 5.1 预设时间范围

点击时间选择器，选择预设选项：

| 选项 | 说明 |
|-----|-----|
| 最近 15 分钟 | 默认选项 |
| 最近 1 小时 | 查看近一小时 |
| 最近 4 小时 | 查看近四小时 |
| 最近 1 天 | 查看近 24 小时 |
| 最近 7 天 | 查看近一周 |

### 5.2 自定义时间范围

1. 点击时间选择器
2. 选择「自定义」
3. 设置开始时间和结束时间
4. 点击「确定」

### 5.3 相对时间

在查询语句中使用相对时间：

```sql
* | select * where __time__ > unix_timestamp() - 3600
```

上述语句查询最近 1 小时（3600 秒）的日志。

## 六、查询结果解读

### 6.1 系统字段

每条日志都包含以下系统字段：

| 字段 | 说明 | 示例 |
|-----|-----|-----|
| `__time__` | 日志时间（Unix 时间戳） | `1706340000` |
| `__source__` | 日志来源 IP | `10.0.0.1` |
| `__topic__` | 日志主题 | （通常为空） |

### 6.2 业务字段

Galaxy-Client 日志的业务字段：

| 字段 | 说明 |
|-----|-----|
| `message` | 日志内容（包含格式化的完整信息） |
| `level` | 日志级别（info/warn/error） |

### 6.3 message 字段格式

```
[用户名] [GID] [sonVersion] [version] [号数量: X] 实际内容
```

示例解析：

```
[lixiaolu02] [YYDwStXQbutWaV2LC0JYJutQE5gBKtEx_win32_win-10-22H2] [sonVersion1] [5.4.2-release01] [号数量: 1] [初始化] [handlerGetUserList] ...
```

| 部分 | 含义 |
|-----|-----|
| `lixiaolu02` | CAS 用户名 |
| `YYDwStXQ...` | 客户端 GID（唯一标识） |
| `sonVersion1` | 子版本号 |
| `5.4.2-release01` | 客户端版本 |
| `号数量: 1` | 当前登录的微信号数量 |
| `[初始化]...` | 实际日志内容 |

## 七、保存和历史

### 7.1 保存查询

经常使用的查询可以保存：

1. 执行查询后，点击「保存为快速查询」
2. 输入名称
3. 点击「确定」

### 7.2 查看历史

点击查询框右侧的「历史」按钮，可以看到最近执行过的查询。

### 7.3 快速查询

点击「快速查询」，可以看到已保存的查询列表。

## 九、常见操作示例

### 9.1 查询某用户今天的日志

```
lixiaolu02
```

时间范围选择「今天」

### 9.2 查询错误日志

```
error or Error or ERROR
```

或使用字段搜索：

```
level:error
```

### 9.3 查询特定功能的日志

```
lixiaolu02 and handleSendFriendToFront
```

### 9.4 查询最近 100 条日志

```
* | select * order by __time__ desc limit 100
```

### 9.5 统计日志级别分布

```
* | select level, count(*) as cnt group by level
```

## 常见问题

### Q1: 查不到日志？

检查以下几点：

1. **时间范围是否正确**：默认只查最近 15 分钟
2. **关键词是否正确**：注意大小写和拼写
3. **是否有权限**：联系管理员确认

### Q2: 查询很慢？

1. 缩小时间范围
2. 添加更多过滤条件
3. 避免使用 `select *`

### Q3: 如何导出结果？

点击结果区右上角的「下载」按钮，选择导出格式（CSV/JSON）。
