## 一、概述

微信助手的群发消息功能负责将云端下发的任务分发给逆向客户端执行。整个下发流程涉及以下核心环节：

1. **MQ消息接收**：通过 MQTT 协议从云端接收任务消息
2. **任务路由与转换**：根据任务类型路由到对应的 task-mqtt 服务进行转换
3. **入站消息调度**：通过 dispatchInBound 进行消息分发
4. **任务状态缓存**：缓存任务状态用于后续回执关联和超时检测
5. **逆向消息发送**：通过 IPC 管道将任务发送给逆向客户端
6. **超时检测**：定时检测任务状态，处理超时任务

该流程同时支持**企业微信**和**个人微信**两种场景，两者在任务类型、回调配置、回执机制上存在差异。

## 三、MQ消息接收

### 3.1 MQTT客户端连接与订阅

**文件**: `src/msg-center/core/mq/mqttClientBase.js`

微信助手通过 MQTT 协议与云端建立长连接，订阅特定的 topic 来接收任务消息。

```javascript
// 连接建立后订阅 topic
client.subscribe(filterTopics, { qos: 1 }, function (err, granted) {
    // 订阅成功后开始接收消息
});
```

关键配置：
- QoS 级别：1（至少一次），确保消息不丢失
- 订阅 topic 由配置决定，通常按机器绑定

### 3.2 消息接收与预处理

```javascript
client.on("message", async function (topic, message) {
    // 1. 将 Buffer 转为 UTF-8 字符串
    let messageUtf8 = message.toString("utf8");
    
    // 2. 大数字替换（防止 JSON.parse 精度丢失）
    messageUtf8 = replaceLargeNumbers(messageUtf8);
    
    // 3. 解析 JSON
    const dataInfo = JSON.parse(messageUtf8);
    
    // 4. 构建服务端任务对象
    const serverTaskBO = { ...ServerTaskBO, ...msgObjTemp };
    
    // 5. 执行任务
    execute(wxId, serverTaskBO);
});
```

**大数字替换说明**：
- JavaScript 的 `JSON.parse` 对超过 `Number.MAX_SAFE_INTEGER` 的整数会丢失精度
- `replaceLargeNumbers` 将大数字转为字符串处理
- 这对 taskId 等长整数字段尤为重要

### 3.3 消息过期检查

```javascript
// 默认过期时间：3小时
const DEFAULT_EXPIRE_TIME = 3 * 60 * 60 * 1000;

// 检查消息创建时间是否在过期时间内
if (dataInfo.createTime && (Date.now() - dataInfo.createTime > DEFAULT_EXPIRE_TIME)) {
    logUtil.customLog(`[MQTT] 消息过期, taskId=${dataInfo.id}`);
    return; // 丢弃过期消息
}
```

**设计目的**：
- 防止微信助手重启后处理积压的过期任务
- 避免执行已被云端取消或重发的任务

### 3.4 任务去重机制

```javascript
// 通过 taskId 去重，防止同一任务被重复处理
if (taskIdSet.has(dataInfo.id)) {
    logUtil.customLog(`[MQTT] 重复任务, taskId=${dataInfo.id}`);
    return;
}
taskIdSet.add(dataInfo.id);
```

|-----------|------|------|

## 四、任务路由与转换

### 4.1 企业微信任务路由（task-mqtt）

**文件**: `src/msg-center/core/mq/mqttClientBase.js` 中的 `execute` → `runTask` 方法

```javascript
function runTask(wxId, serverTaskBO, registry) {
    // 根据 registry.workWx 判断企微还是微信
    const convertServiceList = registry.workWx 
        ? WorkWxConvertServiceList   // 企微服务列表
        : WxConvertServiceList;      // 微信服务列表
    
    // 遍历服务列表，找到匹配的服务执行
    for (const service of convertServiceList) {
        if (service.filter(serverTaskBO)) {
            service.operate(serverTaskBO, wxId);
            break;
        }
    }
}
```

### 4.2 个人微信任务路由

个人微信的任务路由与企微类似，但使用不同的 ConvertServiceList，对应不同的消息类型和处理逻辑。

        if (serverTask.type === PrismRecordType.CHAT_MSG) {

## 五、入站消息调度

### 5.1 cloudFlowInBound - 来源标记

**文件**: `src/msg-center/dispatch-center/dispatch/cloudFlowInBound.js`

```javascript
function cloudFlowInBound(channelId, wxId, message) {
    logUtil.customLog(`[CloudFlowHandler]:inBound, wxId=${wxId}, message=${message}`);
    
    const jsonObject = message ? JSON.parse(message) : {};
    
    // 添加来源标记 flowSource = 2 (CLOUD)
    jsonObject[GalaxyConstant.FLOW_SOURCE_KEY] = FlowSourceEnum.CLOUND;
    
    // 转发到入站调度器
    dispatchInBound(channelId, wxId, JSON.stringify(jsonObject));
}
```

**FlowSourceEnum 说明**：
- `FRONT` (1): 来自前端
- `CLOUND` (2): 来自云端
- `OWNER` (3): 来自机器人自身

### 5.2 dispatchInBound - 入站调度主函数

**文件**: `src/msg-center/dispatch-center/dispatchInBound.js`

```javascript
function dispatchInBound(channelId, wxId, message) {
    // 1. 记录入站日志
    logUtil.customLog(`[wxid-${wxId}] [${channelId}] message: ${message} [dispatchInBound] 开始执行`);
    
    // 2. 获取注册信息
    let registry = null;
    if (wxId) {
        registry = RegistryConfig.getRegistryByKey(wxId, 'wxid');
    } else {
        registry = RegistryConfig.getRegistryByKey(channelId, 'id');
    }
    
    // 3. 校验注册信息
    if (!registry) {
        logUtil.customLog(`[wxid-${wxId}] [${channelId}] [dispatchInBound]:尚未和逆向建立注册信息`);
        return;
    }
    
    // 4. 解析消息类型并转换
    const jsonObject = JSON.parse(message);
    const type = typeConvert(jsonObject.type || jsonObject.body.type);
    
    // 5. 处理入站任务
    processInBoundTask({channelId, wxId, message, jsonObject, type, workWxFlag: registry.workWx});
}
```

### 5.3 注册信息校验

注册信息存储在 `RegistryConfig` 中，包含：

```javascript
{
    wxid: "1688857807622564",       // 微信ID
    id: "channel_xxx",             // 通道ID
    workWx: true,                  // 是否企业微信
    pipeLineWrapper: {
        pipeCode: "1379174031136", // IPC管道编码
        wxid: "1688857807622564",
        workWx: true,
        // ...
    }
}
```

如果注册信息不存在，说明逆向客户端尚未连接（IPC未建立），任务无法下发。

### 5.4 消息类型转换

```javascript
function typeConvert(type) {
    let typeTemp = type;
    // 企业微信发送消息的各种类型统一为前缀
    if (type.startsWith(GalaxyTaskType.WORK_WX_SEND_PREFIX)) {
        typeTemp = GalaxyTaskType.WORK_WX_SEND_PREFIX; // "sendmsg_"
    }
    return typeTemp;
}
```

**转换示例**：
- `sendmsg_sendtext` → `sendmsg_`
- `sendmsg_sendimage` → `sendmsg_`
- `sendmsg_sendvideo` → `sendmsg_`
- `sendmessage` → `sendmessage` (不转换)

## 六、任务状态缓存

### 6.1 processInBoundTask - 任务处理

**文件**: `src/msg-center/dispatch-center/dispatchInBound.js`

这是下发流程中最关键的步骤之一，负责：
1. 判断是否需要任务状态管理
2. 缓存任务状态（用于后续回执关联和超时检测）
3. 发送消息给逆向

```javascript
function processInBoundTask(params) {
    const {channelId, wxId, message, jsonObject, type, workWx, workWxFlag} = params;
    
    // 判断是否需要任务状态管理
    if (type && (TaskCallBack.taskCallbackMap[type] || WkTaskCallBack.taskCallbackMap[type])) {
        
        // 获取第一阶段回调类型
        let firstCallbackType = null;
        if (workWx || workWxFlag) {
            firstCallbackType = WkTaskCallBack.taskCallbackMap[type].firstCallbackType;
        } else {
            firstCallbackType = TaskCallBack.taskCallbackMap[type].firstCallbackType;
        }
        
        // 构建锁的键
        let key = `${wxId}:${firstCallbackType}`;
        const galaxyTaskLock = `galaxyTaskLock-${key}`;
        
        // 获取锁并执行
        lock.acquire(galaxyTaskLock, (done) => {
            try {
                // 构建任务状态对象
                const galaxyTaskStatus = {
                    ...GalaxyTaskStatus,
                    firstMessageStatus: GalaxyTaskStatusConstant.NOT_RECEIVE_STATUS,   // 0
                    secondMessageStatus: GalaxyTaskStatusConstant.NOT_RECEIVE_STATUS,  // 0
                    thirdMessageStatus: GalaxyTaskStatusConstant.NOT_RECEIVE_STATUS,   // 0
                    username: wxId,
                    taskId: jsonObject.taskId,
                    createTime: new Date().getTime()
                };
                
                // 缓存任务状态
                if (jsonObject.taskId) {
                    GalaxyTaskCache.GALAXY_TASK_STATUS_MAP[jsonObject.taskId] = galaxyTaskStatus;
                    GalaxyTaskCache.setTaskMsgTypeMap(jsonObject);
                }
                
                // 发送消息给逆向
                inBoundAct(channelId, wxId, message);
                done();
            } catch(error) {
                done();
            }
        });
    } else {
        // 非发送消息类任务，直接转发
        inBoundAct(channelId, wxId, message);
    }
}
```

### 7.2 发送消息流程

```javascript
sendMessage(wxId, channelId, message) {
    // 1. 参数校验
    if ((!wxId && !channelId) || !message) {
        logUtil.customLog(`[发送给逆向时参数错误] ...`);
        return;
    }
    
    // 2. 获取注册信息
    let registry = wxId 
        ? RegistryConfig.getRegistryByKey(wxId, "wxid")
        : RegistryConfig.getRegistryByKey(channelId, "id");
    
    if (!registry) {
        logUtil.customLog(`[wxid-${wxId}] IPC尚未连接成功`);
        return;
    }
    
    // 3. 重复任务检查
    let { taskId, type } = JSON.parse(message) || {};
    let lockKey = `${registry.wxid}-${taskId}-${type}`;
    if (taskId && this.taskLock.has(lockKey)) {
        logUtil.customLog(`taskId已经存在 已经被锁 ${registry.wxid} ${taskId} ${type}`);
        return;
    }
    
    // 4. 获取管道编码并发送
    const pipeCode = registry.pipeLineWrapper.pipeCode;
    this.pipeLineSend(message, wxId, pipeCode);
    
    // 5. 设置任务锁（5秒过期）
    if (taskId) {
        this.taskLock.set(lockKey, true);
        setTimeout(() => {
            this.taskLock.delete(lockKey);
        }, 5000);
    }
}
```

### 7.3 重复任务过滤

**锁机制**：
- 锁的key格式：`${wxid}-${taskId}-${type}`
- 锁的有效期：5秒
- 目的：防止同一任务在短时间内被重复下发

```javascript
// 构建锁key
let lockKey = `${registry.wxid}-${taskId}-${type}`;

// 检查是否已锁定

    logUtil.customLog(`taskId已经存在 已经被锁 ${registry.wxid} ${taskId} ${type}`);
    return; // 跳过重复任务
}

// 发送后设置锁
this.taskLock.set(lockKey, true);
setTimeout(() => {
    this.taskLock.delete(lockKey);
}, 5000);
```

## 八、超时检测机制

### 8.1 GalaxyTaskStatusTimer 定时器

**文件**: `src/msg-center/business/timer/GalaxyTaskStatusTimer.js`

```javascript
const GalaxyTaskStatusTimer = {
    checkGalaxyTaskStatus() {
        const galaxyTaskStatusMap = GalaxyTaskCache.GALAXY_TASK_STATUS_MAP;
        const galaxyTaskStatusKeys = Object.keys(galaxyTaskStatusMap);
        
        for (let i = 0; i < galaxyTaskStatusKeys.length; i++) {
            const entryKey = galaxyTaskStatusKeys[i];
            const entryValue = galaxyTaskStatusMap[entryKey];
            
            // 计算已等待时间
            const time = new Date().getTime() - entryValue.createTime;
            
            // 检查是否已完成（三段回执都收到）
            if (RECEIVE_STATUS === entryValue.firstMessageStatus 
                && RECEIVE_STATUS === entryValue.secondMessageStatus
                && RECEIVE_STATUS === entryValue.thirdMessageStatus) {
                // 任务完成，删除
                delete galaxyTaskStatusMap[entryKey];
                continue;
            }
            
            // 检查是否超时
            if (time > imgAndVideoOverTime) {
                // 超时处理
                this.handleOverTime(entryValue, entryKey);
            }
        }
    }
};
```

### 8.2 超时判断逻辑

```
任务状态 = GALAXY_TASK_STATUS_MAP[taskId]

如果 三条消息都已收到:

如果 当前时间 - 创建时间 > 超时阈值:
    → 任务超时，上报超时结果
否则:
    → 继续等待
```

### 11.2 会话ID格式差异

| 维度 | 企业微信 | 个人微信 |
|------|---------|---------|
| 私聊 | `S:发送者_接收者` | 直接使用 toUserName |
| 群聊 | `R:群ID` | chatroom@chatroom |
| 关键字段 | `conversationid` | `toUserName`/`chatroom` |

### 11.3 三段式回执差异

| 阶段 | 企业微信 | 旧版个人微信 | 微信4.0 |
|------|---------|-------------|---------|
| 第一条 | `recvmsg` 或 `sendmsg_*` | `msgreport` | `sendmessage` |
| 第二条 | `sendmsg_*` 或 `recvmsg` | `sendmessage` | `recvmsg` |
| 第三条 | `msgreport` | `recvmsg` | (两段式完成) |

### 11.4 消息类型映射差异

| 维度 | 企业微信 | 个人微信 |
|------|---------|---------|
| 统一前缀 | `sendmsg_`（转换为前缀匹配） | `sendmessage`（单一类型） |
| 回调映射key | `sendmsg_` | `sendmessage` |
| 锁粒度 | `wxId:recvmsg` | `wxId:msgreport` (旧版) / `wxId:sendmessage` (4.0) |

## 十二、异常场景分析

### 12.1 逆向未连接

**现象**：日志出现 `[dispatchInBound]:尚未和逆向建立注册信息` 或 `IPC尚未连接成功`

**原因**：
- 逆向客户端进程未启动
- IPC管道连接失败
- 注册信息被清除

**影响**：任务直接丢弃，不会进入超时检测（因为未缓存状态）

### 12.2 文件下载失败

**现象**：日志出现下载相关错误

**原因**：
- URL无效或过期
- 网络问题
- 磁盘空间不足

**影响**：
- 如果返回 `null`，任务提前终止
- 如果超时，由 `FileDownloadUtil` 的超时机制处理

### 12.3 重复任务

**现象**：日志出现 `taskId已经存在 已经被锁`

**原因**：
- 云端重发了相同的任务
- MQTT QoS=1 的重复投递

**影响**：第二次任务被过滤，只执行一次

### 12.4 任务超时

**现象**：日志出现 `[设置任务超时返回]`

**原因**（可能）：
- 逆向客户端未响应（进程异常/IPC断开）
- 逆向执行时间过长（大文件传输等）
- 三段式回执匹配失败（key不匹配）

**影响**：
- 上报 status=6（超时）到云端
- 任务状态从缓存中清除

### 12.5 频繁发送告警

**现象**：日志出现 `企业微信提示频繁，任务已被暂停`

**触发条件**：
- 逆向返回 `recvmsg` 且内容为 "消息发送过于频繁，请稍后再试"
- 或内容为 "Abnormal request frequency"

**处理**：
- 通知前端
- 缓存1小时防止重复告警

## 十三、排查指南

### 13.1 如何确认任务是否下发成功

搜索以下日志（按时间顺序）：

1. `work-wx-friend-chat - serverTask` → 确认MQTT收到任务
2. `[CloudFlowHandler]:inBound` → 确认进入入站流程

5. `[发送消息给逆向]` → **最关键：确认已发送给逆向**

如果第5步存在，说明任务已成功下发给逆向。

### 13.2 如何排查超时问题

1. 确认 `[发送消息给逆向]` 日志存在 → 任务已发给逆向
2. 搜索 `[接收逆向推送消息]` 配合 wxid 和 会话ID → 检查逆向是否返回
3. 搜索 `[设置任务超时返回]` → 确认超时上报
4. 如果逆向无返回 → 问题在逆向侧

### 13.3 关键日志搜索策略

## 附录A：日志格式说明

微信助手的日志格式为：
```
[用户名] [设备标识] [sonVersion版本] [版本号] [号数量: N] 日志内容
```

示例：
```
[wanglili06] [0mFfBxQksaAo8S3ra48QA4aRFclb3wAL_win32_10.0.22631] [sonVersion4] [5.5.0-release01] [号数量: 1] 日志内容
```

各字段含义：
- `wanglili06`：机器名/用户名
- `0mFfBx...`：设备唯一标识
- `sonVersion4`：子版本号
- `5.5.0-release01`：客户端版本
- `号数量: 1`：当前管理的机器人数量
