# 企微与微信消息上报流程 - 核心处理与转换服务

> 本文档详细说明云端出站处理 (cloudFlowOutBound)、各种 convert-service
> 的处理逻辑、企微消息记录上报、任务结果上报、以及 MQTT 发送的完整机制。

---

## 一、云端出站处理 (cloudFlowOutBound)

### 1.1 职责

文件：`src/msg-center/dispatch-center/dispatch/cloudFlowOutBound.js`

cloudFlowOutBound 接收经过三段式回执处理后的消息，遍历 convert-service 列表，
将消息转换为云端格式后通过 MQTT 上报。

### 1.2 核心逻辑

```javascript
function cloudFlowOutBound(message, pipeLineWrapper) {
    logUtil.customLog(
        `[CloudFlowHandler]:outBound执行convert-service:${message}`
    );

    if (message.startsWith('{') && message.endsWith('}')) {
        let msgObj = JSON.parse(message);
        let clientRecord = {...ClientMsgBO, ...msgObj};

        if (pipeLineWrapper.workWx) {
            // 企业微信：遍历 WorkWxConvertServiceList
            for (let i = 0; i < WorkWxConvertServiceList.length; i++) {
                const service = WorkWxConvertServiceList[i];
                if (service.filter(clientRecord)) {
                    service.operate(message, pipeLineWrapper);
                }
            }
        } else {
            // 个人微信：遍历 WxConvertServiceList
            getRealObj(clientRecord); // 清理null值
            for (let i = 0; i < WxConvertServiceList.length; i++) {
                const service = WxConvertServiceList[i];
                if (service.filter(clientRecord)) {
                    // 非登录类消息需要检查wxid
                    if (clientRecord.type
                        && !NOT_NEED_LOGIN_TYPE_SET.has(clientRecord.type)
                        && !pipeLineWrapper.wxid) {
                        break;
                    }
                    service.operate(message, pipeLineWrapper);
                }
            }
        }
    }
}
```

**重要特点：没有 break 机制**——同一条消息可能被多个 service 处理。
例如一条 msgreport 消息可能同时被 WorkWxFriendChatResponse（任务结果上报）
和 WorkWxMsgRecordService（消息记录上报）处理。

## 二、企微消息记录上报 (WorkWxMsgRecordService)

### 2.1 职责

文件：`src/msg-center/business/convert-service/workwx/workWxMsgRecordService.js`

处理企业微信的 **RECV_MSG (recvmsg)** 类型消息，将聊天记录构建为云端格式后
通过 MQTT 上报。这是**聊天记录同步**的核心服务。

### 2.2 filter 过滤

```javascript
filter(clientRecord) {
    return clientRecord.type === GalaxyCallBackType.RECV_MSG;
    // 即 type === 'recvmsg'
}
```

### 2.3 operate 处理流程

```
operate(clientMsg, pipeLineWrapper)
    │
    ├─ 1. 解析消息(parseClientMsg)
    │     获取 clientMsgBO, ownerWxId
    │
    ├─ 2. 检查消息时效性
    │     过滤过早的消息(IpcUtil检查)
    │
    ├─ 3. 判断群聊/私聊
    │     群聊: conversation_id 以 "R:" 开头
    │     私聊: conversation_id 格式 "S:xxx_xxx"
    │
    ├─ 4. 处理特殊消息类型
    │     ├─ 语音通话(content_type=53/54)
    │     ├─ 退群(content_type=1004/1005)
    │     └─ 群公告等
    │
    ├─ 5. 构建 recordInfo 基础信息
    │     { fromUsername, chatroom, taskId, content, ... }
    │
    ├─ 6. 根据 content_type 处理不同消息类型
    │     ├─ 文本(0,2): 解析 textmsg 数组
    │     ├─ 图片(7,14,101,123): 获取图片信息
    │     ├─ 视频(5,23,103): 获取视频信息
    │     ├─ 文件(8,15,102): 获取taskId + 缓存等待filepath
    │     ├─ 链接(13,1011): 解析 linkmsg 数组
    │     ├─ 语音(16): 缓存等待filepath
    │     ├─ 表情(29): 解析表情信息
    │     ├─ 名片(41): 解析联系人卡片
    │     ├─ 小程序(78): 解析小程序信息
    │     ├─ 视频号(141): 解析 finder 数组
    │     ├─ 直播(146): 解析 finderlive 数组
    │     └─ 接龙(213): 解析 solitairemsg
    │
    ├─ 7. 尝试获取 taskId (自己发送的消息)
    │     MsgTaskInfoCache.msgIdToTaskIdInfo[msgId]
    │     (最多重试3次，每次间隔200ms)
    │
    ├─ 8. 构建 clientMsgCloud
    │     { type: CHATROOM_MSG/FRIEND_MSG, recordInfo, username, ... }
    │
    └─ 9. MqttSendService.sendMessage() 上报
```

| 41 | 名片 | 联系人名片 |
| 78 | 小程序 | 小程序卡片 |
| 141 | 视频号卡片 | 视频号分享 |
| 146 | 视频号直播 | 直播分享 |

## 三、企微任务结果上报 (WorkWxFriendChatResponse)

### 3.1 职责

文件：`src/msg-center/business/convert-taskreport/workwx/wkFriendChatResponse.js`

处理企业微信**发送消息任务**的最终结果上报。当三段式回执完成后，
msgreport 消息（已携带 taskId）经过 cloudFlowOutBound 到达此服务。

### 3.2 filter 过滤

```javascript
filter(clientRecord) {
    // 获取企微发送消息的回调配置
    const byTaskType = WorkWxRelation4TaskAndCallback
        .taskCallbackMap[GalaxyTaskType.WORK_WX_SEND_PREFIX];
    if (!byTaskType) return false;
    // 匹配第三个回调类型 (msgreport)
    return byTaskType.thirdCallbackType === clientRecord.type;
}
```

### 3.3 operate 处理流程

```javascript
operate(clientMsg, pipeLineWrapper) {
    logUtil.customLog(`WorkWxFriendChatResponse 发消息任务:[${clientMsg}]`);

    // 1. 上报任务执行结果 (调用 wkTaskResponse)
    this.wkTaskResponse(clientMsg, pipeLineWrapper);

    // 2. 如果成功，构建消息记录上报数据
    const clientMsgBO = this.parseClientMsg(clientMsg);
    if (clientMsgBO.status != 0) return;

    // 3. 区分群聊/私聊
    const conversationid = clientMsgBO.conversation_id;
    const isQun = conversationid.startsWith("R:");

    // 4. 构建 recordInfo
    const recordInfo = {
        ...RecordInfo,
        fromUsername: ownerWxId,
        taskId: clientMsgBO.taskId || null,
        wechatMsgType: clientMsgBO.flag,
        content: clientMsgBO.msg,
    };

    // 5. 设置群聊/私聊信息
    if (isQun) {
        recordInfo.chatroom = conversationid;
        clientCloud.type = PrismRecordType.CHATROOM_MSG;
    } else {
        const arr = conversationid.split("_");
        recordInfo.toUsernames = [arr[arr.length - 1]];
        clientCloud.type = PrismRecordType.FRIEND_MSG;
    }
}
```

### 3.4 关键日志

| 日志模板 | 说明 |
|----------|------|
| `WorkWxFriendChatResponse 发消息任务:[${msg}]` | 进入任务结果处理 |

## 四、wkTaskResponse 任务结果上报

### 4.1 职责

文件：`src/msg-center/business/baseConvertResponse.js`

wkTaskResponse 是企业微信**任务结果上报**的核心方法，被多个任务响应服务共用。
它将任务执行状态（成功/失败/原因）封装为标准格式后通过 MQTT 发送。

### 4.2 实现

```javascript
wkTaskResponse(clientMsg, pipeLineWrapper) {
    const clientMsgBO = this.parseClientMsg(clientMsg);

    // taskId 校验
    if (!clientMsgBO.taskId) {
        logUtil.customLog(
            `[msgConvert][wkTaskResponse]taskId is null:${clientMsg}`
        );
        return;
    }
    logUtil.customLog(
        `[msgConvert][wkTaskResponse]taskId not null:${clientMsg}`
    );

    const ownerWxId = pipeLineWrapper.wxid;
    const taskInfoBO = {};

    // 设置版本和状态
    taskInfoBO.javaClient = clientVersion;
    const status = clientMsgBO.status == 0
        ? CommonConstant.SUCCESS_TASK    // 3
        : CommonConstant.FAIL_TASK;      // 4
    taskInfoBO.status = status;
    taskInfoBO.taskId = clientMsgBO.taskId;
    taskInfoBO.taskSuccess = (status == CommonConstant.SUCCESS_TASK);

    // 失败时添加原因
    if (status == CommonConstant.FAIL_TASK) {
        taskInfoBO.reason = clientMsgBO.reason
            || WxFailReasonUtil.getFailReasonWithDefault(clientMsgBO.status);
    }

    // 构建上报消息
    const clientMsgCloud = {
        taskInfo: taskInfoBO,
        type: PrismRecordType.TASK_RESULT_MSG,    // 604
        createTime: clientMsgBO.createTime ?? clientMsgBO.timer,
        currTime: clientMsgBO.timer,
        javaClientVersion: clientVersion,
        clientType: 'weixinzhushou'
    };

    // MQTT 发送
    MqttSendService.sendMessage(ownerWxId, clientMsgCloud);
}
```

### 4.3 任务状态码

| 状态码 | 常量名 | 说明 |
|:---:|------|------|
| 3 | `SUCCESS_TASK` | 任务执行成功 |
| 4 | `FAIL_TASK` | 任务执行失败 |
| 5 | `RECEIVE_TASK` | 任务已接收(中间状态) |
| 6 | `OVER_TIME_TASK` | 任务超时 |
| 7 | `MIDDLE_STATUS` | 中间状态 |

### 4.4 关键日志

| 日志模板 | 含义 | 重要性 |
|----------|------|--------|
| `[msgConvert][wkTaskResponse]taskId not null:${msg}` | taskId 存在，上报成功 | ⭐⭐⭐ |
| `[msgConvert][wkTaskResponse]taskId is null:${msg}` | taskId 为空，无法上报 | ⭐⭐⭐ |
| `[wxid-${wxid}][msgConvert][通知任务结果]:${msg}` | 任务结果已发送到MQTT | ⭐⭐ |

**`taskId is null` vs `taskId not null` 是判断任务上报是否成功的关键日志！**

## 五、MQTT 发送服务 (mqttSend)

### 5.1 职责

文件：`src/msg-center/dispatch-center/mqttSend.js`

MqttSendService 是 MQTT 发送的统一出口，所有上报消息最终都通过它发送到云端。

### 5.2 sendMessage 方法

```javascript
sendMessage(wxId, clientMsgCloud) {
    // 添加公共字段
    clientMsgCloud.clientMsgSource = 'weixinzhushou';
    clientMsgCloud.clientType = 'weixinzhushou';
    clientMsgCloud.username = wxId;
    clientMsgCloud.version = ApplicationEnv.version;

    // 通过 MQTT publish 发送
    mqttClient.publish(topic, JSON.stringify(clientMsgCloud));
}
```

## 六、企微与个微上报差异对比

### 6.1 convert-service 差异

| 特性 | 企业微信 | 个人微信 |
|------|---------|---------|
| 服务列表 | WorkWxConvertServiceList | WxConvertServiceList |
| 消息记录服务 | WorkWxMsgRecordService | MsgRecordService |
| 任务成功响应 | WorkWxFriendChatResponse | SendTextMsgResponse 等 |
| 任务失败响应 | WorkWxFriendChatResponseFail | FriendChatResponseFail |
| 上报方法 | wkTaskResponse | taskResponse / taskResponseV2 |
| null值清理 | 不清理 | getRealObj 清理 |

### 6.2 taskResponse 差异

| 特性 | wkTaskResponse (企微) | taskResponse (个微) |
|------|------|------|
| taskId为空时 | 直接 return，不上报 | 仍然上报 |
| taskSuccess字段 | 有 | 无 |
| clientType字段 | 'weixinzhushou' | 无 |
| javaClientVersion | 有 | 无 |
| 日志标识 | `[wkTaskResponse]` | `[通知任务结果]` |

### 6.3 消息处理差异

| 特性 | 企业微信 | 个人微信 |
|------|---------|---------|
| 会话ID格式 | `R:群ID` / `S:发_收` | `wxid@chatroom` / `wxid` |
| 发送者ID字段 | send_id (数字) | from (wxid字符串) |
| 消息内容结构 | textmsg/filemsg数组 | content字符串或data对象 |
| 文件处理 | FILE_KEY_MAP缓存 | 直接解析 |
| 语音处理 | voiceCallCache | 直接处理 |

## 七、前端出站处理 (frontFlowOutBound)

### 7.1 职责

文件：`src/msg-center/dispatch-center/dispatch/frontFlowOutBound.js`

将消息发送到 Electron 前端窗口，用于实时显示逆向消息。

### 7.2 过滤规则

前端出站会过滤部分消息类型，不是所有逆向消息都需要发到前端：
- `recvmsg` 类型不发送到前端（避免频繁推送）
- 应用 aspect 切面处理
