# 企微与微信消息上报流程 - 超时检测与排查指南

> 本文档详细说明任务超时检测机制、GALAXY_TASK_STATUS_MAP 状态管理、
> 常见异常场景分析、以及完整的日志排查指南。

---

## 一、超时检测机制

### 1.1 GalaxyTaskStatusTimer

文件：`src/msg-center/business/timer/GalaxyTaskStatusTimer.js`

GalaxyTaskStatusTimer 是一个定时任务，周期性扫描 `GALAXY_TASK_STATUS_MAP`，
检测哪些任务已超时未完成，并自动上报超时结果。

### 1.2 检测逻辑

```javascript
const GalaxyTaskStatusTimer = {
    checkGalaxyTaskStatus() {
        const galaxyTaskStatusMap = GalaxyTaskCache.GALAXY_TASK_STATUS_MAP;
        const keys = Object.keys(galaxyTaskStatusMap);

        for (let i = 0; i < keys.length; i++) {
            const entryKey = keys[i];
            const entryValue = galaxyTaskStatusMap[entryKey];

            // 计算已等待时间
            const time = new Date().getTime() - entryValue.createTime;

            logUtil.customLog(
                `任务超时id ${entryValue.taskId} :` +
                `[${entryKey}]的超时时间为:[${imgAndVideoOverTime}]`
            );

            // 检查1：任务是否已完成(三个状态都为1)
            if (RECEIVE_STATUS === entryValue.firstMessageStatus
                && RECEIVE_STATUS === entryValue.secondMessageStatus
                && RECEIVE_STATUS === entryValue.thirdMessageStatus) {

                logUtil.customLog(
                    `任务未超时 ${entryValue.taskId} :` +
                    `[${entryKey}]成功,删除任务`
                );
                delete galaxyTaskStatusMap[entryKey];
                continue;
            }

            // 检查2：任务是否超时
            if (time > imgAndVideoOverTime) {
                // 构建超时任务信息
                const taskInfo = {
                    ...TaskInfo,
                    status: CommonConstant.OVER_TIME_TASK,  // 6
                    reason: OVER_TIME_REASON,               // "消息发送超时"
                    javaClient: clientVersion,
                    taskId: entryValue.taskId,
                };

                const clientMsgCloud = {
                    ...ClientMsg,
                    username: entryValue.username,
                    type: PrismRecordType.TASK_RESULT_MSG,  // 604
                    taskInfo
                };

                // MQTT上报超时
                MqttSendService.sendMessage(
                    entryValue.username, clientMsgCloud
                );

                logUtil.customLog(
                    `taskId[${entryValue.taskId}]` +
                    `[设置任务超时返回]:[${entryKey}]`
                );

                delete galaxyTaskStatusMap[entryKey];
            }
        }
    }
};
```

### 1.3 超时时间配置

```javascript
// 从配置中获取超时时间(毫秒)
const imgAndVideoOverTime =
    ApplicationConfigEnv['over_time.galaxy.task.img.and.video.status'];
```

超时时间通过 Apollo 配置管理，通常为 **60秒** 左右，所有类型任务统一使用
`imgAndVideoOverTime` 这一个超时时间。

### 1.4 超时检测关键日志

| 日志模板 | 说明 |
|----------|------|
| `任务超时id ${taskId} :[${key}]的超时时间为:[${overTime}]` | 正在检测任务超时状态 |
| `任务未超时 ${taskId} :[${key}]成功,删除任务,未到超时时间:[${overTime}],time:[${time}]` | 任务已完成，从缓存删除 |
| `taskId[${taskId}][设置任务超时返回]:[${key}]` | **任务超时，已上报 status=6** |

## 二、GALAXY_TASK_STATUS_MAP 状态管理

### 2.1 数据模型

文件：`src/msg-center/core/bo/galaxy/galaxyTaskStatus.js`

```javascript
class GalaxyTaskStatus {
    constructor() {
        this.firstMessageStatus = 0;   // 第一条回执状态
        this.secondMessageStatus = 0;  // 第二条回执状态
        this.thirdMessageStatus = 0;   // 第三条回执状态
        this.createTime = Date.now();  // 创建时间戳
        this.taskId = '';              // 任务ID
        this.username = '';            // 微信ID
    }
}

const GalaxyTaskStatusConstant = {
    RECEIVE_STATUS: 1,          // 已接收
    NOT_RECEIVE_STATUS: 0,      // 未接收
    OVER_TIME_REASON: '消息发送超时'
};
```

### 2.2 状态生命周期

```
1. 任务下发时创建 (dispatchInBound.processInBoundTask):
   GALAXY_TASK_STATUS_MAP[taskId] = {
       firstMessageStatus: 0,
       secondMessageStatus: 0,
       thirdMessageStatus: 0,
       createTime: Date.now(),
       taskId: taskId,
       username: wxId
   }

2. 三段式回执完成时更新 (thirdMsgHandler):
   GALAXY_TASK_STATUS_MAP[taskId] = {
       firstMessageStatus: 1,  ← 更新
       secondMessageStatus: 1, ← 更新
       thirdMessageStatus: 1,  ← 更新
       ...
   }

3. 定时器检测时判断:
   a. 三个状态都为1 → 任务完成，删除缓存
   b. 超过超时时间 → 上报超时(status=6)，删除缓存
```

### 2.3 状态流转图

```
正常流程:
    创建{0,0,0} ──→ thirdMsgHandler ──→ {1,1,1} ──→ 定时器删除(已完成)

超时流程(第二条缺失):
    创建{0,0,0} ──→ (三段式匹配失败) ──→ 状态不变{0,0,0}
         │
         │  等待超时...
         ▼
    定时器检测 ──→ time > imgAndVideoOverTime ──→ 上报status=6 ──→ 删除

超时流程(逆向无响应):
    创建{0,0,0} ──→ (逆向完全没有返回) ──→ 状态不变{0,0,0}
         │
         │  等待超时...
         ▼
    定时器检测 ──→ time > imgAndVideoOverTime ──→ 上报status=6 ──→ 删除
```

## 三、常见异常场景分析

### 3.1 第二条回执缺失（本次工单场景）

**现象**：任务超时上报，但消息实际已发送成功

**原因**：逆向对特定消息类型（如图片）没有返回第二条 `sendmsg_*` 回执

**日志特征**：
```
[接收逆向推送消息] messageType-recvmsg          ← 第一条正常
[接收逆向推送消息] messageType-AddOrUpdateFiles  ← 文件更新(非回执)
[接收逆向推送消息] messageType-msgreport         ← 第三条到达
[msgConvert]收到消息res: errorOrder              ← 进入兜底
[没有匹配到taskId] errorOrder                   ← 匹配失败
[msgConvert][wkTaskResponse]taskId is null       ← 无法上报
taskId[xxx][设置任务超时返回]                     ← 最终超时
```

**排查步骤**：
1. 搜索 `[发送消息给逆向]` 确认任务已下发
2. 搜索 `client_id` 追踪三段式回执
3. 检查是否有 `sendmsg_*` 类型的返回
4. 如果只有 recvmsg 和 msgreport，确认第二条缺失

### 3.2 逆向完全无响应

**现象**：任务超时，没有任何逆向回执日志

**日志特征**：
```
[发送消息给逆向] wxid: [wxid-xxx] type=[sendmsg_sendtext]  ← 已下发
(后续无任何该任务的回执日志)
taskId[xxx][设置任务超时返回]                                ← 超时
```

**可能原因**：
- 逆向进程崩溃
- IPC 管道断开
- 企微/微信客户端异常
- 网络问题导致消息未送达

### 3.3 三段式匹配错乱

**现象**：消息被错误关联，taskId 匹配到错误的任务

**可能原因**：
- 同一用户短时间内对同一目标发送多条消息
- 消息节点过期被清理
- client_id 冲突（极罕见）

**日志特征**：
```
[msgConvert] msgResNodeMap 数量过大(>200)
[onMsgResNodeMapOverMax] msgResNodeMap xxx
```

### 3.4 消息节点过期

**现象**：第三条消息到达时节点已被清理

**日志特征**：
```
清理过期的 msgResNodeMap 条目: [key]
[msgConvert]收到消息res: errorOrder    ← 节点不存在，进入兜底
```

### 3.5 MQTT 发送失败

**现象**：任务结果构建成功但未上报到云端

**日志特征**：
```
[msgConvert][wkTaskResponse]taskId not null  ← 构建成功
(但云端未收到结果)
```

## 四、日志排查指南

### 4.1 如何搜索逆向回执消息

**❌ 错误方式**：
```
用 taskId 直接搜索逆向消息
(逆向返回的 recvmsg/msgreport 不携带 taskId！)
```

**✅ 正确方式**：

| 搜索条件 | 说明 | 适用场景 |
|----------|------|---------|
| `wxid AND conversation_id` | 按微信ID+会话搜索 | 最常用 |
| `client_id值` | 按消息唯一标识搜索 | 追踪特定消息链 |
| `wxid AND recvmsg` | 搜索第一条回执 | 确认逆向是否响应 |
| `wxid AND sendmsg_sendimage` | 搜索第二条回执 | 确认特定类型回执 |
| `wxid AND msgreport` | 搜索第三条回执 | 确认服务端确认 |
| `taskId值 AND 设置任务超时返回` | 搜索超时记录 | 确认是否超时 |
| `taskId值 AND wkTaskResponse` | 搜索上报结果 | 确认是否成功上报 |

### 4.2 完整排查流程

```
Step 1: 确认任务是否下发
   搜索: "[发送消息给逆向] wxid: [wxid-xxx] type=[sendmsg_xxx]"
   └── 找到 → 任务已下发，继续 Step 2
   └── 没找到 → 问题在下发环节，检查 dispatchInBound

Step 2: 确认逆向是否有响应
   搜索: "[接收逆向推送消息] messageType-recvmsg" AND conversation_id
   └── 找到 → 逆向有响应，记录 client_id，继续 Step 3
   └── 没找到 → 逆向无响应，检查 IPC 连接和逆向进程状态

Step 3: 用 client_id 追踪三段式回执
   搜索: client_id 的值
   └── 应该找到3条消息：
       ① recvmsg (第一条)
       ② sendmsg_* (第二条，携带 taskId)
       ③ msgreport (第三条)
   └── 如果缺失某条 → 确认缺失的是哪一条

Step 4: 检查三段式匹配结果
   搜索: "[msgConvert]收到消息res"
   └── 找到"第三条" → 三段式成功完成，继续 Step 5
   └── 找到"errorOrder" → 匹配失败，检查缺失原因

Step 5: 检查任务上报结果
   搜索: "[wkTaskResponse]taskId"
   └── 找到"not null" → 任务成功上报
   └── 找到"is null" → taskId 为空，无法上报

Step 6: 检查超时状态
   搜索: "[设置任务超时返回]" AND taskId
   └── 找到 → 任务最终超时上报(status=6)
   └── 没找到 → 查看"任务未超时"确认是否已正常完成
```

## 六、总结：上报流程完整日志链路

以一条**企微群发图片消息**为例，完整的日志链路如下：

```
1. [CloudFlowHandler]:inBound, wxId=xxx, message={...taskId...type:sendmsg_sendimage...}
2. [dispatchInBound] 开始执行
3. [DispatchCenter]:下任务时往缓存里放任务:{...},缓存:{firstMessageStatus:0,...}
4. [发送消息给逆向] wxid: [wxid-xxx] type=[sendmsg_sendimage]

   === IPC发送完成，等待逆向响应 ===

5. [接收逆向推送消息] messageType-recvmsg
6. pipeCode:xxx selectCode:xxx 接收到IPC消息:{...type:recvmsg...}
7. [DispatchCenter] [msgHandleBase]: outBoundMsg sendType = recvmsg
8. [msgConvert]收到消息res：第一条：{first:{client_id:xxx,type:recvmsg}}

9. [接收逆向推送消息] messageType-sendmsg_sendimage
10. [DispatchCenter] [msgHandleBase]: outBoundMsg sendType = sendmsg_sendimage
11. [msgConvert]收到消息res：第二条：{first:{...},second:{taskId:xxx,...}}

12. [接收逆向推送消息] messageType-msgreport
13. [DispatchCenter] [msgHandleBase]: outBoundMsg sendType = msgreport
14. [msgConvert]收到消息res：第三条：{first:{...},second:{...},third:{...}}
15. [msgConvert]返回的第三条消息内容：{...taskId:xxx...}

16. [CloudFlowHandler]:outBound执行convert-service:{...}
17. WorkWxFriendChatResponse 发消息任务:[...]
18. [msgConvert][wkTaskResponse]taskId not null:{...}
19. [wxid-xxx][msgConvert][通知任务结果]:{type:604,taskInfo:{status:3,...}}

   === 上报完成 ===
```

如果第二条缺失，第8步后直接跳到第12步，然后：
```
12. [msgConvert]收到消息res: errorOrder
13. [没有匹配到taskId] errorOrder
14. [msgConvert][wkTaskResponse]taskId is null
    ...
15. 任务超时id xxx
16. taskId[xxx][设置任务超时返回]
```
