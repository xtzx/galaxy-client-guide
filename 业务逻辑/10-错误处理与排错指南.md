## 一、错误处理机制

### 1.1 日志系统

项目使用 `electron-log` + `ali-sls` 双重日志：

```javascript
// src/init/log.js

const logUtil = {
    customLog: (message, options = {}) => {
        const { level = 'info', errorKey, wxId, taskId } = options;

        // 1. 本地日志
        log[level](message);

        // 2. 错误上报到云端
        if (level === 'error') {
            slsLog.reportError({
                message,
                errorKey,
                wxId,
                taskId,
                stack: options.error?.stack
            });
        }
    }
};
```

### 1.2 错误分类

| 错误类型 | errorKey | 处理方式 |
|---------|----------|---------|
| 连接错误 | `ConnectionError` | 重连 |
| 任务执行错误 | `TaskError` | 上报失败状态 |
| 系统错误 | `SystemError` | 记录并告警 |
| 代码错误 | `CodeError` | 记录堆栈 |

## 二、常见错误与解决方案

### 2.1 IPC 连接问题

#### 错误：无法连接到逆向服务

```
[IPC] 连接失败: Error: Cannot find module PipeCore.dll
```

**原因：**
1. DLL 文件未下载
2. DLL 文件路径错误
3. 逆向服务未启动

**解决方案：**
```bash
# 1. 检查 DLL 是否存在
ls ./bin/

# 2. 重新下载逆向文件
node ./scripts/downloadDevFile.js 5.4.2-release01

# 3. 检查进程是否启动
# Windows 任务管理器查看 BasicService.exe
```

#### 错误：IPC 连接断开

```
[IPC] 连接断开: wxid_xxx
```

**原因：**
1. 微信/企微客户端关闭
2. 逆向服务崩溃
3. 管道通信异常

**解决方案：**
```javascript
// 代码中自动重连逻辑
// src/msg-center/core/reverse/asyncSelectTask.js

if (disconnected) {
    // 从注册表移除
    RegistryConfig.removeRegistry(pipeLineWrapper.id);

    // 通知前端
    frontSend.sendLoginStatus(wxId, 'offline');

    // 5秒后重新扫描
    setTimeout(() => {
        initIpcTask.scanConnections();
    }, 5000);
}
```

### 2.2 MQTT 连接问题

#### 错误：MQTT 连接失败

```
[MQTT] 连接失败: Error: Connection refused
```

**原因：**
1. 网络问题
2. MQTT Broker 地址错误
3. 认证失败（密码签名错误）

**解决方案：**
```javascript
// 1. 检查网络连接
// 2. 检查 MQTT 配置
// src/msg-center/core/mq/mqttConfig.js

const config = {
    endPoint: 'xxx.mqtt.aliyuncs.com',
    port: 1883,
    // 检查签名是否正确
};

// 3. 自动重连已内置
client.on('error', (error) => {
    logUtil.error('[MQTT] 连接错误', error);
    // 自动重连
});
```

#### 错误：消息未收到

**排查步骤：**
1. 检查订阅主题是否正确
2. 检查消息是否过期（默认3小时）
3. 检查去重逻辑

```javascript
// 开启调试日志
// src/msg-center/core/mq/mqttClientBase.js

client.on('message', (topic, message) => {
    console.log('[MQTT Debug] 收到消息:', topic, message.toString());
});
```

### 2.3 任务执行问题

#### 错误：任务超时

```
[Task] 任务超时: taskId=xxx
```

**原因：**
1. 逆向服务响应慢
2. 三段式消息未完整接收
3. 任务队列阻塞

**解决方案：**
```javascript
// 检查任务缓存
const cache = GalaxyTaskCache.get(taskId);
console.log('任务状态:', cache);

// 检查消息节点
const node = MsgHandleBase.msgResNodeMap[key];
console.log('消息节点:', node);
```

#### 错误：发送消息失败

```
[Chat] 消息发送失败: status=1
```

**常见原因与解决：**

| 状态码 | 原因 | 解决方案 |
|-------|------|---------|
| 1 | 一般失败 | 检查目标用户是否存在 |
| 2 | 被拉黑 | 无法恢复 |
| 3 | 频率限制 | 降低发送频率 |
| 4 | 账号异常 | 检查微信状态 |

---

### 2.4 数据库问题

#### 错误：数据库锁定

```
SQLITE_BUSY: database is locked
```

**原因：**
1. 并发写入
2. 长事务未提交

**解决方案：**
```javascript
// 使用 async-lock 避免并发
const lock = new AsyncLock();

await lock.acquire('db-write', async () => {
    await db.save(data);
});

// 或调整 SQLite 配置
const sequelize = new Sequelize({
    dialect: 'sqlite',
    dialectOptions: {
        busyTimeout: 10000  // 等待10秒
    }
});
```

## 四、错误上报流程

### 4.1 错误捕获

```javascript
// 全局未捕获异常
process.on('uncaughtException', (error) => {
    logUtil.customLog(`[uncaughtException] ${error.message}`, {
        level: 'error',
        errorKey: 'UncaughtException',
        error
    });
});

// Promise 未处理拒绝
process.on('unhandledRejection', (reason) => {
    logUtil.customLog(`[unhandledRejection] ${reason}`, {
        level: 'error',
        errorKey: 'UnhandledRejection'
    });
});
```

## 五、健康检查

### 5.1 连接状态检查

```javascript
// 检查所有连接状态
function healthCheck() {
    const registryList = RegistryConfig.getAllRegistries();

    registryList.forEach(registry => {
        const status = {
            wxId: registry.wxid,
            ipc: registry.pipeLineWrapper?.available,
            mqtt: registry.mqttClient?.connected,
            lastPong: registry.lastPongTime
        };

        logUtil.customLog('[HealthCheck]', { ...status });
    });
}

// 定期检查
setInterval(healthCheck, 60000);
```

### 5.2 自动恢复

```javascript
// MQTT 断线自动重连
client.on('close', () => {
```

## 六、常见问题 FAQ

### Q1: 微信显示已登录但系统未识别

**A:** 检查 IPC 连接是否建立：
1. 查看日志中是否有 `[IPC] 连接成功`
2. 检查 DLL 是否正确注入
3. 重启微信客户端

### Q2: 消息发送成功但云端未收到回执

**A:** 检查三段式消息：
1. 查看 `msgResNodeMap` 是否有对应节点
2. 检查第三条消息是否超时
3. 检查 MQTT 上报是否成功

### Q3: 任务执行成功率低

**A:** 排查步骤：
1. 检查任务是否过期（3小时）
2. 检查目标用户是否有效
3. 检查发送频率是否过高
4. 检查账号是否被限制

### Q4: 内存占用持续增长

**A:** 可能的内存泄漏：
1. 检查 `msgResNodeMap` 是否正确清理
2. 检查 `GalaxyTaskCache` 是否正确清理
3. 检查事件监听器是否移除

```javascript
// 定期清理
setInterval(() => {
    MsgHandleBase.cleanExpiredNodes();
    GalaxyTaskCache.cleanExpired();
}, 10 * 60 * 1000);
```

### Q5: 应用崩溃后如何恢复

**A:**
1. 查看崩溃日志
2. 检查 `activityInfo` 标记
3. 重启应用自动恢复

```javascript
// 崩溃检测
// src/msg-center/start/appStart.js
function judgeCrashAndReport() {
    const info = store.getActivityInfo();
    if (info && info.status !== 1) {
        // 上次未正常退出
        reportCrash(info);
    }
}
```
