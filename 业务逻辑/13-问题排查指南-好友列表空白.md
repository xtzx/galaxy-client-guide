# 问题排查指南：好友列表空白

> 本文档记录了一次实际问题的排查过程，可作为类似问题的排查模板

---

## 一、问题描述

### 1.1 现象

- 用户登录后，前端好友列表页面显示**空白**
- 后台日志显示逆向服务返回了数据（31条记录）

### 1.2 关键日志

```
[lixiaolu02] ... [handlerGetUserList] [wxid-wxid_h3l3mra44vtn22]
逆向返回好友&群列表总数:[31] 过滤后数: 31
```

看起来有数据，但前端为什么是空白？

---

## 二、数据流分析

### 2.1 正常数据流

```
① 前端请求 getuserlist
    ↓
② 逆向返回 getuserlist（只有 wxid 数组）
    ↓
③ 客户端分页请求 userinfolist（请求详细信息）
    ↓
④ 逆向返回 userinfolist（应该是对象数组，包含 nickname/headimg 等）
    ↓
⑤ formatChatroomOrUserToList() 分离群和好友
    ↓
⑥ handleSendFriendToFront() 发送给前端
```

### 2.2 关键处理点

| 步骤 | 方法 | 文件位置 | 作用 |
|-----|------|---------|-----|
| ② | `handlerGetUserList()` | 行 191-274 | 接收 wxid 列表，发起分页请求 |
| ④ | `execute()` | 行 128-151 | 处理 userinfolist 响应 |
| ⑤ | `formatChatroomOrUserToList()` | **行 551-567** | 分离群和好友（**问题点**） |
| ⑥ | `handleSendFriendToFront()` | 行 696-770 | 发送给前端 |

---

## 三、排查步骤

### 3.1 第一步：确认初始数据是否正常

**阿里云 SLS 查询**：

```sql
* and lixiaolu02 and handlerGetUserList | select message
```

**预期结果**：

```
逆向返回好友&群列表总数:[31] 过滤后数: 31
```

✅ 数据正常，继续下一步

---

### 3.2 第二步：确认分页请求是否发出

**查询**：

```sql
* and lixiaolu02 and fetchRoomMembers and forItem | select message
```

**预期结果**：

```
[fetchRoomMembers] [forItem] [wxid-wxid_xxx] [1604] [31] pageNum=1
```

✅ 分页请求已发出

---

### 3.3 第三步：确认分页响应是否处理

**查询**：

```sql
* and lixiaolu02 and handleRoomMembers | select message
```

**预期结果**：

```
WxUserListResponseMsgHandler handleRoomMembers ... [好友列表全部获取成功]
```

✅ 响应已收到

---

### 3.4 第四步：确认数据发送状态

**查询**：

```sql
* and lixiaolu02 and handleSendFriendToFront | select message
```

**实际结果**：

```
[handleSendFriendToFront] 无数据需要发送到前端  ❌
```

⚠️ **问题定位**：数据在发送前变成空了！

---

### 3.5 第五步：确认分类结果

**查询群数据分类**：

```sql
* and lixiaolu02 and "群总数" | select message
```

**实际结果**：

```
wxUserListResponseMsgHandler [userinfolist] ... 群总数：0 好友总数：0  ❌
```

⚠️ **问题进一步缩小**：`formatChatroomOrUserToList()` 返回了空数组！

---

### 3.6 第六步：确认原始数据格式

**查询逆向返回的原始数据**：

```sql
* and lixiaolu02 and "发送前端WS消息" and userinfolist | select message
```

**实际结果**：

```json
{
  "body": {
    "data": {
      "array": ["wxid_xxx", "wxid_yyy", "wxid_zzz", ...]  // ❌ 纯字符串数组
    },
    "type": "userinfolist"
  }
}
```

**问题根因确认**：逆向返回的是**字符串数组**，而非**对象数组**！

---

## 四、问题根因

### 4.1 代码分析

```javascript
// wxUserListResponseMsgHandler.js 第 551-567 行
formatChatroomOrUserToList(userIdList) {
    const chatroomIdList = [];
    const friendIdList = [];

    userIdList.forEach((item) => {
        // 当 item = "wxid_xxx" (字符串) 时：
        // item.wxid = undefined
        // item.nickname = undefined

        if (`${item.wxid}`.indexOf("@chatroom") > -1) {
            // "undefined".indexOf("@chatroom") = -1，不会进入
            chatroomIdList.push(item);
        } else {
            if (item.nickname) {
                // item.nickname = undefined，条件为 false
                // 所以不会添加到 friendIdList ❌
                friendIdList.push(item);
            }
        }
    });

    return { chatroomIdList, friendIdList };
    // 结果：{ chatroomIdList: [], friendIdList: [] }
}
```

### 4.2 期望数据格式

```javascript
// 逆向应该返回的格式（对象数组）
{
    "objarray": [
        {
            "wxid": "wxid_xxx",
            "nickname": "张三",      // ← 必须有
            "headimg": "http://...",
            "contacttype": 1
        },
        // ...
    ]
}
```

### 4.3 实际数据格式

```javascript
// 逆向实际返回的格式（字符串数组）
{
    "data": {
        "array": ["wxid_xxx", "wxid_yyy", ...]  // ← 没有 nickname 等字段
    }
}
```

## 五、结论

| 类型 | 说明 |
|-----|-----|
| **问题类型** | 数据格式不匹配 |
| **责任方** | 逆向服务（BasicService.exe） |
| **客户端是否有 Bug** | 否 |
| **解决方案** | 联系逆向服务开发者修复返回数据格式 |

### 5.1 给逆向开发者的问题描述

```
【问题】userinfolist 响应数据格式异常

【用户】lixiaolu02
【微信号】wxid_h3l3mra44vtn22
【时间】2026-01-27

【期望格式】
{
    "objarray": [
        {"wxid": "xxx", "nickname": "张三", "headimg": "...", "contacttype": 1}
    ]
}

【实际格式】
{
    "data": {
        "array": ["wxid_xxx", "wxid_xxx", ...]
    }
}

【影响】客户端无法解析好友信息，前端显示空白

【佐证日志】
1. "群总数：0 好友总数：0" - 数据分类失败
2. "[handleSendFriendToFront] 无数据需要发送到前端" - 无数据可发送
```

## 六、排查清单模板

当遇到**好友/群列表显示空白**问题时，按以下顺序排查：

```
□ 1. 检查 handlerGetUserList 日志（初始数据）
    关键词: handlerGetUserList AND 逆向返回

□ 2. 检查 fetchRoomMembers 日志（分页请求）
    关键词: fetchRoomMembers AND forItem

□ 3. 检查 handleRoomMembers 日志（分页响应）
    关键词: handleRoomMembers

□ 4. 检查 handleSendFriendToFront 日志（发送状态）
    关键词: handleSendFriendToFront

□ 5. 检查分类结果日志
    关键词: wxUserListResponseMsgHandler AND 群总数

□ 6. 检查原始数据格式
    关键词: 发送前端WS消息 AND userinfolist

□ 7. 检查是否有错误日志
    关键词: error OR codeError
```

## 七、阿里云 SLS 日志查询指南

### 7.1 基础信息

| 项目 | 值 |
|-----|---|
| 日志服务名称 | `gaotu-wxzs-client-node-log` |
| 地域 | `cn-beijing` |
| 端点 | `gaotu-new.cn-beijing-intranet.log.aliyuncs.com` |

### 7.2 查询语法基础

#### 基本搜索

```sql
-- 精确搜索（包含关键词）
* and 关键词

-- 多关键词 AND
* and 关键词1 and 关键词2

-- 多关键词 OR
* and (关键词1 or 关键词2)

-- 排除某关键词
* and 关键词1 not 关键词2
```

#### 指定字段搜索

```sql
-- 按用户名搜索
* and lixiaolu02

-- 按微信 ID 搜索
* and wxid_h3l3mra44vtn22

-- 按日志级别搜索
* and level:error
* and level:warn
```

#### 字符串包含

```sql
-- message 字段包含某字符串
message: "handleSendFriendToFront"

-- 通配符搜索
message: handle*
```

### 7.3 常用查询模板

#### 查找某用户的所有日志

```sql
* and lixiaolu02 | select * order by __time__ desc limit 100
```

#### 查找错误日志

```sql
* and lixiaolu02 and (error or Error or ERROR) | select message, __time__
```

#### 查找某时间段的日志

```sql
* and lixiaolu02 | select * where __time__ > 1706300000 and __time__ < 1706400000
```

#### 统计日志数量

```sql
* and lixiaolu02 | select count(*) as cnt
```

#### 按类型分组统计

```sql
* and lixiaolu02 | select message, count(*) as cnt group by message order by cnt desc limit 20
```

### 7.4 查询注意事项

#### 时间范围

- **默认只查最近 15 分钟**，需要手动调整时间范围
- 在查询框上方选择时间范围：「最近 1 小时」「最近 24 小时」「自定义」

#### 查询限制

- 单次查询最多返回 **10000** 条
- 如果日志量大，使用 `limit` 限制返回数量
- 使用 `order by __time__ desc` 获取最新日志

#### 特殊字符转义

```sql
-- 查询包含 [ ] 的内容，需要用引号包裹
* and "[handleSendFriendToFront]"

-- 查询包含冒号的内容
* and "群总数：0"
```

#### 字段说明

| 字段 | 说明 |
|-----|-----|
| `__time__` | 日志时间戳（Unix 秒） |
| `__source__` | 日志来源 |
| `message` | 日志内容 |
| `level` | 日志级别（INFO/WARN/ERROR） |

### 7.6 Web 控制台使用步骤

1. 登录阿里云控制台
2. 进入「日志服务 SLS」
3. 选择对应的 Project 和 Logstore
4. 点击「查询/分析」
5. 在搜索框输入查询语句
6. 设置时间范围
7. 点击「查询/分析」按钮

### 7.7 常见问题

| 问题 | 解决方案 |
|-----|---------|
| 查不到日志 | 检查时间范围是否正确 |
| 返回空结果 | 检查关键词是否正确，注意大小写 |
| 日志量太大 | 增加过滤条件或使用 `limit` |
| 查询超时 | 缩小时间范围或简化查询语句 |
