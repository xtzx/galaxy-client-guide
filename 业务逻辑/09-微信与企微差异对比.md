# 微信与企微差异对比

> 微信和企业微信在项目中的实现差异

---

## 一、概述

项目同时支持微信（WeChat）和企业微信（WXWork），通过 `workWx` 标志位区分：

```javascript
// 判断是否是企业微信
if (registry.workWx) {
    // 企业微信逻辑
} else {
    // 微信逻辑
}
```

---

## 二、架构差异

### 2.1 文件组织

```
src/msg-center/business/
├── convert-service/
│   ├── loginService.js           # 微信登录（通用）
│   ├── recvMsgService.js         # 微信消息接收
│   └── workwx/                   # 企微专用服务
│       ├── workWxLoginService.js
│       └── workWxRecvMsgService.js
│
├── convert-response/
│   ├── sendTextmsgResponse.js    # 微信回执
│   └── workwx/                   # 企微专用回执
│       └── workWxSendTextmsgResponse.js
│
├── task-mqtt/
│   ├── mqttChatService.js        # 微信群发
│   └── wkwx/                     # 企微专用任务
│       └── mqttWorkWxChatService.js
```

### 2.2 处理器映射

```javascript
// src/msg-center/core/mq/mqttClientBase.js

// 微信任务处理器列表
const WxConvertServiceList = [
    mqttChatService,
    mqttKickOutService,
    mqttFriendPassService,
    // ...
];

// 企业微信任务处理器列表
const WorkWxConvertServiceList = [
    mqttWorkWxChatService,
    mqttWorkWxKickOutService,
    mqttWorkFriendPassService,
    // ...
];

// 根据 workWx 标志选择处理器
const serviceList = workWxFlag ? WorkWxConvertServiceList : WxConvertServiceList;
```

## 三、登录差异

### 3.1 登录流程对比

| 步骤 | 微信 | 企业微信 |
|-----|------|---------|
| 启动方式 | 启动 WeChat.exe | 启动 WXWork.exe |
| DLL 注入 | Galaxy.dll | GalaxyWork.dll |
| 登录方式 | 扫码登录 | 扫码/账号密码 |
| 进程识别 | 进程名 WeChat.exe | 进程名 WXWork.exe |

### 3.2 登录数据结构

```javascript
// 微信登录信息
{
    type: "login",
    wxid: "wxid_xxx",
    nickname: "昵称",
    headImgUrl: "头像URL",
    // ...
}

// 企业微信登录信息
{
    type: "login",
    wxid: "wmid_xxx",           // 注意：企微ID格式不同
    corpId: "企业ID",
    nickname: "昵称",
    headImgUrl: "头像URL",
    // ...
}
```

### 3.3 代码实现

```javascript
// src/msg-center/business/convert-service/loginService.js

const LoginService = {
    async handleLogin(pipeLineWrapper, clientOtherMsgBO, ownerWxId, workWxFlag) {
        // 1. 绑定用户信息（微信/企微通用）
        this.bindWxId(ownerWxId, clientOtherMsgBO, registry, workWxFlag);

        // 2. 注册到云端
        const registeredUrl = workWxFlag
            ? logicRegisteredCloudWorkWx   // 企微注册接口
            : logicRegisteredCloud;        // 微信注册接口

        await httpFetch({ url: registeredUrl, data: robotLoginRecord });

        // 3. 建立 MQTT 连接（主题不同）
        await MqttHelper.connectMqtt({
            wxId: ownerWxId,
            workWx: workWxFlag             // 标识企微
        });
    }
};
```

## 四、消息差异

### 4.1 消息类型对比

| 消息类型 | 微信 weChatMsgType | 企微 workWxMsgType |
|---------|-------------------|-------------------|
| 文本 | 1 | 1 |
| 图片 | 3 | 3 |
| 名片 | 42 | 42 |
| 视频 | 43 | 43 |
| 表情 | 47 | 47 |
| 链接 | 49 | 49 |
| 小程序 | 33/36 | 不支持 |
| 企微专属 | - | 10000+ |

### 4.2 消息发送差异

```javascript
// 微信发送文本
{
    type: "sendtextmsg",
    wxid: "wxid_xxx",
    toUser: "接收者wxid",
    content: "消息内容"
}

// 企业微信发送文本
{
    type: "sendtextmsg",
    wxid: "wmid_xxx",
    toUser: "接收者wmid",
    content: "消息内容",
    atList: ["@成员1", "@成员2"]  // 企微支持@功能
}
```

### 4.3 消息处理器

```javascript
// src/msg-center/dispatch-center/handle/msgHandleBase.js

// 根据 workWx 标志选择不同的处理器
if (workWxFlag) {
    WorkWxMsgHandle.outBoundMsg(msgObj, pipeLineWrapper, sendType);
} else {
    WxMsgHandle.outBoundMsg(msgObj, pipeLineWrapper, sendType);
}
```

        // ...

    client.subscribe(topics, { qos: 1 });

## 十、开发注意事项

### 10.1 新增功能时

```javascript
// 1. 判断是否需要区分微信/企微
if (needsSeparateLogic) {
    // 微信逻辑
    // task-mqtt/xxxService.js

    // 企微逻辑
    // task-mqtt/wkwx/xxxWorkWxService.js
}

// 2. 注册到对应的处理器列表
WxConvertServiceList.push(newWxService);
WorkWxConvertServiceList.push(newWorkWxService);
```

### 10.2 测试时

- 微信和企微需要分别测试
- 注意 ID 格式差异（wxid_ vs wmid_）
- 企微可能需要企业管理员权限

### 10.3 日志区分

```javascript
// 日志中标识是微信还是企微
logUtil.customLog(`[${workWxFlag ? 'WorkWx' : 'Wx'}] 操作完成`, {
    wxId,
    workWx: workWxFlag
});
```
