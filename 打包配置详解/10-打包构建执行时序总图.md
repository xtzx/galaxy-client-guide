# 10 — 打包构建执行时序总图

> 本文档用时序图完整展示微信助手从源码到安装包的构建全过程，分为**开发环境**和**生产环境（CI/CD）**两条线路。

---

## 一、总览：两种构建模式

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        构建模式对比                                     │
├──────────────────────────────┬──────────────────────────────────────────┤
│      开发环境                │       生产环境（CI/CD）                  │
├──────────────────────────────┼──────────────────────────────────────────┤
│ 命令：npm run start:dev      │ 命令：npm run build                     │
│ 不打包，直接运行源码         │ 完整打包生成 .exe 安装包                │
│ 读取 runtime.yml             │ 读取 runtime.yml → 复制为 app-config.yml│
│     + runtime.dev.yml 覆盖   │ DLL 从远程下载                          │
│ DLL 使用本地已有文件         │ 产物上传到阿里云 OSS                    │
│ 无 Webpack 打包主进程        │ Webpack 打包主进程                      │
│ 无签名                       │ eToken 硬件签名                        │
└──────────────────────────────┴──────────────────────────────────────────┘
```

## 二、开发环境启动时序

```
npm run start:dev
│
│  实际命令：
│  cross-env ELECTRON_NODE_ENV=dev PRODUCT=weixinzhushou electron --inspect=5678 ./src/electron.js
│
▼
┌─────────────────────────────────────────────────────────────────────────┐
│ 阶段 A：Electron 主进程启动                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ① electron --inspect=5678 ./src/electron.js                           │
│     │  • 设置环境变量 ELECTRON_NODE_ENV=dev, PRODUCT=weixinzhushou     │
│     │  • --inspect=5678 开启 Node.js 调试端口                          │
│     │  • 直接运行源码（不经过 Webpack 编译）                           │
│     │                                                                   │
│  ② electron.js 执行初始化                                              │
│     │  global.readyStartTime = Date.now()                              │
│     │  initLog()                       ← 初始化日志系统                │
│     │  addAppEvent()                   ← 注册应用生命周期事件          │
│     │    ├─ requestSingleInstanceLock() ← 单实例锁                     │
│     │    └─ app.on('ready', ...)       ← 注册 ready 回调              │
│     │  addIpcEvent()                   ← 注册 IPC 通信处理器           │
│     │  addStoreEvent()                 ← 注册存储事件                  │
│     │  AppStart.run()                  ← 启动消息中心                  │
│     │                                                                   │
└─────┼───────────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ 阶段 B：配置加载（开发环境特有的双配置合并）                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  getAppConfig() 首次调用时：                                            │
│     │                                                                   │
│     ├─ ① 读取 config/weixinzhushou/runtime.yml                        │
│     │     → 解析为 JS 对象（完整配置：modules, elfkey, switch 等）     │
│     │                                                                   │
│     ├─ ② isDev === true → 读取 config/weixinzhushou/runtime.dev.yml   │
│     │     → 解析为 JS 对象（覆盖配置：switch.report, switch.debug）    │
│     │                                                                   │
│     ├─ ③ Object.assign(appConfig, devConfig)  ⚠️ 浅合并               │
│     │     → switch.report.kibana = false  （关闭日志上报）              │
│     │     → switch.report.sentry = false  （关闭错误上报）              │
│     │     → switch.debug.wsLogger = true  （开启 WS 日志）             │
│     │                                                                   │
│     └─ ④ 缓存到模块级变量 appConfig                                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ 阶段 C：app.on('ready') 触发                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ① setProtocol(appConfig.protocol)     ← 注册 wxzsclient:// 协议      │
│  ② initWindow()                        ← 创建主窗口                    │
│     ├─ createStateWindow({...})         ← 1060×680 窗口                │
│     ├─ mainWindow.loadURL(url)          ← 加载前端页面                  │
│     │    开发环境：http://127.0.0.1:3000/load.html                     │
│     │    生产环境：file:///...build/load.html                          │
│     └─ initUpdater()                    ← 初始化自动更新               │
│  ③ initRegedit()                        ← 初始化注册表                  │
│  ④ judgeHostsAndRunBat()                ← 检测 hosts 文件              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ 阶段 D：渲染进程初始化                                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  load.html 加载后：                                                     │
│  ① ipc.sendSync('get-app-config')      ← 获取运行时配置                │
│     → 主进程返回完整 appConfig 对象                                     │

│  ③ project.getSwitch('debug.wsLogger') → true   → WS 日志开启         │
│  ④ 渲染前端 React 页面                                                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 四、开发环境 vs 生产环境关键差异对照

| 环节 | 开发环境 | 生产环境（CI/CD） |
|------|---------|------------------|
| **启动命令** | `npm run start:dev` | `npm run build` (→ `scripts/build.js`) |
| **主进程代码** | 直接运行 `src/electron.js` 源码 | 未经过 Webpack 打包（当前项目特殊） |
| **环境变量** | `ELECTRON_NODE_ENV=dev` | 由版本号推断 (`release`→`prod`) |
| **配置来源** | `runtime.yml` + `runtime.dev.yml` 合并 | `runtime.yml` → 复制为 `app-config.yml` |
| **DLL 文件** | 使用本地 `extraResources/Inject/` 已有文件 | 从内网 CI 服务器下载对应版本 |
| **签名** | 无 | eToken 硬件签名（SHA-256） |
| **产物** | 无（直接运行） | `.exe` 安装包 + `latest.yml` |
| **页面加载** | `http://127.0.0.1:3000/load.html`（dev server） | `file:///...build/load.html`（本地文件） |
| **日志上报** | 全部关闭 | Kibana/Sentry/StarEyes 全部开启 |
| **WS 调试日志** | 开启 | 关闭 |
