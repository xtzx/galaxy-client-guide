# 05 — 其他配置项与完整配置清单

> 除了 `directories`、`publish`、`nsis`、`win` 四大块之外，electron-builder 还有大量顶层配置、其他平台配置、钩子函数和环境变量。本文档做全面梳理。

---

## 一、顶层通用配置

这些字段直接写在配置文件的根级别，对所有平台生效。

### 1.1 应用标识类

| 字段 | 类型 | 默认值 | 含义 | 当前值 |
|------|------|--------|------|--------|
| `appId` | string | `com.electron.${name}` | 应用唯一标识符 | `com.baijiahulian.tqclient.` |
| `productName` | string | package.json name | 产品显示名称 | `高途微信助手` |
| `copyright` | string | `Copyright © year ${author}` | 版权信息 | `Copyright © 2021 PZTD` |
| `buildVersion` | string | 自动 | 构建版本号（Windows: FileVersion） | 未配置 |

**appId 的重要性**：

```
appId 影响以下系统行为：
├── Windows 注册表键名
├── Windows 通知的应用标识
├── macOS 的 CFBundleIdentifier
├── 用户数据目录名（%APPDATA%/{appId}）
├── electron-updater 识别已安装应用
├── 协议注册（URL Scheme 关联）
└── 系统权限授予（macOS 隐私权限）

⚠ 一旦发布，appId 不能随意更改，否则：
   - 用户数据目录变化，旧数据丢失
   - 自动更新失效
   - 需要用户重新授予权限
```

### 1.2 文件打包类

| 字段 | 类型 | 默认值 | 含义 | 当前值 |
|------|------|--------|------|--------|
| `files` | array | 包含所有 | 打包文件包含/排除规则 | 已配置 |
| `extraFiles` | array | 无 | 复制到安装根目录的额外文件 | 未配置 |
| `extraResources` | array | 无 | 复制到 resources 目录的额外资源 | 在 win 下配置 |
| `extraMetadata` | object | 无 | 覆盖 package.json 中的字段 | 未配置 |

**extraMetadata 用法**：

```yaml
extraMetadata:
  main: dist/main.js
  channel: beta
  homepage: https://example.com
```

打包时会将这些字段合并到最终的 `package.json` 中，不修改源文件。

### 1.3 ASAR 配置

| 字段 | 类型 | 默认值 | 含义 | 当前值 |
|------|------|--------|------|--------|
| `asar` | boolean \| object | `true` | 是否使用 asar 打包 | `false` |
| `asarUnpack` | string \| string[] | 无 | 需要从 asar 中解压的文件 | 未配置 |

**asar 对象形式**：

```yaml
asar:
  smartUnpack: true          # 自动解压 native 模块
  ordering: "order.txt"      # 文件排序（优化加载速度）
```

### 1.4 产物命名

| 字段 | 类型 | 默认值 | 含义 | 当前值 |
|------|------|--------|------|--------|
| `artifactName` | string | `${productName}-${version}.${ext}` | 输出文件名模板 | `Weixinzhushou-${os}-${version}.${ext}` |

**可用的模板变量**：

| 变量 | 说明 | 示例 |
|------|------|------|
| `${productName}` | 产品名称 | `高途微信助手` |
| `${name}` | package.json name | `galaxy-client` |
| `${version}` | 版本号 | `5.5.0-release01` |
| `${channel}` | 更新通道 | `latest` |
| `${platform}` / `${os}` | 平台 | `win` |
| `${arch}` | 架构 | `x64` |
| `${ext}` | 扩展名 | `exe` |

### 1.5 构建行为

| 字段 | 类型 | 默认值 | 含义 | 当前值 |
|------|------|--------|------|--------|
| `compression` | string | `"normal"` | 压缩级别：`store`/`normal`/`maximum` | 未配置 |
| `npmRebuild` | boolean | `true` | 构建时是否自动重编 native 模块 | 未配置 |
| `nodeGypRebuild` | boolean | `false` | 是否使用 node-gyp 重编 | 未配置 |
| `npmArgs` | string[] | 无 | 传递给 npm 的额外参数 | 未配置 |
| `buildDependenciesFromSource` | boolean | `false` | 是否从源码编译依赖 | 未配置 |
| `electronVersion` | string | 自动检测 | Electron 版本 | 未配置 |
| `electronDownload` | object | 无 | Electron 下载配置（镜像等） | 未配置 |
| `extends` | string | 无 | 继承其他配置文件 | 未配置 |
| `removePackageScripts` | boolean | `true` | 是否移除 package.json 中的 scripts | 未配置 |
| `removePackageKeywords` | boolean | `true` | 是否移除 package.json 中的 keywords | 未配置 |

### 1.6 协议注册与文件关联

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `protocols` | array | 无 | URL Scheme 协议注册 |
| `fileAssociations` | array | 无 | 文件类型关联 |

```yaml
protocols:
  - name: "Galaxy Protocol"
    schemes:
      - galaxy
      - galaxy-dev
    role: Viewer

fileAssociations:
  - ext: galaxy
    name: "Galaxy Document"
    description: "Galaxy 文档"
    mimeType: application/x-galaxy
    icon: build/document.ico
```

### 1.7 拆包 / 多包

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `detectUpdateChannel` | boolean | `true` | 自动从版本号检测更新通道 |
| `generateUpdatesFilesForAllChannels` | boolean | `false` | 为所有通道生成更新清单 |
| `releaseInfo` | object | 无 | 发布说明信息 |

```yaml
releaseInfo:
  releaseName: "v5.5.0 正式版"
  releaseNotes: "1. 修复了群发消息的问题\n2. 优化了启动速度"
  releaseNotesFile: CHANGELOG.md
  releaseDate: "2026-03-16"
```

## 二、macOS 平台配置（mac）

虽然当前项目只构建 Windows，但了解 macOS 配置有助于未来扩展。

### 2.1 核心字段

```yaml
mac:
  target:
    - target: dmg
      arch: [x64, arm64]
    - target: zip
      arch: [universal]

  category: public.app-category.productivity
  icon: build/icon.icns
  darkModeSupport: true
  minimumSystemVersion: "10.13"
  
  hardenedRuntime: true
  gatekeeperAssess: false
  
  entitlements: build/entitlements.mac.plist
  entitlementsInherit: build/entitlements.mac.plist
  
  identity: null
  notarize: false
  
  type: distribution
  extendInfo:
    NSMicrophoneUsageDescription: "需要访问麦克风进行录音"
```

### 2.2 macOS 签名与公证

```
┌────────────────────────────────────────────────────────────┐
│              macOS 签名与公证流程                            │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  代码签名（Code Signing）                                  │
│  ├── 需要 Apple Developer 证书                             │
│  ├── identity: "Developer ID Application: ..."             │
│  ├── hardenedRuntime: true（macOS 10.14+ 必须）           │
│  └── entitlements.plist 声明所需权限                       │
│                                                            │
│  公证（Notarization）                                      │
│  ├── 将签名后的应用上传到 Apple 服务器验证                 │
│  ├── Apple 检查恶意代码、签名有效性                       │
│  ├── 通过后「装订」公证票据到应用中                       │
│  └── macOS 10.15+ 未公证的应用会被 Gatekeeper 阻止       │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 2.3 DMG 配置

```yaml
dmg:
  background: build/background.png
  icon: build/volume.icns
  iconSize: 80
  window:
    width: 540
    height: 380
  contents:
    - x: 130
      y: 220
      type: file
    - x: 410
      y: 220
      type: link
      path: /Applications
  format: UDZO
```

## 四、钩子函数

钩子函数允许在构建的不同阶段插入自定义逻辑。

### 4.1 可用钩子

| 钩子 | 触发时机 | 参数 | 典型用途 |
|------|---------|------|---------|
| `beforeBuild` | 构建开始前 | context | 检查环境、安装依赖 |
| `beforePack` | 打包前 | context | 修改文件、注入配置 |
| `afterPack` | 打包完成后（安装包生成前） | context | 注入额外文件、修改配置 |
| `afterSign` | 签名完成后 | context | macOS 公证 |
| `afterAllArtifactBuild` | 所有产物生成后 | buildResult | 上传、通知、清理 |
| `onNodeModuleFile` | 遍历 node_modules 中的每个文件 | file | 过滤不需要的文件 |
| `artifactBuildStarted` | 开始生成某个产物 | context | 日志、监控 |
| `artifactBuildCompleted` | 某个产物生成完成 | context | 逐个上传 |

### 4.2 配置方式

```yaml
beforeBuild: ./scripts/beforeBuild.js
afterPack: ./scripts/afterPack.js
afterSign: ./scripts/afterSign.js
afterAllArtifactBuild: ./scripts/afterBuild.js
```

### 4.3 钩子函数签名

```javascript
// afterPack.js
exports.default = async function(context) {
  const {
    appOutDir,          // 打包输出目录
    packager,           // Packager 实例
    electronPlatformName, // 'win32' | 'darwin' | 'linux'
    arch,               // 架构
    targets             // 目标格式列表
  } = context;

  console.log(`Platform: ${electronPlatformName}`);
  console.log(`Output: ${appOutDir}`);
  console.log(`Version: ${packager.appInfo.version}`);
};
```

```javascript
// afterAllArtifactBuild.js
exports.default = async function(buildResult) {
  const {
    outDir,          // 输出目录
    artifactPaths,   // 所有产物路径数组
    platformToTargets, // 平台到目标的映射
    configuration    // 构建配置
  } = buildResult;

  console.log('All artifacts:', artifactPaths);
  return artifactPaths; // 可以返回额外的产物路径
};
```

## 五、环境变量

electron-builder 支持通过环境变量控制构建行为，且环境变量优先级高于配置文件。

### 5.1 代码签名相关

| 环境变量 | 对应配置字段 | 说明 |
|----------|-------------|------|
| `CSC_LINK` | `win.certificateFile` / `mac.identity` | 证书文件路径或 base64 |
| `CSC_KEY_PASSWORD` | `win.certificatePassword` | 证书密码 |
| `CSC_NAME` | `win.certificateSubjectName` | 证书主体名称 |
| `CSC_FOR_PULL_REQUEST` | — | PR 构建时是否签名（默认 false） |
| `WIN_CSC_LINK` | `win.certificateFile` | Windows 专用证书路径 |
| `WIN_CSC_KEY_PASSWORD` | `win.certificatePassword` | Windows 专用证书密码 |
| `APPLE_ID` | — | Apple 账号（公证用） |
| `APPLE_ID_PASSWORD` | — | Apple 应用专用密码 |
| `APPLE_TEAM_ID` | — | Apple 团队 ID |

### 5.2 构建控制

| 环境变量 | 说明 |
|----------|------|
| `GH_TOKEN` | GitHub token（publish to GitHub） |
| `AWS_ACCESS_KEY_ID` | AWS 访问密钥（S3 publish） |
| `AWS_SECRET_ACCESS_KEY` | AWS 密钥 |
| `EP_PRE_RELEASE` | 设为 true 将版本标记为 prerelease |
| `EP_DRAFT` | 设为 true 将 GitHub release 标记为 draft |
| `ELECTRON_BUILDER_ALLOW_UNRESOLVED_DEPENDENCIES` | 允许未解析的依赖 |
| `DEBUG` | 设为 `electron-builder` 启用详细日志 |

### 5.3 缓存与下载

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `ELECTRON_CACHE` | `~/.cache/electron` | Electron 二进制缓存目录 |
| `ELECTRON_BUILDER_CACHE` | `~/.cache/electron-builder` | 构建工具缓存目录 |
| `ELECTRON_MIRROR` | `https://github.com/electron/electron/releases/download/` | Electron 下载镜像 |
| `ELECTRON_CUSTOM_DIR` | — | 自定义 Electron 下载目录 |
| `npm_config_electron_mirror` | — | npm 配置的 Electron 镜像 |

**国内镜像配置**：

```bash
# 淘宝镜像
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
```

## 六、CLI 命令行参数

```bash
electron-builder [options]
```

### 6.1 常用参数

| 参数 | 缩写 | 说明 |
|------|------|------|
| `--win` | `-w` | 构建 Windows |
| `--mac` | `-m` | 构建 macOS |
| `--linux` | `-l` | 构建 Linux |
| `--x64` | — | 构建 x64 架构 |
| `--ia32` | — | 构建 ia32 架构 |
| `--arm64` | — | 构建 ARM64 架构 |
| `--dir` | — | 仅输出目录（不生成安装包） |
| `--config` | `-c` | 指定配置文件路径 |
| `--publish` | `-p` | 发布选项：`always`/`never`/`onTag`/`onTagOrDraft` |
| `--prepackaged` | — | 使用预打包的目录 |

### 6.2 当前项目使用的命令

```bash
# 测试环境构建
electron-builder -w -c config/weixinzhushou/build.yml -p never

# 生产环境构建（命令相同，区别在 package.json 版本号）
electron-builder -w -c config/weixinzhushou/build.yml -p never
```

## 八、配置优先级

当多个位置都有配置时，优先级从高到低：

```
1. CLI 参数（--config.xxx=value）
2. 环境变量（CSC_LINK, GH_TOKEN 等）
3. 指定的配置文件（-c config/build.yml）
4. 默认配置文件（electron-builder.yml）
5. package.json 中的 build 字段
6. electron-builder 内置默认值
```
