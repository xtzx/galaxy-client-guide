# 构建与发布流程详解

> 项目的构建工具链、发布流程及存在的问题

## 一、构建工具概览

### 1.1 工具职责

| 工具 | 版本 | 职责 |
|-----|------|-----|
| **Webpack** | 4.46.0 | 代码打包、转译（ES6+→ES5）、压缩 |
| **Electron Builder** | 23.6.0 | 应用打包、生成安装程序、代码签名 |
| **Babel** | 7.x | JavaScript 转译 |

### 1.2 两个工具的关系

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         理想构建流程                                     │
└─────────────────────────────────────────────────────────────────────────┘

  源代码                    打包后的代码                   安装程序
    │                           │                           │
    │     Webpack               │    Electron Builder       │
    │   (代码层面处理)          │   (应用层面处理)          │
    │                           │                           │
    ▼                           ▼                           ▼
┌─────────┐    ────────►   ┌─────────┐    ────────►    ┌─────────┐
│ src/    │                │ public/ │                 │ .exe    │
│ 450+ 文件│                │ 1个文件 │                 │ 安装包   │
│ ES6+    │                │ ES5压缩 │                 │         │
└─────────┘                └─────────┘                 └─────────┘
```

**两者互补，不是替代关系。**

## 二、当前构建配置

### 2.1 package.json 脚本

```json
{
  "main": "src/electron.js",
  "scripts": {
    "start:dev": "cross-env ELECTRON_NODE_ENV=dev electron --inspect=5678 ./src/electron.js",
    "pre-build:test": "cross-env ELECTRON_NODE_ENV=test webpack --config webpack.electron.config.js",
    "app-build:test": "electron-builder -w -c config/weixinzhushou/build.yml -p never",
    "app-build:prod": "electron-builder -w -c config/weixinzhushou/build.yml -p never",
    "build": "node ./scripts/build.js"
  }
}
```

### 2.2 Webpack 配置

```javascript
// webpack.electron.config.js
module.exports = {
    mode: 'production',
    entry: 'src/electron.js',       // 入口
    output: {
        path: './public',
        filename: 'electron.js',    // 输出到 public/electron.js
    },
    module: {
        rules: [{
            test: /\.js$/,
            use: 'babel-loader',    // ES6+ 转译
            exclude: /node_modules/
        }]
    },
    target: 'electron-main',
};
```

### 2.3 Electron Builder 配置

```yaml
# config/weixinzhushou/build.yml
appId: com.baijiahulian.tqclient.
productName: 高途微信助手
asar: false                  # ⚠️ 不打包成 asar，源码暴露

files:
  - '**/*'                   # 打包所有文件
  - '!**/*.pfx'              # 排除证书
  - '!**/config'
  - '!**/scripts'

win:
  target: nsis
  certificateFile: newSignCert.pfx
  certificatePassword: gaotuketang
  sign: sign-script/sign.js  # 自定义签名脚本
```

## 三、构建流程分析

### 3.1 Test 环境构建流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Test 环境构建流程                                        │
└─────────────────────────────────────────────────────────────────────────────┘

1️⃣ npm run pre-build:test
   │
   │  执行: webpack --config webpack.electron.config.js
   │
   │  ┌───────────────────────────────────────────────────────────────────┐
   │  │  src/electron.js  →  babel-loader  →  public/electron.js         │
   │  │  (450+ 文件)          (转译)           (单文件，压缩)             │
   │  └───────────────────────────────────────────────────────────────────┘
   │
   ▼
2️⃣ npm run app-build:test
   │
   │  执行: electron-builder -w -c config/weixinzhushou/build.yml
   │
   │  ┌───────────────────────────────────────────────────────────────────┐
   │  │  读取 package.json                                                │
   │  │        ↓                                                          │
   │  │  main: "src/electron.js"  ← ⚠️ 指向源码，不是 public/            │
   │  │        ↓                                                          │
   │  │  打包 files: '**/*' (包括整个 src 目录)                          │
   │  │        ↓                                                          │
   │  │  asar: false (不压缩)                                             │
   │  │        ↓                                                          │
   │  │  输出: dist/weixinzhushou/Weixinzhushou-win-x.x.x.exe            │
   │  └───────────────────────────────────────────────────────────────────┘
   │
   ▼
❌ 问题：public/electron.js 没有被使用！
```

## 四、存在的问题

### 4.1 Webpack 打包未被使用

| 问题 | 现状 | 影响 |
|-----|------|-----|
| `main` 入口错误 | 指向 `src/electron.js` | Webpack 产物被忽略 |
| `pre-build:prod` 不存在 | 只有 test 版本 | 生产构建跳过 webpack |
| `build.js` 跳过 webpack | 直接执行 app-build | 代码未被优化 |

### 4.2 源码暴露风险

| 配置 | 当前值 | 风险 |
|-----|-------|-----|
| `asar` | `false` | 源码可直接查看 |
| `files` | `'**/*'` | 包含开发文件 |

### 4.3 签名配置混乱

```yaml
# build.yml 中配置了 pfx 文件
certificateFile: newSignCert.pfx
certificatePassword: gaotuketang

# 但 sign-script/sign.js 使用了 eToken
sign: sign-script/sign.js
# 内部使用 signtool + eToken
```

可能存在两套签名配置，需要确认哪个生效。

## 五、代码签名机制

### 5.1 签名流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           代码签名流程                                       │
└─────────────────────────────────────────────────────────────────────────────┘

  Electron Builder                    sign-script/sign.js
       │                                     │
       │  对每个 exe/dll 文件调用            │
       ├────────────────────────────────────►│
       │                                     │
       │                                     ▼
       │                           ┌─────────────────────┐
       │                           │  signtool.exe       │
       │                           │                     │
       │                           │  /f sign.cer       │ ← 证书文件
       │                           │  /csp eToken       │ ← 硬件加密狗
       │                           │  /tr timestamp     │ ← DigiCert时间戳
       │                           └─────────────────────┘
       │                                     │
       │◄────────────────────────────────────┤
       │  签名完成                            │
```

### 5.2 签名相关文件

| 文件 | 位置 | 说明 |
|-----|------|-----|
| `sign.cer` | 构建机器本地 | 证书文件（被 .gitignore 忽略） |
| `sign.js` | `sign-script/sign.js` | 自定义签名脚本 |
| `eToken` | 物理 USB 设备 | 加密狗，需插在构建机器上 |
| `signtool.exe` | 构建机器本地 | Windows SDK 签名工具 |

### 5.3 为什么需要签名

| 原因 | 说明 |
|-----|------|
| Windows SmartScreen | 未签名程序会被拦截 |
| 杀毒软件误报 | 未签名程序容易被标记 |
| 企业信任 | 证明软件来源 |
| 自动更新校验 | 确保更新包未被篡改 |
