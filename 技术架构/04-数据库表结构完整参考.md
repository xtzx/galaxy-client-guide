## 表分类

### 分类一：新旧体系对照

| 数据域 | 旧体系（企微） | 新体系（微信4.0） |
|--------|---------------|-------------------|
| 好友/客户信息 | `external_user` + `external_user_relation` | `wk_external_users` |
| 群聊信息 | `chatroom_info` | `wk_conversations` |
| 群成员信息 | `chatroom_member_info` + `room_members_user` + `room_members_external` | `conversation` + `conversation_member` |
| 企业信息 | `corp` | `wk_corps` |
| 好友关系（个微） | `friend` + `friend_relation` | 沿用 `friend` + `friend_relation` |

### 分类二：按业务域

```
好友域
├── friend                    # 好友基本信息（个微，全局共享，所有机器人共用）
├── friend_relation           # 机器人↔好友关系（个微，按 owner_wxid 区分）
├── wk_external_users         # 好友/客户信息（微信4.0，按 account_id 区分）
├── external_user             # 外部联系人基本信息（企微旧，已逐步废弃）
└── external_user_relation    # 外部联系人↔员工关系（企微旧，已逐步废弃）

群聊域
├── chatroom_info             # 群聊基本信息（个微/旧版）
├── chatroom_member_info      # 群成员信息（个微/旧版）
├── wk_conversations          # 群聊信息（微信4.0）
├── conversation              # 会话信息
├── conversation_member       # 会话成员
├── room_members_user         # 群内部成员（企微旧）
└── room_members_external     # 群外部成员（企微旧）

企业域
├── corp                      # 企业信息（企微旧）
└── wk_corps                  # 企业信息（微信4.0）

任务域
└── task_info                 # 任务执行记录
```

## 各表详细字段定义

### 1. `friend` — 好友基本信息

> 实体：`src/sqlite/entities/friend.js`
> Model：`src/msg-center/business/dao-model/friendModel.js`
> Service：`src/msg-center/business/dao-service/friendService.js`

存储好友的基本信息（昵称、头像等），**全局共享**，不区分机器人。通过 `friend_relation` 表关联到具体机器人。

| 字段名 | 数据库列名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|-----------|------|--------|--------|------|
| id | id | INTEGER | ✗ | 自增 | 主键 |
| username | username | STRING(128) | ✗ | — | 好友微信ID，**唯一** |
| alias | alias | STRING(255) | ✓ | — | 微信号（别名） |
| city | city | STRING(255) | ✓ | — | 城市 |
| conRemark | con_remark | STRING(255) | ✓ | — | 好友备注 |
| ext | ext | STRING(255) | ✓ | — | 扩展信息 |
| headUrl | head_url | STRING(255) | ✓ | — | 头像URL |
| nickname | nickname | STRING(128) | ✓ | — | 昵称 |
| province | province | STRING(255) | ✓ | — | 省份 |
| sex | sex | TINYINT | ✓ | — | 性别（0-未知，1-男，2-女） |
| createTime | create_time | DATE | ✗ | NOW | 创建时间 |
| updateTime | update_time | DATE | ✓ | NOW | 更新时间 |

**索引**：`friend_wxid_index` — 唯一索引（username）

**关键操作**：
- 写入：好友列表同步时 `friendService.js`
- 删除：当所有机器人都不再与该好友有关系时，由 `friendDeleteBoost.js` 清理

### 2. `friend_relation` — 机器人与好友关系

> 实体：`src/sqlite/entities/friend_relation.js`
> Model：`src/msg-center/business/dao-model/friendRelationModel.js`
> Service：`src/msg-center/business/dao-service/friendRelationService.js`

存储**每个机器人**与好友的一对一关系。一个好友可与多个机器人关联。

| 字段名 | 数据库列名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|-----------|------|--------|--------|------|
| id | id | INTEGER | ✗ | 自增 | 主键 |
| friendRemark | friend_remark | STRING(128) | ✓ | — | 机器人对好友的备注 |
| friendWxid | friend_wxid | STRING(128) | ✗ | — | 好友微信ID |
| ownerWxid | owner_wxid | STRING(128) | ✗ | — | 机器人微信ID |
| reportFlag | report_flag | TINYINT | ✗ | — | 上报标志（0-未上报，1-已上报） |
| friendRecordTime | friend_record_time | INTEGER | ✗ | — | 成为好友的时间戳 |
| createTime | create_time | DATE | ✗ | NOW | 创建时间 |
| updateTime | update_time | DATE | ✓ | NOW | 更新时间 |

**索引**：
- 复合唯一索引（owner_wxid, friend_wxid）
- 普通复合索引（owner_wxid, friend_wxid）

**关键操作**：
- 写入：`friendRelationService.saveFriendRelation()`
- 删除：`friendRelationService.deleteFriendRelation(ownerWxId, wxid)`
- 被谁使用：`handleDelContactService.js`、`activeDeleteFriendService.js`（已废弃）

### 3. `wk_external_users` — 外部用户/好友信息（微信4.0 新表）

> 实体：`src/sqlite/entities/wk_external_users.js`
> Model：`src/msg-center/business/dao-model/workwx/wkExternalUsersModel.js`
> Service：`src/msg-center/business/dao-service/workwx/wkExternalUserService.js`

微信4.0 体系下的好友/客户信息表，**按 account_id（机器人ID）区分**。**替代了旧版的 `external_user` + `external_user_relation` 双表结构。**

| 字段名 | 数据库列名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|-----------|------|--------|--------|------|
| id | id | INTEGER | ✗ | 自增 | 主键 |
| accountId | account_id | STRING(128) | ✗ | — | 机器人账号ID |
| wxid | wxid | STRING(128) | ✗ | — | 客户/好友微信ID |
| sex | sex | TINYINT | ✓ | — | 性别（0-未知，1-男，2-女） |
| nickname | nickname | STRING(255) | ✓ | — | 昵称 |
| remark | remark | STRING(255) | ✓ | — | 备注名 |
| headimg | headimg | STRING(255) | ✓ | — | 头像URL |
| bigheadimg | bigheadimg | STRING(255) | ✓ | — | 大头像URL |
| corpId | corp_id | STRING(128) | ✓ | — | 所属企业ID |
| city | city | STRING(128) | ✓ | — | 城市 |
| country | country | STRING(128) | ✓ | — | 国家 |
| province | province | STRING(128) | ✓ | — | 省份 |
| addCustomerTime | add_customer_time | INTEGER | ✓ | — | 添加客户时间戳 |
| reportFlag | report_flag | TINYINT | ✓ | 0 | 上报标志（0-未上报，1-已上报） |

**索引**：`wxid` 唯一索引

**⚠️ 关键：此表是微信4.0判断"是否已是好友"的核心依据**

**关键操作**：
- 写入：`friendUpdateService.js`（添加好友时 addOrUpdate）、`workWxFriendMultData.js`（全量同步）
- 删除：`workWxFriendDeleteService.js`（DeleteCustomers 消息）
- 查询是否好友：`mqttAddChatroomFriendWx4Service.js`（群内加好友前置检查）、`noFriendBoost.js`（非好友检测）、`mqttWorkDeleteFriendService.js`（删除前查corpId）

**⚠️ 已知Bug**：`handleDelContactService.js` 删除好友时未清理此表（见下方"数据一致性问题"章节）

## 数据一致性问题

### 已知Bug：删除好友时 `wk_external_users` 表未清理

**问题描述**：

微信4.0 体系下，好友数据同时写入了 `friend_relation` 表和 `wk_external_users` 表。但 `handleDelContactService.js`（处理新版 HandleDelContact 协议）在删除好友时，只清理了 `friend_relation` 和 `friend` 表，遗漏了 `wk_external_users` 表。

**影响范围**：

| 受影响场景 | 文件 | 影响 |
|-----------|------|------|
| 群内加好友 | `mqttAddChatroomFriendWx4Service.js` | 误判"已是好友"，不执行加好友 |
| 非好友检测 | `noFriendBoost.js` | 可能误判为好友 |
| 删除好友查corpId | `mqttWorkDeleteFriendService.js` | 影响较小 |

**对比正确实现（`workWxFriendDeleteService.js`处理 DeleteCustomers）**：

```
workWxFriendDeleteService（正确） ✅
  ├── 通知前端
  ├── wkExternalUserService.saveExternalUsersOp('delete', ...) ← 清理了
  └── 上报云端

handleDelContactService（有Bug） ❌
  ├── 通知前端
  ├── FriendRelationService.deleteFriendRelation()
  ├── FriendDeleteBoost.deleteNoFriendRobot()
  ├── 上报云端
  └── （缺失）wkExternalUserService 删除 ← 遗漏！
```

### 完整的删除好友代码路径一览

| # | 文件 | 触发消息 | 清理 friend_relation | 清理 friend | 清理 wk_external_users | 状态 |
|---|------|---------|---------------------|-------------|----------------------|------|
| 1 | `handleDelContactService.js` | HandleDelContact | ✅ | ✅（条件） | ❌ **遗漏** | 微信4.0主力路径 |
| 2 | `activeDeleteFriendService.js` | 旧协议 | ✅ | ✅（条件） | ❌ | 已废弃（filter=false） |
| 3 | `workWxFriendDeleteService.js` | DeleteCustomers | ❌ | ❌ | ✅ | 企微旧协议 |
| 4 | `deleteFriendTaskResponse.js` | 任务回执 | — | — | — | 仅上报，不清理DB |
| 5 | `workWxDeleteFriendTaskResponse.js` | 任务回执 | — | — | — | 仅上报，不清理DB |
| 6 | `batchDeleteFriendResponse.js` | 批量回执 | — | — | — | 仅上报，不清理DB |
