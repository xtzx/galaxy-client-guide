## 三、通信1: IPC通信（核心）

### 3.1 概述

IPC（Inter-Process Communication）是项目**最核心**的通信方式，用于 Electron 主进程与逆向服务（BasicService.exe）之间的双向通信。

### 3.4 DLL函数接口

```javascript
// src/msg-center/core/reverse/dll/clibrary.js

const ffi = require('ffi-napi');

const functions = {
    // 连接IPC服务器，返回管道句柄(pipeCode)
    IpcConnectServer: [int, [int]],           // (processId) => pipeCode
    
    // 检查管道是否有消息 (>0有消息, 0无消息, <0错误)
    IpcSelectCltChannel: [int, [int]],        // (pipeCode) => selectCode
    
    // 发送消息到管道
    IpcClientSendMessage: [bool, [charPtr, int, int]],  // (buffer, length, pipeCode)
    
    // 从管道接收消息
    IpcClientRecvMessage: [string, [int, int, int]],    // (pipeCode, selectCode, 0)
    
    // 关闭管道连接
    IpcClientClose: [bool, [int]],            // (pipeCode)
    
    // 检查进程是否可用于连接
    isUseProcess: [bool, [int]],              // (processId)
};
```

### 3.5 连接建立流程

```javascript
// 简化的连接流程
async function establishConnection() {
    while (true) {
        // 1. 获取所有微信/企微进程ID
        let processIds = IpcUtil.getProcessIds();  // tasklist | findstr
        
        // 2. 过滤不可用的进程
        processIds = processIds.filter(pid => Clibrary.isUseProcess(pid));
        
        // 3. 为新进程建立连接
        for (let processId of processIds) {
            let pipeCode = Clibrary.IpcConnectServer(processId);
            
            if (pipeCode > 0) {
                // 保存连接信息
                RegistryConfig.add({
                    id: processId,
                    pipeLineWrapper: { pipeCode, processId },
                });
                
                // 启动消息轮询
                AsyncSelectTask.run(registry.pipeLineWrapper);
            }
        }
        
        await sleep(5000);  // 每5秒扫描一次
    }
}
```

## 四、通信2: Electron IPC

### 4.1 概述

Electron IPC 用于主进程与渲染进程之间的通信，主要处理系统功能调用、配置获取、文件操作等。

### 4.2 通信模式

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Electron IPC 通信模式                                   │
└─────────────────────────────────────────────────────────────────────────────┘

【模式1: 同步请求】
    渲染进程                                          主进程
        │  ipcRenderer.sendSync('get-is-gray')        │
        ├────────────────────────────────────────────►│
        │◄────────────────────────────────────────────┤
        │  event.returnValue = true                   │
    
    特点：阻塞渲染进程，等待返回
    场景：快速配置获取

【模式2: 异步单向】
    渲染进程                                          主进程
        │  ipcRenderer.send('runInject')              │
        ├────────────────────────────────────────────►│
        │                                             │ 执行操作
        
    特点：发送后不等待返回
    场景：触发操作（启动服务、打开文件）

【模式3: Promise异步】
    渲染进程                                          主进程
        │  ipcRenderer.invoke('convert-base64-to-file')│
        ├────────────────────────────────────────────►│
        │                                             │ 异步处理
        │◄────────────────────────────────────────────┤
        │  Promise resolve                            │
    
    特点：返回Promise
    场景：耗时操作

【模式4: 主进程推送】
    主进程                                          渲染进程
        │  webContents.send('get-all-config', data)   │
        ├────────────────────────────────────────────►│
        │                                             │ 接收处理
    
    特点：主进程主动推送
    场景：状态更新、通知
```

### 4.3 核心文件

```
src/event/ipc.js              # 主进程IPC事件注册
src/utils.js                  # sendToRenderMsg 推送函数
extraResources/load/inject.js # 预加载脚本（渲染进程）
```

### 4.4 主要业务场景

| 事件名 | 方向 | 用途 |
|-------|------|------|
| `runInject` | 渲染→主进程 | 拉起逆向服务 |
| `userLogin` / `userLogout` | 渲染→主进程 | 用户登录/登出 |
| `get-app-config` | 渲染→主进程 | 获取应用配置 |
| `set-user-info` | 渲染→主进程 | 设置用户信息 |
| `open-url` / `open-file` | 渲染→主进程 | 打开URL/文件 |
| `app-relaunch` / `app-exit` | 渲染→主进程 | 重启/退出应用 |
| `get-all-config` | 主进程→渲染 | 推送机器人配置更新 |
| `logout-notify` | 主进程→渲染 | 推送登出通知 |
| `cas-auto-complete-info` | 双向 | CAS登录自动填充 |

### 4.5 代码示例

```javascript
// src/event/ipc.js - 主进程注册事件
const ipc = require('electron-better-ipc/source/main');

module.exports = () => {
    // 同步事件
    ipc.on('get-is-gray', (event) => {
        event.returnValue = store.getIsGray();
    });
    
    // 异步单向事件
    ipc.on('runInject', () => {
        runInject();  // 启动逆向服务
    });
    
    // Promise事件
    ipc.answerRenderer('convert-base64-to-file', async (arg) => {
        return await convertBase64ToFile(arg);
    });
};

// src/utils.js - 主进程推送
exports.sendToRenderMsg = (msg, text) => {
    const { mainWindow } = app;
    mainWindow?.webContents?.send(msg, text);
};
```

### 4.6 @electron/remote 使用

项目使用 `@electron/remote` 让渲染进程直接访问主进程模块：

```javascript
// src/init/window.js - 主进程初始化
const remoteMain = require('@electron/remote/main');
remoteMain.initialize();
remoteMain.enable(mainWindow.webContents);

// extraResources/load/inject.js - 渲染进程使用
window.eleRemote = require('@electron/remote');
window.isDev = !eleRemote.app.isPackaged;  // 直接访问 app 模块
```

**用途**：
- 判断打包状态：`eleRemote.app.isPackaged`
- 获取应用路径：`eleRemote.app.getAppPath()`

## 六、通信4: MQTT通信

### 6.1 概述

MQTT 用于主进程与云端（瓦力后台）的实时任务通信，采用发布/订阅模式。

### 6.2 架构

```
    │  │   瓦力云后台     │───── 发布任务 ────►│   MQTT Broker    │            │
    │  │  (任务管理)      │                    │ tcp:1883         │            │
    │  └─────────────────┘                    └────────┬─────────┘            │
    │                                                  │                      │
    └──────────────────────────────────────────────────┼──────────────────────┘
                                                       │
                                                       │ 订阅/发布
                                                       ▼
    ┌─────────────────────────────────────────────────────────────────────────┐
    │                          Galaxy-Client                                  │
    │                                                                         │
    │  ┌─────────────────────────────────────────────────────────────────┐   │
    │  │                     mqttClientBase.js                           │   │
```

### 6.4 消息处理流程

```
MQTT Broker
     │
     │ 消息推送
     ▼

     │
     ├─► 1. 过期检查（默认3小时）──► 超期丢弃
     │
     ├─► 2. 去重检查 ──────────────► 重复丢弃
     │
     ├─► 3. 解析任务类型
     │
```

## 九、技术要点总结

| 通信 | 库/技术 | 关键特性 |
|------|--------|---------|
| IPC | ffi-napi + Windows Named Pipe | 原生DLL调用，高性能 |
| Electron IPC | electron-better-ipc | 支持Promise，更好的错误处理 |
| WebSocket | ws 库 | 轻量级，自动端口探测 |
| MQTT | mqtt.js | QoS=1确保至少一次投递 |
