## 一、数据持久化概述

### 1.1 存储方式汇总

| 存储方式 | 技术实现 | 存储位置 | 用途 |
|---------|---------|---------|------|
| **electron-store** | JSON文件 | `%APPDATA%\galaxy-client\config.json` | 应用配置 |
| **SQLite** | sequelize | `%APPDATA%\galaxy-client\xxx.db` | 业务数据 |

## 二、electron-store 本地配置

### 2.1 核心文件

```
src/common/store.js    # electron-store 封装
```

### 2.2 存储内容

```javascript
// src/common/store.js

const Store = require('electron-store');
const store = new Store();

module.exports = {
    // ═══════════════════════════════════════════════════════════════════
    // GID - 设备唯一标识符
    // ═══════════════════════════════════════════════════════════════════
    getGid() { return store.get('gid'); },
    setGid(gid) { store.set('gid', gid); },
    
    // ═══════════════════════════════════════════════════════════════════
    // 用户信息
    // ═══════════════════════════════════════════════════════════════════
    getUserId() { return store.get('userId'); },
    setUserId(userId) { store.set('userId', userId); },
    clearUserId() { store.set('userId', ''); },
    
    getUserInfo() { return store.get('userInfo'); },
    setUserInfo(info) { store.set('userInfo', info); },
    clearUserInfo() { store.set('userInfo', {}); },
    
    // ═══════════════════════════════════════════════════════════════════
    // 窗口状态
    // ═══════════════════════════════════════════════════════════════════
    getWindowState() { return store.get('windowState'); },
    setWindowState(windowState) { store.set('windowState', windowState); },
    clearWindowState() { store.set('windowState', {}); },
    
    // ═══════════════════════════════════════════════════════════════════
    // 环境设置
    // ═══════════════════════════════════════════════════════════════════
    getEnvSettings() { return store.get('envSettings'); },
    setEnvSettings(settings) { store.set('envSettings', settings); },
    
    // ═══════════════════════════════════════════════════════════════════
    // 灰度标识
    // ═══════════════════════════════════════════════════════════════════
    setIsGray(bool) {
        if (typeof bool === 'boolean') {
            store.set('isGray', bool);
        }
    },
    getIsGray() {
        const isGray = store.get('isGray');
        return typeof isGray === 'boolean' ? isGray : false;
    },
    
    // ═══════════════════════════════════════════════════════════════════
    // 自动填充信息（CAS登录）
    // ═══════════════════════════════════════════════════════════════════
    setAutoCompleteInfo(info) { store.set('autoCompleteInfo', info); },
    getAutoCompleteInfo() { return store.get('autoCompleteInfo'); },
    clearAutoCompleteInfo() { store.set('autoCompleteInfo', {}); },
    
    // ═══════════════════════════════════════════════════════════════════
    // 注册列表
    // ═══════════════════════════════════════════════════════════════════
    setRegistryList(registryList) { store.set('registryList', registryList); },
    getRegistryList() { return store.get('registryList'); },
    
    // ═══════════════════════════════════════════════════════════════════
    // 活动信息
    // ═══════════════════════════════════════════════════════════════════
    getActivityInfo() { return store.get('activityInfo'); },
    setActivityInfo(activityInfo) { store.set('activityInfo', activityInfo); },
};
```

### 2.3 存储必要性分析

| 存储项 | 必要性 | 原因 |
|--------|--------|------|
| **gid** | ⭐⭐⭐ 必须 | 设备唯一标识，云端用于区分客户端，不能每次重新生成 |
| **userId** | ⭐⭐⭐ 必须 | 登录状态保持，避免每次启动都要重新登录 |
| **userInfo** | ⭐⭐ 重要 | 用户信息缓存，减少重复请求 |
| **isGray** | ⭐⭐ 重要 | 灰度发布控制，决定功能版本 |
| **windowState** | ⭐ 可选 | 用户体验优化，记住窗口位置 |
| **autoCompleteInfo** | ⭐ 可选 | 自动填充登录信息，便捷性 |
| **registryList** | ❓ 待评估 | 理论上可从内存重建，但有助于快速恢复状态 |

### 2.4 存储位置

```
Windows: %APPDATA%\galaxy-client\config.json
         C:\Users\<用户名>\AppData\Roaming\galaxy-client\config.json

macOS:   ~/Library/Application Support/galaxy-client/config.json
```

## 三、注册表操作

### 3.1 核心文件

```
src/event/regedit.js    # 注册表操作封装
```

### 3.2 具体操作

```javascript
// src/event/regedit.js

const regedit = require('regedit');
const log = require('electron-log');

// ═══════════════════════════════════════════════════════════════════
// 操作1: 修改 WXWork.exe 的 Image File Execution Options
// ═══════════════════════════════════════════════════════════════════
const modifyImageFileRegistry = () => {
    const key = 'HKLM\\SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion\\Image File Execution Options\\WXWork.exe';
    const valueName = 'GlobalFlag';
    const newValue = 0x00000200; // 十六进制数值
    
    regedit.createKey(key, (error) => {
        if (error) {
            log.info('Error create regedit image', error);
            return;
        }
        regedit.putValue({ 
            [key]: { 
                [valueName]: { value: newValue, type: 'REG_DWORD' } 
            } 
        }, (err) => {
            if (err) {
                log.info('Error modifying registry image:', err);
                return;
            }
            log.info('Registry image modified successfully.');
        });
    })
};

// ═══════════════════════════════════════════════════════════════════
// 操作2: 设置 UAC 策略
// ═══════════════════════════════════════════════════════════════════
const modifydrawFileRegistry = () => {
    const key = 'HKLM\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Policies\\System';
    const valueName = 'EnableLUA';
    const newValue = 0x00000001; // 确保 UAC 启用
    
    regedit.putValue({ 
        [key]: { 
            [valueName]: { value: newValue, type: 'REG_DWORD' } 
        } 
    }, (err) => {
        if (err) {
            log.info('Error modifying registry draw:', err);
            return;
        }
        log.info('Registry draw modified successfully.');
    });
};

// 初始化入口
const initRegedit = () => {
    modifyImageFileRegistry();
    modifydrawFileRegistry();
}

module.exports = { initRegedit };
```

### 3.3 注册表操作详解

| 注册表键 | 值名 | 值 | 用途 |
|---------|------|-----|------|
| `HKLM\...\Image File Execution Options\WXWork.exe` | `GlobalFlag` | `0x200` | 启用堆栈跟踪调试标志，用于进程监控 |
| `HKLM\...\Policies\System` | `EnableLUA` | `1` | 确保UAC（用户账户控制）启用 |

### 3.4 合理性评估

**Image File Execution Options 的作用：**
- 这是 Windows 系统级的调试机制
- `GlobalFlag=0x200` 启用 **FLG_HEAP_ENABLE_TAIL_CHECK**
- 用于内存调试和进程监控

**安全性考量：**

| 评估维度 | 评价 |
|---------|------|
| **技术必要性** | ✅ 对于DLL注入和进程通信，需要特定的进程标志 |
| **安全风险** | ⚠️ 修改 HKLM 需要管理员权限 |
| **用户感知** | ❌ 用户可能不知道软件修改了系统注册表 |
| **杀软反应** | ⚠️ 可能被杀毒软件标记为可疑行为 |
| **企业内部使用** | ✅ 作为内部工具，可接受 |

**建议改进：**
1. 在安装时明确告知用户
2. 卸载时清理注册表
3. 记录操作日志

## 四、asar 打包策略

### 4.1 当前配置

```yaml
# config/weixinzhushou/build.yml

asar: false    # ❌ 不使用 asar 打包
```

### 4.2 asar 打包对比

| 特性 | 打包成 asar | 不打包（当前） |
|------|------------|---------------|
| **文件结构** | 单个 .asar 文件 | 保持原始目录结构 |
| **代码保护** | 有一定保护 | 代码完全暴露 |
| **启动速度** | 稍快 | 稍慢 |
| **包体积** | 稍小 | 稍大 |
| **调试难度** | 较难 | 容易 |
| **原生模块** | 需要 asarUnpack | 无兼容问题 |

### 4.3 不使用 asar 的原因

**核心原因：ffi-napi 调用 DLL 的兼容性问题**

```yaml
# 如果用 asar，需要排除这些原生模块：
asarUnpack:
  - "**/node_modules/ffi-napi/**"     # DLL调用
  - "**/node_modules/ref-napi/**"     # 内存引用
  - "**/node_modules/sharp/**"        # 图片处理
  - "**/node_modules/sqlite3/**"      # 数据库
  - "**/bin/**"                       # 二进制文件
```

项目使用 `ffi-napi` 调用 Windows DLL（PipeCore.dll、ReUtils64.dll），这些原生模块在 asar 包内无法正常工作。

**权衡取舍：**
- ✅ 优点：更好的兼容性、调试便利
- ❌ 缺点：代码保护弱、包体积大

对于企业内部工具，这是合理的选择。

### 4.4 如果要启用 asar

```yaml
# 启用 asar 的配置方式
asar: true
asarUnpack:
  - "**/node_modules/ffi-napi/**"
  - "**/node_modules/ref-napi/**"
  - "**/node_modules/sharp/**"
  - "**/node_modules/sqlite3/**"
  - "**/node_modules/regedit/**"
  - "**/bin/**"
  - "**/extraResources/**"
```

## 五、内存缓存策略

### 5.1 主要缓存

| 缓存名 | 文件位置 | 用途 | 生命周期 |
|-------|---------|------|---------|
| `RegistryConfig` | `core/registry-config/` | 微信连接注册信息 | 应用运行期间 |
| `pipeCodeCache` | `core/cache/common/` | IPC管道句柄缓存 | 应用运行期间 |
| `mqttTaskMapLock` | `core/mq/mqttClientBase.js` | MQTT任务去重锁 | 100分钟 |
| `GalaxyTaskCache` | `core/cache/` | 任务状态缓存 | 任务完成后清理 |
| `userInfoCache` | `core/cache/` | 用户信息缓存 | 应用运行期间 |

### 5.3 去重锁清理

```javascript
// MQTT任务去重锁
let mqttTaskMapLock = new Map();

// 每10分钟清理一次
setInterval(() => {
    const now = Date.now();
    const expireTime = 6000 * 1000;  // 100分钟过期
    
    for (const [taskId, lockInfo] of mqttTaskMapLock.entries()) {
        if (now - lockInfo.timestamp > expireTime) {
            mqttTaskMapLock.delete(taskId);
        }
    }
}, 600 * 1000);
```

## 七、数据存储最佳实践

### 7.1 分层存储原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                        数据存储分层                                   │
└─────────────────────────────────────────────────────────────────────┘

    实时性要求高、频繁访问
           │
           ▼
    ┌──────────────────┐
    │    内存缓存       │  ← RegistryConfig, pipeCodeCache
    │  （最快访问）      │
    └────────┬─────────┘
             │
    需要持久化、重启恢复
             │
             ▼
    ┌──────────────────┐
    │   本地文件存储     │  ← electron-store (JSON)
    │  （配置类数据）    │     SQLite (业务数据)
    └────────┬─────────┘
             │
    需要多端同步、数据一致
             │
             ▼
    ┌──────────────────┐
    │    云端存储       │  ← API 上报
    │  （最终一致性）    │
    └──────────────────┘
```

### 7.2 存储选择建议

| 数据类型 | 推荐存储 | 原因 |
|---------|---------|------|
| 设备标识(gid) | electron-store | 需要跨会话持久化 |
| 用户登录状态 | electron-store | 避免重复登录 |
| 窗口状态 | electron-store | 用户体验 |
| 微信连接状态 | 内存缓存 | 运行时数据，重启重建 |
| 好友列表 | SQLite + 内存 | 数据量大，需要查询 |
| 任务状态 | 内存缓存 | 临时数据，完成即清理 |
