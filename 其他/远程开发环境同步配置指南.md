## IDE 设置同步

### 方案 A: 使用 Settings Sync（推荐）

Cursor/VS Code 内置了 Settings Sync 功能，可以通过 GitHub 或 Microsoft 账号同步设置。

**在本机 Mac 上操作：**

1. 打开 Cursor/VS Code
2. 按 `Cmd + Shift + P`，输入 `Settings Sync: Turn On`
3. 选择要同步的内容：
   - ✅ Settings（设置）
   - ✅ Keyboard Shortcuts（快捷键）
   - ✅ Extensions（扩展）
   - ✅ User Snippets（代码片段）
   - ✅ UI State（UI 状态）
4. 使用 GitHub 或 Microsoft 账号登录
5. 等待同步完成

**在远程 Windows 窗口操作：**

1. 通过 SSH 连接到 Windows
2. 按 `Ctrl + Shift + P`，输入 `Settings Sync: Turn On`
3. 使用**相同账号**登录
4. 选择 "Replace Local"（用云端覆盖本地）
5. 设置会自动同步

### 方案 B: 手动复制设置文件

如果不想使用云同步，可以手动复制配置文件：

**Mac 配置文件位置：**
```bash
# settings.json
~/Library/Application Support/Cursor/User/settings.json

# keybindings.json
~/Library/Application Support/Cursor/User/keybindings.json

# snippets
~/Library/Application Support/Cursor/User/snippets/
```

**Windows 配置文件位置：**
```powershell
# settings.json
C:\Users\<用户名>\AppData\Roaming\Cursor\User\settings.json

# keybindings.json
C:\Users\<用户名>\AppData\Roaming\Cursor\User\keybindings.json

# snippets
C:\Users\<用户名>\AppData\Roaming\Cursor\User\snippets\
```

## 远程插件自动安装配置

### 配置默认远程扩展

在 **本机 Mac** 的 `settings.json` 中添加以下配置，指定连接远程时自动安装的扩展：

```json
{
  "remote.SSH.defaultExtensions": [
    // 语言支持
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "ms-vscode.vscode-typescript-next",

    // Git 增强
    "eamodio.gitlens",
    "mhutchie.git-graph",

    // 开发工具
    "formulahendry.auto-rename-tag",
    "christian-kohler.path-intellisense",
    "streetsidesoftware.code-spell-checker",

    // 主题与图标（这些实际在本地运行，但可以配置）
    // "pkief.material-icon-theme",

    // 其他常用扩展
    "usernamehw.errorlens",
    "gruntfuggly.todo-tree"
  ]
}
```

### 一键导出/导入扩展列表

**在本机 Mac 上导出已安装扩展：**

```bash
# 导出扩展列表
code --list-extensions > ~/extensions.txt

# 或者 Cursor
cursor --list-extensions > ~/extensions.txt
```

**查看导出的列表：**
```bash
cat ~/extensions.txt
```

**在远程 Windows 上安装扩展（通过 SSH 连接后的终端）：**

```powershell
# 方法 1: 从列表批量安装
Get-Content extensions.txt | ForEach-Object { code --install-extension $_ }

# 方法 2: 直接安装常用扩展
code --install-extension dbaeumer.vscode-eslint
code --install-extension esbenp.prettier-vscode
code --install-extension eamodio.gitlens
```

#### gcmp 命令详解（从 Mac zsh 同步）

这是从 Mac 本机 zsh 配置同步过来的快捷命令：

**Mac 原版（zsh）：**
```bash
gcmp() {
    git add . && git commit -m "feat: $1" && git push --no-verify
}
```

**Windows 版本（PowerShell）：**
```powershell
function gcmp {
    param([string]$message)
    if (-not $message) {
        Write-Host "Usage: gcmp <commit message>" -ForegroundColor Yellow
        return
    }
    git add .
    git commit -m "feat: $message"
    git push --no-verify
}
```

**使用示例：**
```powershell
gcmp '添加用户登录功能'
# 等同于执行：
# git add .
# git commit -m "feat: 添加用户登录功能"
# git push --no-verify
```

## 常见问题

### Q: 新终端窗口没有加载 Git 别名？

**A:** PowerShell 执行策略可能阻止了 Profile 加载。执行以下命令：

```powershell
# 查看当前策略
Get-ExecutionPolicy

# 如果是 Restricted，改为 RemoteSigned
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q: 某些扩展在远程不工作？

**A:** 确认扩展已安装到远程：

1. 在远程窗口打开扩展面板
2. 找到扩展，查看是否有 "Install in SSH" 按钮
3. 如果有，点击安装

### Q: Settings Sync 冲突怎么办？

**A:** 选择 "Replace Local" 或 "Merge"：
- **Replace Local**: 完全使用云端配置覆盖本地
- **Merge**: 合并配置，保留两边的设置
