## 二、系统功能模块

### 2.1 登录/登出

#### 当前流程

```
Electron 启动
    ↓
load.html 加载
    ↓
getUserInfo() → 校验 Cookie/Token
    ↓ (未登录)          ↓ (已登录)
CAS 登录页            menu.html
(inject.js 注入)       (主页面)
    ↓
登录成功 → Cookie 写入
    ↓
跳转 menu.html
```

**依赖的 Electron 能力：**
- `extraResources/load/inject.js`：CAS 登录页预注入脚本
- `ipc.send('userLogin')`：通知主进程用户已登录
- `ipc.send('userLogout')`：通知主进程登出
- `remote.session`：Cookie 读写

#### 影响分析

| 影响点 | 严重程度 | 说明 |
|--------|---------|------|
| CAS 登录注入脚本失效 | 高 | inject.js 无法在浏览器中运行 |
| Cookie 管理变化 | 中 | 从 Electron session 改为浏览器 Cookie |
| 登录态传递 | 高 | 需要新的认证机制 |

#### 处置方案

**改为标准 Web CAS 登录流程：**

```
浏览器访问应用 URL
    ↓
前端路由守卫检查 Token
    ↓ (无 Token)                ↓ (有 Token)
重定向到 CAS 登录页           展示主页面
(标准浏览器跳转)
    ↓
CAS 登录成功
    ↓
回调到应用 URL（带 ticket 参数）
    ↓
前端调用后端接口验证 ticket
    ↓
后端返回 Token（JWT 或 Session）
    ↓
前端存储 Token（Cookie / localStorage）
    ↓
展示主页面
```

**改造清单：**

| 序号 | 改造项 | 文件 | 说明 |
|------|--------|------|------|
| 1 | 新增 AuthGuard 路由守卫 | 新建 `common/auth.js` | 检查 Token，未登录重定向 |
| 2 | 新增 CAS 回调处理 | 新建 `entries/callback/` | 处理 CAS 回调 ticket |
| 3 | 废弃 load 入口 | `entries/load/` | load.html 不再需要 |
| 4 | 废弃 vpn 入口 | `entries/vpn/` | VPN 错误页不再需要 |
| 5 | 改造登出逻辑 | `menu/component/Header.js` | `ipc.send('userLogout')` → API 调用 + 清除 Token |
| 6 | 改造 Cookie 管理 | `common/cookie.ts` | `remote.session` → `document.cookie` |

### 2.2 自动更新

#### 当前流程

```
主进程检查更新（electron-updater）
    ↓
发现新版本 → IPC 通知渲染进程
    ↓
渲染进程展示更新弹窗
    ↓
用户确认 → 下载更新 → 重启安装
```

**依赖的 IPC 通道：**
- `updateStart`：更新开始
- `downloadProgress`：下载进度
- `update-downloaded`：下载完成
- `updateError`：更新错误
- `checkDownloadFile`：检查下载文件
- `wsKillJava` / `killJavaFail` / `ws-kill-java-only`：更新前杀进程

#### 处置方案

**完全废弃** - Web 应用无需客户端更新机制。

| 序号 | 改造项 | 文件 | 说明 |
|------|--------|------|------|
| 1 | 删除更新相关 IPC 监听 | `menu/App.js` | 删除 updateStart 等监听 |
| 2 | 删除 UpdateModal 组件 | `menu/component/UpdateModal/` | 整个组件废弃 |
| 3 | 删除 Version 组件的更新逻辑 | `menu/component/Version.js` | 保留版本显示，删除更新功能 |
| 4 | 删除 sub 的 Version 更新逻辑 | `sub/component/Version/index.js` | 同上 |

**可选优化**：Web 应用可通过以下方式实现「更新提示」：
- Service Worker 检测新版本 → 提示用户刷新页面
- 或在每次 API 调用时检查版本号 → 版本不一致时提示刷新


**依赖的 Electron 能力：**

| 依赖 | 文件 | 说明 |
|------|------|------|
| `ipc.send('runInject')` | `menu/store/thunks.js` | 启动个微注入 |
| `ipc.send('runQyWxInject')` | `menu/store/thunks.js` | 启动企微注入 |
| `ipc.invoke('hasLoginedAccount')` | `numberManage/wechatTab/index.js` | 检测企微登录 |

| `ipc.on('is-init-bs-connecting')` | `menu/App.js` | BS 初始化状态 |

#### 影响分析

| 影响点 | 严重程度 | 说明 |
|--------|---------|------|
| 启动注入功能废弃 | 高 | C++ 客户端自行管理，前端不再控制 |
| 企微登录检测功能废弃 | 高 | 依赖本地 DLL，浏览器无法实现 |
| 方舟控制功能废弃 | 中 | C++ 客户端自行管理 |
| 账号列表来源变化 | 高 | 从 WebSocket getAllConfig → 云端 API |

#### 处置方案

**号管理页面改造为「查看和监控」模式：**

| 当前功能 | 改造后 | 说明 |
|---------|--------|------|
| 启动注入按钮 | **废弃** | C++ 客户端在本地自动启动注入 |
| 企微登录检测 | **改为云端查询** | `GET /api/wx/accounts/{id}/status` |
| 方舟控制 | **废弃** | C++ 客户端自管理 |
| 账号列表 | **改为云端查询** | `GET /api/wx/accounts` |
| 添加账号 | **保留** | 可能需要配合 C++ 客户端 |
| BS 连接状态 | **改为云端查询** | C++ 客户端上报状态到云端 |

**号管理页面新增能力：**
- 展示 C++ 客户端连接状态（在线/离线）
- 展示各微信账号状态（已登录/未登录/异常）
- 远程重启指令（通过云端下发 MQTT 指令）

### 2.4 环境设置（EnvSettingsButton）

#### 当前功能

```
环境设置按钮
├── 获取环境配置 (ipc.invoke('get-env-settings'))
├── 设置环境配置 (ipc.send('set-env-settings'))
└── 切换环境 (dev/test/prod)
```

#### 处置方案

**改为 localStorage 存储：**

```javascript
// 当前
const settings = await ipc.invoke('get-env-settings');
ipc.send('set-env-settings', newSettings);

// 改造后
const settings = JSON.parse(localStorage.getItem('envSettings') || '{}');
localStorage.setItem('envSettings', JSON.stringify(newSettings));
```

## 三、业务功能模块

### 3.1 通用影响分析

所有 sub 业务模块共同受到以下影响：

#### A. 消息发送方式变化

```javascript
// 当前：通过 ipc.sendToHost 发送给 menu，menu 转发给 WebSocket
function sendMessage(body, cmdId = 'forward') {
  ipc.sendToHost('sub-message', { cmdId, channelId, body });
}

// 改造后方案一：通过 postMessage 发送给 menu，menu 调用 HTTP API
function sendMessage(body, cmdId = 'forward') {
  window.parent.postMessage({
    source: 'galaxy-sub',
    type: 'sub-message',
    channelId,
    payload: { cmdId, body }
  }, MENU_ORIGIN);
}

// 改造后方案二：sub 直接调用 HTTP API（推荐）
async function sendMessage(body, cmdId = 'forward') {
  const result = await api.post(`/api/wx/forward`, { channelId, ...body });
  return result;
}
```

**推荐方案二**：sub 页面直接调用 HTTP API，减少通信链路。这样 sub 不再需要经过 menu 中转，架构更简洁。

#### B. 文件上传方式变化

```javascript
// 当前：通过 WebSocket upload 命令
sendMessage({ type: 'file', path: '/local/path' }, 'upload');

// 改造后：浏览器直传 OSS
async function uploadFile(file) {
  // 1. 获取 STS Token
  const { accessKeyId, accessKeySecret, stsToken, bucket, region } = 
    await api.get('/api/oss/sts-token');
  
  // 2. 使用 ali-oss SDK 上传
  const client = new OSS({ accessKeyId, accessKeySecret, stsToken, bucket, region });
  const result = await client.put(`uploads/${Date.now()}-${file.name}`, file);
  
  return result.url;
}
```

#### C. 文件选择方式变化

```javascript
// 当前：通过 IPC 获取文件
ipc.sendToHost('get-file-path');  // → 获取剪贴板文件路径
ipc.sendToHost('get-file-entity', { path }); // → 获取文件信息

// 改造后：浏览器 File API
// 文件选择
const fileInput = document.createElement('input');
fileInput.type = 'file';
fileInput.accept = 'image/*,video/*,.doc,.docx,.pdf';
fileInput.onchange = (e) => {
  const file = e.target.files[0];
  // 直接使用 file 对象
};
fileInput.click();

// 剪贴板
const clipboardItems = await navigator.clipboard.read();
for (const item of clipboardItems) {
  if (item.types.includes('image/png')) {
    const blob = await item.getType('image/png');
    const file = new File([blob], 'clipboard.png', { type: 'image/png' });
    // 使用 file 对象
  }
}
```

#### D. 链接预览方式变化

```javascript
// 当前：通过 IPC 跨域请求
const html = await ipc.callMain('cross-origin-request', { url });

// 改造后：通过云端代理
const { title, description, image } = await api.post('/api/proxy/link-preview', { url });
```

```javascript
// src/common/upload.js（改造后）
import OSS from 'ali-oss';
import api from './ajax';

let ossClient = null;
let tokenExpireTime = 0;

async function getOSSClient() {
  if (ossClient && Date.now() < tokenExpireTime) {
    return ossClient;
  }
  
  const { accessKeyId, accessKeySecret, stsToken, bucket, region, expiration } = 
    await api.get('/api/oss/sts-token');
  
  ossClient = new OSS({
    accessKeyId,
    accessKeySecret,
    stsToken,
    bucket,
    region,
  });
  tokenExpireTime = new Date(expiration).getTime() - 60000; // 提前1分钟刷新
  
  return ossClient;
}

export async function uploadFile(file, options = {}) {
  const client = await getOSSClient();
  const ext = file.name.split('.').pop();
  const key = `uploads/${Date.now()}-${Math.random().toString(36).slice(2)}.${ext}`;
  
  const result = await client.put(key, file, {
    progress: options.onProgress,
  });
  
  return result.url;
}

export async function uploadBase64(base64Data, filename) {
  const byteString = atob(base64Data.split(',')[1]);
  const mimeType = base64Data.match(/data:(.*?);/)[1];
  const ab = new ArrayBuffer(byteString.length);
  const ia = new Uint8Array(ab);
  for (let i = 0; i < byteString.length; i++) {
    ia[i] = byteString.charCodeAt(i);
  }
  const file = new File([ab], filename, { type: mimeType });
  return uploadFile(file);
}
```

#### 改造方案

```javascript
// src/common/paste.js（改造后）
export async function getClipboardFiles() {
  try {
    const items = await navigator.clipboard.read();
    const files = [];
    
    for (const item of items) {
      for (const type of item.types) {
        if (type.startsWith('image/')) {
          const blob = await item.getType(type);
          const file = new File([blob], `clipboard.${type.split('/')[1]}`, { type });
          files.push(file);
        }
      }
    }
    
    return files;
  } catch (err) {
    console.warn('Clipboard API not available:', err);
    return [];
  }
}

export function convertBase64ToFile(base64, filename) {
  const arr = base64.split(',');
  const mime = arr[0].match(/:(.*?);/)[1];
  const bstr = atob(arr[1]);
  let n = bstr.length;
  const u8arr = new Uint8Array(n);
  while (n--) {
    u8arr[n] = bstr.charCodeAt(n);
  }
  return new File([u8arr], filename, { type: mime });
}

export async function convertUrlToFile(url) {
  const response = await fetch(url);
  const blob = await response.blob();
  const filename = url.split('/').pop() || 'download';
  return new File([blob], filename, { type: blob.type });
}
```
