# 18 — 运行时配置管理（runtime.yml 与 Apollo）

> **文档定位**：梳理 `galaxy-client` 的运行时配置分层体系，包括静态 YAML 配置、  
> 环境分层配置、Apollo 动态配置的加载、读取、覆盖机制。  
> 配置系统是所有模块的基础依赖，决定了 MQTT 接入点、API 地址、功能开关等运行时行为。

---

## 1. 配置分层设计

```
┌─────────────────────────────────────────────────────────────────┐
│                      配置层次（优先级从低到高）                      │
│                                                                 │
│  ┌───────────────────────────────────────────────────────┐      │
│  │ Layer 1: applicationBase.js                           │      │
│  │ 静态基础配置（公共参数，所有环境共享）                     │      │
│  └────────────────────┬──────────────────────────────────┘      │
│                       │ 被覆盖                                   │
│  ┌────────────────────▼──────────────────────────────────┐      │
│  │ Layer 2: applicationRd/Qa/Prod.js                     │      │
│  │ 环境分层配置（MQTT、API URL 等环境相关参数）               │      │
│  └───────────────────────────────────────────────────────┘      │
│                                                                 │
│  ┌───────────────────────────────────────────────────────┐      │
│  │ Layer 3: runtime.yml                                  │      │
│  │ 产品线配置（模块开关、版本号、产品特有参数）                │      │
│  └────────────────────┬──────────────────────────────────┘      │
│                       │ 合并                                     │
│  ┌────────────────────▼──────────────────────────────────┐      │
│  │ Layer 4: runtime.dev.yml（仅开发环境）                  │      │
│  │ 开发覆盖（关闭上报、开启调试日志）                        │      │
│  └───────────────────────────────────────────────────────┘      │
│                                                                 │
│  ┌───────────────────────────────────────────────────────┐      │
│  │ Layer 5: Apollo 动态配置                               │      │
│  │ 运行时远程下发（任务间隔、消息过期时间等）                  │      │
│  └───────────────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────────┘
```

### 1.1 两套配置系统

项目中存在**两套独立的配置系统**，服务不同目的：

| 配置系统 | 入口 | 服务对象 | 内容 |
|---------|------|---------|------|
| `appConfig`（runtime.yml） | `utils.getAppConfig()` | 窗口初始化、更新、inject | 产品线参数、功能模块列表、版本信息 |
| `ApplicationConfigEnv` | `application-config/index.js` | msg-center 业务逻辑 | MQTT、API URL、超时参数、队列参数 |

## 2. runtime.yml — 产品线配置

### 2.1 加载机制

**文件**：`galaxy-client/src/utils.js`

```javascript
exports.getAppConfig = () => {
    if (!appConfig) {
        const configRoot = `../config/${this.getProduct()}`;
        const configFilePath = path.resolve(__dirname, 
            isDev ? `${configRoot}/runtime.yml` : '../../app-config.yml'
        );
        appConfig = this.getYamlConfig(configFilePath);
        if (isDev) {
            const devConfigFilePath = path.resolve(__dirname, `${configRoot}/runtime.dev.yml`);
            const devConfig = this.getYamlConfig(devConfigFilePath);
            Object.assign(appConfig, devConfig);
        }
    }
    return appConfig;
};
```

加载规则：

| 环境 | 配置文件路径 | 说明 |
|------|------------|------|
| 开发 | `config/{product}/runtime.yml` + `runtime.dev.yml` | 合并加载 |
| 生产 | `app-config.yml`（extraResources 映射） | electron-builder 重命名后的 runtime.yml |

`getProduct()` 返回值由环境变量 `PRODUCT` 决定，默认 `weixinzhushou`。

const isDev = !app.isPackaged;
const version = require('../../package.json').version || app.getVersion();

const VERSION_TYPE = utils.getVersionType(version);

const urls = {
    loginUrlMap: {
        prod: 'https://cas.baijia.com/cas/login?service=...',

### 4.1 环境类型判定

```javascript
exports.getVersionType = version => {
    if (version.includes('release')) return 'prod';
    else if (version.includes('test')) return 'test';
    return 'vt';
};
```

| 版本号 | 类型 | 说明 |
|--------|------|------|
| `1.0.0-release.1` | `prod` | 生产环境 |
| `1.0.0-test.1` | `test` | 测试环境 |
| `1.0.0` | `vt` | VT 环境 |

### 4.2 urls.type 的传播

`urls.type` 被 `application-config/index.js` 用作环境判定的备选值：

```javascript
let env = process.env.ELECTRON_NODE_ENV || type;
```

如果没有设置 `ELECTRON_NODE_ENV`，则依据版本号推导环境。

## 5. Apollo 动态配置

### 5.1 配置获取

**文件**：`galaxy-client/src/msg-center/core/utils/getApolloConfig.js`

```javascript
const apolloConfig = {
    taskFailWaitTime: ApplicationEnv["fail.galaxy.task.status"],
    taskDefaultWaitTime: 200,
    mqttTaskExpireHours: 3,
    initStatus: false
};

const initApolloConfig = async () => {
    if (apolloConfig.initStatus) return;
    getTaskFailTime();       // client.wxzs.fail.task.time
    getTaskWaitTime();       // client.wxzs.wait.task.time
    getMqttTaskExpireHours(); // client.wxzs.mqtt.task.expire.hours
};

initApolloConfig();  // 模块加载时立即执行
module.exports = apolloConfig;
```

### 5.2 Apollo 配置项

| Apollo Key | 本地变量 | 默认值 | 说明 |
|------------|---------|--------|------|
| `client.wxzs.fail.task.time` | `taskFailWaitTime` | 3000ms | 微信任务失败等待时间 |
| `client.wxzs.wait.task.time` | `taskDefaultWaitTime` | 200ms | 群发消息间隔时间 |
| `client.wxzs.mqtt.task.expire.hours` | `mqttTaskExpireHours` | 3 小时 | MQTT 任务过期时间 |

### 5.3 获取流程

```javascript
const getTaskWaitTime = async () => {
    try {
        const res = await httpFetch({
            url: getApolloConfigUrl,
            data: { key: "client.wxzs.wait.task.time" },
            method: 'get'
        });
        if (res) {
            apolloConfig.taskDefaultWaitTime = +JSON.parse(res)?.data || 200;
        }
        apolloConfig.initStatus = true;
    } catch (e) {
        logUtil.customLog(`获取阿波罗配置异常`);
    }
};
```

特点：
- 模块加载时自动获取（`initApolloConfig()` 在模块底部调用）
- 获取失败时使用默认值，不影响启动
- `initStatus` 标记防止重复获取
- 三个配置项**并行获取**（不使用 await）

### 5.4 Apollo URL 配置

| 环境 | URL |
|------|-----|
| dev | `https://test-qun.umeng100.com/wechat/assist/common/apollo/get` |
| qa | `https://test-qun.umeng100.com/wechat/assist/common/apollo/get` |
| prod | `https://qun.umeng100.com/wechat/assist/common/apollo/get` |

### 5.5 Apollo 配置的使用位置

| 文件 | 使用的配置 | 说明 |
|------|-----------|------|
| `mqTask.js` | `apolloConfig.taskDefaultWaitTime` | 消息发送间隔 |
| `dispatchOutBound.js` | `apolloConfig.taskFailWaitTime` | 失败任务延迟 |
| `mqttClientBase.js` | `apolloConfig.mqttTaskExpireHours` | MQTT 消息过期判定 |

## 6. 前端配置读取

### 6.1 IPC 接口

前端通过 `ipc.callMain('get-app-config')` 获取运行时配置：

```javascript
// galaxy-client/src/event/ipc.js
ipc.answerRenderer('get-app-config', () => {
    return utils.getAppConfig();
});
```

返回完整的 `appConfig` 对象（runtime.yml 内容）。

### 6.2 前端使用方式

```javascript
// galaxy/src/common/project.js
const appConfig = await ipc.callMain('get-app-config');

// 读取具体配置
const modules = appConfig.modules;     // 功能模块列表
const product = appConfig.product;     // 产品品牌
const multiple = appConfig.multiple;   // 最大微信数
```

### 6.3 模块动态更新

前端还可以通过服务端 API 动态更新 `modules`：

```javascript
// galaxy/src/entries/menu/store/thunks.js
export function askAndSetModules() {
    getModuleByElfKey({
        elfKey: project.getConfig('elfkey'),
        userName: getGlobalData('casId'),
    }).then((res) => {
        if (res?.code === 200 && res.data) {
            ipc.send('set-modules', res.data.extendModule);
        }
    });
}
```

## 7. 配置加载时序

```
应用启动
    │
    ├─ utils.js 被 require
    │   └─ getAppConfig() 缓存 runtime.yml 内容
    │
    ├─ urls.js 被 require
    │   └─ 计算 type (prod/test/vt)
    │
    ├─ application-config/index.js 被 require
    │   └─ 根据 env 选择 Rd/Qa/Prod 配置
    │
    ├─ getApolloConfig.js 被 require
    │   └─ initApolloConfig() 异步获取 Apollo 配置
    │       ├─ getTaskFailTime()      → HTTP 请求
    │       ├─ getTaskWaitTime()      → HTTP 请求
    │       └─ getMqttTaskExpireHours() → HTTP 请求
    │
    ├─ electron.js 启动
    │   └─ addIpcEvent() → 注册 'get-app-config' handler
    │
    └─ 前端 menu.html 加载
        └─ ipc.callMain('get-app-config') → 获取配置
```

## 8. 配置变更的热更新

### 8.1 isGray 灰度标记

```javascript
// galaxy-client/src/event/store.js
store.onDidChange('isGray', (newValue) => {
    // 灰度标记变更 → 重新加载配置/URL
});
```

`isGray` 变更可以触发配置源切换，实现灰度发布场景下的配置热更新。

### 8.2 modules 动态更新

前端通过 `set-modules` IPC 事件更新模块列表，无需重启客户端。

### 8.3 Apollo 配置刷新

当前 Apollo 配置**仅在模块加载时获取一次**，不支持运行时刷新。如需刷新，需要重启客户端。

通过 `PRODUCT` 环境变量选择产品线，决定加载哪个目录下的 `runtime.yml`。
