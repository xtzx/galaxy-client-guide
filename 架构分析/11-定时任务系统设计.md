# 11 — 定时任务系统设计

> **文档定位**：梳理 `galaxy-client` 主进程中所有定时执行的任务，包括调度框架、每个定时器的职责、异常处理策略。  
> 定时任务是维持系统正常运转的关键基础设施，负责心跳保活、消息补偿、状态检查、系统监控等核心功能。

---

## 1. 定时任务框架选型

### 1.1 node-cron

项目使用 `node-cron@^3.0.2` 作为定时调度框架。

| 对比维度 | node-cron | setInterval | node-schedule |
|---------|-----------|-------------|---------------|
| cron 表达式 | ✅ 支持秒级 | ❌ | ✅ |
| 内存占用 | 轻量 | 最轻量 | 较重 |
| 异常隔离 | 每次回调独立 | 需手动 try-catch | 每次回调独立 |
| 精度 | 秒级 | 毫秒级 | 秒级 |
| 适用场景 | 周期任务调度 | 简单延迟/间隔 | 复杂 cron 调度 |

选择 `node-cron` 的原因：
- 支持标准 cron 表达式，语义清晰
- 轻量级，适合 Electron 桌面应用
- 回调独立执行，一个任务异常不影响下一次调度

### 1.2 与 setInterval 的共存

除了 `node-cron` 的主调度循环，代码中还有多处使用 `setInterval`/`setTimeout` 实现特定场景的定时逻辑：

| 机制 | 使用场景 |
|------|---------|
| `node-cron` | 全局统一调度（每 10 秒） |
| `setInterval` | 缓存清理（`msgHandleBase`）、MQTT 保活（`mqttHelper`）、堆快照（`v8Heap`） |
| `setTimeout` | 延迟重试（`reverseSend`）、延迟处理（`dispatchOutBound`）、sleep 实现 |

## 2. 调度入口：schedual.js

**文件**：`galaxy-client/src/msg-center/start/schedual.js`

```javascript
const cron = require('node-cron');
const HeartTimer = require('../business/timer/HeartBeatTimer');
const PingTimer = require('../business/timer/PingTimer');
const delaySendFrontmsgTimer = require('../business/timer/delaySendFrontmsgTimer');
const ProcessMakeUpTaskTimer = require('../business/timer/ProcessMakeUpTaskTimer');
const GalaxyTaskStatusTimer = require('../business/timer/GalaxyTaskStatusTimer');
const { setRecordInfo } = require('../../common/recordActivityInfo');

function scheduleRun() {
    cron.schedule('*/10 * * * * *', () => {
      HeartTimer.heartBeat();
      delaySendFrontmsgTimer.start();
      PingTimer.ping();
      ProcessMakeUpTaskTimer.taskMakeUpTask();
      GalaxyTaskStatusTimer.checkGalaxyTaskStatus();
      setRecordInfo(0);
    });
}
module.exports = scheduleRun;
```

### 2.1 cron 表达式

`*/10 * * * * *` 表示**每 10 秒执行一次**。

```
┌───────── 秒 (0-59)   → */10 = 每10秒
│ ┌─────── 分 (0-59)   → * = 每分钟
│ │ ┌───── 时 (0-23)   → * = 每小时
│ │ │ ┌─── 日 (1-31)   → * = 每天
│ │ │ │ ┌─ 月 (1-12)   → * = 每月
│ │ │ │ │ ┌ 周 (0-7)   → * = 每天
│ │ │ │ │ │
*/10 * * * * *
```

### 2.2 启动时机

在 `AppStart.run()` 中调用：

```javascript
// galaxy-client/src/msg-center/start/appStart.js
const AppMain = {
    run() {
        global.startTime = Date.now();
        FrontStart.run();
        ReverseStart.run();
        scheduleRun();           // ← 启动定时任务
        MemoryQueueApplication.run();
        judgeCrashAndReport();
    }
}
```

### 2.3 执行顺序

每 10 秒的回调内，6 个任务**按顺序同步调用**（但各任务内部可能有异步操作）：

```
1. HeartBeatTimer.heartBeat()         → 逆向 IPC 心跳检测
2. delaySendFrontmsgTimer.start()     → 延迟消息聚合推送
3. PingTimer.ping()                   → 网络检测 + MQTT 心跳 + 逆向 ping
4. ProcessMakeUpTaskTimer.taskMakeUpTask() → MQTT 消息补偿上报
5. GalaxyTaskStatusTimer.checkGalaxyTaskStatus() → 任务状态超时检查
6. setRecordInfo(0)                   → 活动心跳记录（崩溃检测）
```

## 3. 五个核心定时器详解

### 3.1 HeartBeatTimer — 逆向 IPC 心跳检测

**文件**：`galaxy-client/src/msg-center/business/timer/HeartBeatTimer.js`

**职责**：监控所有已连接微信/企微实例的逆向 IPC（Named Pipe）心跳，超时则标记为不可用并移除。

#### 核心逻辑

```javascript
const heartBeatOverTime = ApplicationConfigEnv['heart.beat.over.time']; // 120 秒

const HeartBeatTimer = {
    heartBeat() {
        let removeList = [];
        const registryList = RegistryConfig.getRegistryList();
        
        if (!registryList?.length) {
            frontSendService.sendGetAllConfig();
            return;
        }
        
        for (let registry of registryList) {
            const currentTime = Math.floor(Date.now() / 1000);
            const lastPongTime = Math.floor(registry.pipeLineWrapper?.lastPongTime / 1000);
            
            if (lastPongTime) {
                // 未登录的实例 → 触发登录
                if (!registry.wxid) {
                    reverseSendService.sendMessage(null, registry.id, loginJson);
                }
                
                const timeInterval = currentTime - lastPongTime;
                if (timeInterval > heartBeatOverTime) {
                    // 虽然心跳超时但仍在读取消息 → 跳过
                    const lastReadTime = Math.floor(registry.pipeLineWrapper.lastReadTime / 1000);
                    if (lastReadTime && currentTime - lastReadTime <= heartBeatOverTime) {
                        continue;
                    }
                    // 真正超时 → 标记不可用，移除
                    registry.pipeLineWrapper.available = IpcConstant.NOT_AVAILABLE;
                    removeList.push(registry.id);
                    RegistryConfig.removePipe(registry.id);
                    notify.onPongOvertime(registry, timeInterval);
                }
            }
            else if (!lastPongTime && registry.scanTime) {
                // 建立连接后从未收到 pong
                const timeInterval = currentTime - Math.floor(registry.scanTime / 1000);
                if (timeInterval > heartBeatOverTime) {
                    removeList.push(registry.id);
                    RegistryConfig.removePipe(registry.id);
                }
            }
        }
        
        if (removeList?.length) {
            frontSendService.sendGetAllConfig();  // 通知前端更新
        }
    }
};
```

#### 心跳判定流程

```
遍历 registryList
    │
    ├─ 有 lastPongTime？
    │   ├─ 未登录 → 发送 login 指令
    │   ├─ 超时（> 120s）？
    │   │   ├─ 仍在读取消息 → 跳过（可能只是 pong 延迟）
    │   │   └─ 真正超时 → 标记不可用 → 移除 → 通知
    │   └─ 未超时 → 正常
    │
    ├─ 无 lastPongTime + 有 scanTime？
    │   └─ 超时 → 连接建立后始终无心跳 → 移除
    │
    └─ 无 scanTime → 异常状态 → 移除
```

#### 配置参数

| 参数 | 值 | 说明 |
|------|---|------|
| `heart.beat.over.time` | 120（秒） | 心跳超时阈值 |
| `heart.beat.time.interval` | 10000（毫秒） | 心跳检测间隔（由 cron 控制） |

### 3.2 delaySendFrontmsgTimer — 延迟聚合推送

**文件**：`galaxy-client/src/msg-center/business/timer/delaySendFrontmsgTimer.js`

**职责**：将高频的群信息变更和好友变更消息聚合后批量推送给前端，避免前端频繁渲染。

#### 核心设计

```
逆向回调 → 写入 chatroomUpdateMap / friendUpdateMap（缓冲）
                    ↓ 每10秒
delaySendFrontmsgTimer.start()
    ├─ chatroomUpdateFront()  → 聚合群变更 → 推送前端
    │   (await sleep 2000ms)
    └─ friendUpdateFront()    → 聚合好友变更 → 推送前端
```

#### 群变更聚合逻辑

```javascript
async chatroomUpdateFront() {
    const chatRoomKeys = Object.keys(this.chatroomUpdateMap);
    for (const wxId of chatRoomKeys) {
        const sendMsgs = this.chatroomUpdateMap[wxId];
        if (sendMsgs?.length > 0) {
            // 获取全量好友列表（用于计算非好友数）
            const friendList = await wkExternalUserService.findAllByAccountId({accountId: wxId});
            const friendListSet = new Set(friendList.map(item => item.wxid));
            
            // 清空缓冲（替换为空数组，新来的消息写入新数组）
            this.chatroomUpdateMap[wxId] = [];
            
            // 按 channelId 分组
            const groupByChannel = sendMsgs.reduce((acc, item) => {
                const key = item.id;
                if (!acc[key]) acc[key] = [];
                acc[key].push(item);
                return acc;
            }, {});
            
            // 按 chatroom 去重聚合（后来的覆盖先来的）
            Object.keys(groupByChannel).forEach(k => {
                const collect = v.map(s => s.data).reduce((acc, item) => {
                    if (!acc[item.chatroom]) {
                        acc[item.chatroom] = item;
                    } else {
                        // 增量更新：只覆盖有值的字段
                        if (item.headimg) acc[item.chatroom].headimg = item.headimg;
                        if (item.ownerId) acc[item.chatroom].ownerId = item.ownerId;
                        if (item.memberCount !== undefined) acc[item.chatroom].memberCount = item.memberCount;
                    }
                    return acc;
                }, {});
                
                sendToFront(JSON.stringify(msgTemplate), k, wxId);
            });
        }
    }
}
```

#### 好友变更聚合逻辑

- 按 `channelId` 分组
- 按 `wxid` 去重，保留最新一条
- 区分 `deleteFriend`（删除）和 `updateFriend`（更新）

#### 写入缓冲的接口

```javascript

friendUpdateSendFront(id, wxid, data)      // 由 convert-response 调用
```

### 3.3 PingTimer — 网络检测 + 多路心跳

**文件**：`galaxy-client/src/msg-center/business/timer/PingTimer.js`

**职责**：综合性保活定时器，同时负责网络检测、逆向 ping、MQTT 心跳、系统信息上报。

#### 核心流程

```javascript
const PingTimer = {
    lastReportTime: 0,
    isOnline: true,
    
    async ping() {
        const registryList = RegistryConfig.getRegistryList();
        
        // 1. 网络检测
        await this.checkInternet(registryList);
        
        // 2. 系统信息上报（每分钟一次）
        if (this.isOnline) {
            this.reportSystemInfo(registryList);
        }
        
        // 3. 遍历每个微信实例
        for (let registry of registryList) {
            // 3a. 发送逆向 ping
            const clientTask = ClientTaskFactory.getPingTask(registry.wxid);
            reverseSendService.sendMessage(registry.wxid, registry.id, JSON.stringify(clientTask));
            
            if (!this.isOnline) continue;  // 无网络时跳过云端操作
            
            // 3b. 未登录实例 → 触发登录
            if (!registry.wxid || !registry.pipeLineWrapper) {
                reverseSendService.sendMessage(null, registry.id, loginJson);
                continue;
            }
            
            // 3c. MQTT 断线重连
            if (!registry.mqttClient?.connected) {
                const now = Date.now();
                const reconnectInterval = 30 * 1000;
                if (now - (registry.lastMqttReconnectTime || 0) > reconnectInterval) {
                    registry.lastMqttReconnectTime = now;
                    await MqttHelper.connectMqtt({wxId: registry.wxid});
                }
            }
            
            // 3d. 发送 MQTT 心跳
            const clientMsg = {
                type: PrismRecordType.HEART_BEAT,
                heartBeatInfo: {
                    usernames: [registry.wxid],
                    deviceStatus: frontConnection.login ? 1 : 0,
                    checkInternetCost, reversePingCost, mqttReconnectCost, ...
                },
                username: registry.wxid,
            };
            MqttSendService.sendMessage(registry.wxid, clientMsg);
        }
    }
};
```

#### 网络检测

```javascript
checkInternet(registryList) {
    return new Promise((resolve) => {
        dns.lookup('baidu.com', (err) => {
            if (err && err.code === "ENOTFOUND") {
                if (this.isOnline) notify.onNetChange(false, registryList);
                this.isOnline = false;
            } else {
                if (!this.isOnline) notify.onNetChange(true, registryList);
                this.isOnline = true;
            }
            resolve(this.isOnline);
        });
    });
}
```

- 使用 DNS 解析 `baidu.com` 判断网络连通性
- 状态变化时通过 `notify` 通知前端

#### 系统信息上报

```javascript
async reportSystemInfo(registryList = []) {
    if (new Date().getTime() - this.lastReportTime < TIME_GAP) return;
    await ProcessReport.reportAll(registryList);
    this.lastReportTime = new Date().getTime();
}
```

- `TIME_GAP = 60000`（1 分钟），即使 cron 每 10 秒调用，也只每分钟上报一次
- 上报内容：CPU 使用率、内存占用等

#### MQTT 重连保护

- 30 秒内不重复重连（`lastMqttReconnectTime`）
- 避免网络波动时频繁创建 MQTT 连接

#### 性能监控

`PingTimer` 内嵌了详细的耗时打点：

| 指标 | 说明 |
|------|------|
| `checkInternetCost` | 网络检测耗时 |
| `reversePingCost` | 逆向 ping 发送耗时 |
| `mqttReconnectCost` | MQTT 重连耗时 |
| `singleRobotCost` | 单个机器人处理耗时 |
| `totalPingCost` | 总耗时，超 5 秒打警告 |

### 3.4 ProcessMakeUpTaskTimer — MQTT 消息补偿

**文件**：`galaxy-client/src/msg-center/business/timer/ProcessMakeUpTaskTimer.js`

**职责**：从补偿队列中取出发送失败的 MQTT 消息，在连接恢复后重新发送。

```javascript
const makeUpTimeOut = ApplicationConfigEnv.makeUpTimeOut;  // 600000ms = 10分钟

const ProcessMakeUpTaskTimer = {
    taskMakeUpTask() {
        for (let i = 0; i < 10; i++) {
            const makeUpMsg = MqttMakeUpManager.NOT_UPLOAD_MQTT_MESSAGE_QUEUE.pop();
            if (makeUpMsg) {
                let wxid = makeUpMsg.wxid;
                const mqttClient = RegistryConfig.getRegistryByKey(wxid, 'wxid')?.mqttClient;
                
                if (mqttClient?.connected) {
                    // 超过10分钟 → 丢弃
                    if (new Date().getTime() - makeUpMsg.timestamp > makeUpTimeOut) {
                        continue;
                    }
                    // MQTT 已连接 → 补偿上报
                    MqttSendService.sendMessage(wxid, makeUpMsg.body);
                }
            } else {
                break;  // 队列为空，退出
            }
        }
    }
};
```

#### 补偿机制

```
MQTT 发送失败
    │
    └─→ mqttMakeUpManager.push(msg)  → 入补偿队列
                                          │
    每10秒 ←──────────────────────────────┘
    ProcessMakeUpTaskTimer.taskMakeUpTask()
        ├─ 每轮最多取 10 条
        ├─ 超过 10 分钟 → 丢弃
        ├─ MQTT 已连接 → 重新发送
        └─ MQTT 未连接 → 消息留在队列（下次处理）
```

| 参数 | 值 | 说明 |
|------|---|------|
| 每轮处理量 | 10 条 | 避免单轮阻塞太久 |
| `makeUpTimeOut` | 600000ms（10 分钟） | 超时消息丢弃 |
| 调度间隔 | 10 秒（cron） | 补偿上报频率 |

### 3.5 GalaxyTaskStatusTimer — 任务状态超时检查

**文件**：`galaxy-client/src/msg-center/business/timer/GalaxyTaskStatusTimer.js`

**职责**：检查 Galaxy 任务的三阶段完成状态，超时则上报超时结果到云端。

```javascript
const imgAndVideoOverTime = ApplicationConfigEnv['over_time.galaxy.task.img.and.video.status'];
// 240000ms = 4分钟

const GalaxyTaskStatusTimer = {
    checkGalaxyTaskStatus() {
        const galaxyTaskStatusMap = GalaxyTaskCache.GALAXY_TASK_STATUS_MAP;
        const keys = Object.keys(galaxyTaskStatusMap);
        
        for (let entryKey of keys) {
            const entryValue = galaxyTaskStatusMap[entryKey];
            const time = Date.now() - entryValue.createTime;
            
            // 三阶段全部完成 → 删除任务
            if (entryValue.firstMessageStatus === RECEIVE_STATUS 
                && entryValue.secondMessageStatus === RECEIVE_STATUS
                && entryValue.thirdMessageStatus === RECEIVE_STATUS) {
                delete galaxyTaskStatusMap[entryKey];
                continue;
            }
            
            // 超时 → 上报超时结果 → 删除任务
            if (time > imgAndVideoOverTime) {
                const clientMsgCloud = {
                    username: entryValue.username,
                    type: PrismRecordType.TASK_RESULT_MSG,
                    taskInfo: {
                        status: CommonConstant.OVER_TIME_TASK,
                        reason: GalaxyTaskStatusConstant.OVER_TIME_REASON,
                        taskId: entryValue.taskId,
                    }
                };
                MqttSendService.sendMessage(entryValue.username, clientMsgCloud);
                delete galaxyTaskStatusMap[entryKey];
            }
        }
    }
};
```

#### 三阶段任务模型

```
任务下发 → 第一阶段回调 → 第二阶段回调 → 第三阶段回调
           (指令确认)     (执行进度)     (结果回报)
```

| 状态 | 含义 |
|------|------|
| 三个阶段均为 `RECEIVE_STATUS` | 任务正常完成，清理缓存 |
| 任一阶段未完成且超时 | 任务超时，上报 `OVER_TIME_TASK` |

| 参数 | 值 | 说明 |
|------|---|------|
| `over_time.galaxy.task.img.and.video.status` | 240000ms（4 分钟） | 图片/视频任务超时时间 |

## 4. setRecordInfo(0) — 活动心跳记录

每 10 秒调用一次 `setRecordInfo(0)` 写入活动信息到 `electron-store`：

```javascript
// galaxy-client/src/common/recordActivityInfo.js
function setRecordInfo(status) {
    store.setActivityInfo({
        status,           // 0=运行中, 1=正常退出
        timestamp: Date.now(),
        wxIdStr: getWxIdStr(),
        casId: store.getUserId(),

    });
}
```

- `status=0` 表示正在运行
- 下次启动时，如果发现 `status=0` 且距离 `timestamp` 超过 30 秒，判定为崩溃（详见文档 13）

## 7. 异常处理策略

### 7.1 每个定时器独立 try-catch

所有 5 个核心定时器的入口方法都包裹在 try-catch 中：

```javascript
heartBeat() {
    try {
        // ... 核心逻辑
    } catch(error) {
        logUtil.customLog(`[codeError] [HeartBeatTimer] ...`, 
            {level: 'error', errorKey: 'heartBeatTimerError'});
    }
}
```

### 7.2 异常不传播

cron 回调内各任务独立执行，一个任务的异常不会影响后续任务：

```javascript
cron.schedule('*/10 * * * * *', () => {
    HeartTimer.heartBeat();           // 即使这里抛异常
    delaySendFrontmsgTimer.start();   // 这里仍然执行
    PingTimer.ping();                 // 这里也执行
    // ...
});
```

注意：这依赖于各任务内部的 try-catch。如果某个任务没有 catch 异常，**会**阻断后续任务。目前所有 5 个任务均有 try-catch 保护。

### 7.3 日志标记

所有异常日志使用 `[codeError]` 前缀和特定的 `errorKey`，便于在 SLS 中按关键字检索：

| errorKey | 对应定时器 |
|----------|-----------|
| `heartBeatTimerError` | HeartBeatTimer |
| `delaySendFrontMsgTimerError` | delaySendFrontmsgTimer |
| `pingTimerError` / `pingTimerError1` | PingTimer |
| `processMakeUpTaskTimerError` | ProcessMakeUpTaskTimer |
| （无独立 key，复用通用错误） | GalaxyTaskStatusTimer |

## 8. 定时任务的性能影响

### 8.1 每轮执行开销

| 定时器 | 主要开销 | 影响 |
|--------|---------|------|
| HeartBeatTimer | 遍历注册列表（通常 ≤ 10 个） | 极低 |
| delaySendFrontmsgTimer | SQLite 查询 + JSON 序列化 | 中等（取决于缓冲大小） |
| PingTimer | DNS 查询 + 逆向 IPC + MQTT 发送 | 较高（含网络 IO） |
| ProcessMakeUpTaskTimer | 最多 pop 10 条 + MQTT 发送 | 低 |
| GalaxyTaskStatusTimer | 遍历任务状态 Map | 极低 |
| setRecordInfo | electron-store 写入 | 极低 |

### 8.2 PingTimer 耗时监控

PingTimer 内嵌了耗时监控，总耗时超过 5 秒会打警告日志：

```javascript
const totalPingCost = Date.now() - pingStartTime;
if (totalPingCost > 5000) {
    logUtil.customLog(`[警告] ping总耗时过长: ${totalPingCost}ms`, {level: 'warn'});
}
```
