# 01 工程目录结构与文件职责

> **适用仓库**：`galaxy-client`（Electron 主进程端）+ `galaxy`（React 前端渲染端）  
> **文档目标**：两仓库目录树全量速查，让开发者看到文件名就知道去哪找代码。  
> **前提**：`galaxy` 仓库的 `electron/` 目录代码已废弃，Electron 主进程完全由 `galaxy-client` 承担。

## 二、src/common/ — 通用模块（21个文件）

该目录存放主进程侧的通用工具函数和模块，不包含业务逻辑。

## 三、src/event/ — Electron 事件注册（9个文件）

每个文件对应一组事件的注册逻辑，在 `electron.js` 的 `bootstrap()` 中被调用。

| 文件名 | 职责 | 关键事件/函数 |
|--------|------|--------------|
| `app.js` | ★ Electron `app` 生命周期事件 | `ready` → 初始化窗口；`before-quit` → 停止逆向服务并退出；`second-instance` → 单实例锁；`render-process-gone` → 崩溃上报 |
| `ipc.js` | ★ IPC 主进程事件处理（electron-better-ipc） | `userLogin`, `userLogout`, `runInject`, `get-app-config`, `set-user-info`, `exist-file`, `get-ws-port`, `readFileAsBase64` 等 |
| `store.js` | electron-store 变更监听 | `userId` 变更 → 重新加载 URL；`isGray` 变更 → 重新加载 URL |
| `updater.js` | electron-updater 自动更新 | 检查更新、下载、安装流程 |
| `regedit.js` | Windows 注册表初始化 | `initRegedit()` |
| `downloadFile.js` | 文件下载事件 | — |
| `extractZip.js` | ZIP 解压事件 | — |
| `ffmpeg.js` | FFmpeg 音视频处理事件 | — |
| `changename.js` | 文件重命名事件 | — |

## 四、src/init/ — 初始化模块（6个文件）

在应用启动时按序执行的初始化操作。

| 文件名 | 职责 | 调用时机 |
|--------|------|----------|
| `initLog.js` | ★ electron-log 日志系统初始化 | `electron.js` 第一个调用（`initLog()`） |

| `slsLog.js` | 阿里云 SLS 日志上报初始化与发送 | 由 `initLog.js` 调用 `initSlsLog()` |
| `remoteDebug.js` | 远程调试支持（Node Inspector + CDP 代理） | `electron.js` 中 `setupRemoteDebug()` |
| `window.js` | ★ 主窗口创建（BrowserWindow 配置、URL 加载、更新器初始化） | `app.on('ready')` → `initWindow()` |
| `habo.js` | Habo 行为埋点上报初始化 | 由 `window.js` 引用 |

## 五、src/msg-center/ — 消息中心（核心架构）

这是整个 `galaxy-client` 最庞大的目录（472+文件），采用分层架构设计。

### 5.1 架构层级概览

```
msg-center/
├── start/                  # 启动入口层
├── core/                   # 核心基础设施层
├── dispatch-center/        # 消息调度层
└── business/               # 业务处理层
```

### 5.2 start/ — 启动入口（4个文件）

| 文件名 | 职责 | 启动方式 |
|--------|------|----------|
| `appStart.js` | ★ 应用级启动编排 | `AppMain.run()` — 依次启动 FrontStart、ReverseStart、scheduleRun、MemoryQueue、崩溃检测 |
| `frontStart.js` | 前端 WebSocket Server 启动 | `FrontServer.start()` |
| `reverseStart.js` | 逆向 IPC 扫描启动 | `initIpcTask.getInstance().run()` |
| `schedual.js` | 定时任务调度器（node-cron，每10秒） | 心跳、Ping、消息补发、任务补偿、Galaxy 状态检查、崩溃检测心跳 |

| 字段 | 说明 | 示例值 |
