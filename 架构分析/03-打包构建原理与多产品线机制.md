## 2. scripts/build.js 全流程脚本解析

**文件路径**：`galaxy-client/scripts/build.js`

### 2.1 入口函数

```javascript
const bootstrap = async () => {
    const currentVersion = process.env.Version;
    if (!currentVersion) {
        console.log("未获取到环境变量%Version%");
        return;
    }
    // Step 1: 修改版本号
    await changePkgVersion(currentVersion);

    // Step 2: 获取版本对应的 DLL 配置
    const currentVersionConfig =
        currentVersion in versionConfig
            ? versionConfig[currentVersion]
            : versionConfig.default;

    // Step 3: 下载、解压、替换 DLL 文件
    deleteAllFilesInDirectorySync(targetDir);
    for (const [key, val] of Object.entries(currentVersionConfig)) {
        const directory = await unzipper.Open.url(request, val);
        // 过滤 mock 文件，解压到 extraResources/Inject/
    }

    // Step 4: 执行构建
    const buildEnv = currentVersion.includes("release") ? "prod" : "test";
    execSync(`npm run app-build:${buildEnv}`, { stdio: "inherit" });

    // Step 5: 上传到 OSS
    await uploadFile(currentVersion);
};
```

### 2.2 关键步骤详解

| 步骤 | 说明 | 代码位置 |
|------|------|---------|
| 修改版本号 | 将 `process.env.Version` 写入 `package.json` | `changePkgVersion()` |
| 下载 DLL | 从内部 CI 服务器下载 Sirius/BasicService/Galaxy DLL | `unzipper.Open.url()` |
| 清理旧文件 | 递归删除 `extraResources/Inject/` 下所有文件 | `deleteAllFilesInDirectorySync()` |
| 判断构建环境 | 版本号含 `release` → prod，否则 → test | `buildEnv` 变量 |
| 执行 electron-builder | 调用 `npm run app-build:prod/test` | `execSync()` |
| 上传 OSS | 将 .exe 安装包上传到阿里云 OSS | `uploadFile()` |

## 3. webpack.electron.config.js：主进程打包配置

**文件路径**：`galaxy-client/webpack.electron.config.js`

```javascript
module.exports = {
    mode: 'production',
    entry: resolve('src/electron.js'),
    output: {
        path: resolve('./public'),
        filename: 'electron.js',
    },
    module: {
        rules: [{
            test: /\.js$/,
            use: { loader: 'babel-loader' },
            exclude: /node_modules/
        }]
    },
    target: 'electron-main',
    stats: 'errors-only',
};
```

### 3.1 配置要点

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `mode` | `'production'` | 开启 Tree Shaking、压缩 |
| `entry` | `src/electron.js` | 主进程入口 |
| `output.path` | `./public` | 输出目录 |
| `output.filename` | `electron.js` | 输出文件名 |
| `target` | `'electron-main'` | 告诉 Webpack 构建目标是 Electron 主进程 |
| `loader` | `babel-loader` | 转译 ES6+ 语法 |
| `exclude` | `/node_modules/` | 原生模块不打包 |

### 3.2 target: electron-main 的影响

- Webpack 不会打包 Node.js 内置模块（`fs`、`path`、`child_process` 等）
- 不会打包 Electron 相关模块（`electron`、`@electron/remote` 等）
- 原生 C++ 模块（`ffi-napi`、`sqlite3`）通过 externals 排除

### 3.3 为什么开发时不需要 Webpack

开发时直接运行 `src/electron.js` 源码：
- Electron 内置完整的 Node.js 运行时
- Babel 通过 `@babel/register` 实时转译（或直接支持 ES6）
- 省去打包步骤，加快启动速度

生产时需要 Webpack 打包：
- 减少文件数量，优化加载性能
- 混淆压缩代码
- 确保语法兼容性

### 3.4 pre-build 命令

```json
"pre-build:test": "cross-env ELECTRON_NODE_ENV=test webpack --config webpack.electron.config.js"
```

此命令将 `src/electron.js` 打包为 `public/electron.js`，作为生产环境的主进程入口。

### 4.2 BUILD_PATH 环境变量的作用

| BUILD_PATH 值 | 效果 | 说明 |
|---------------|------|------|
| `build`（默认） | 生成 4 个 HTML 入口（load/menu/sub/vpn） | Electron 安装包使用 |

| `weixinzhushou` | 产品线特定构建 | electron-builder 打包用 |

### 4.3 config-overrides.js 中的入口选择逻辑

```javascript
// 根据 BUILD_PATH 选择不同的入口点
if (process.env.BUILD_PATH === 'browser') {
    // 浏览器模式：使用 index.browser.js + browser.html
    multipleEntry = [{ entry: 'src/index.browser.js', template: 'public/browser.html' }];
} else {
    // Electron 模式：4 个入口
    multipleEntry = [
        { entry: 'src/entries/load/index.js', ... },
        { entry: 'src/entries/menu/index.js', ... },
        { entry: 'src/entries/sub/index.js', ... },
        { entry: 'src/entries/vpn/index.js', ... },
    ];
}
```


- 运行时通过 `getAppConfig()` 读取 `app-config.yml`
- 实现了不同产品线使用不同配置的机制

## 6. 多环境区分机制

### 6.1 环境分类

| 环境 | ELECTRON_NODE_ENV | 版本号特征 | 配置源 |
|------|-------------------|-----------|--------|
| 开发 | `dev` | 本地运行 | `runtime.yml` + `runtime.dev.yml` 合并 |
| 测试 | `test` | `5.5.0-test` | `applicationQa.js` |
| 生产 | `prod` | `5.5.0-release01` | `applicationProd.js` |

### 6.2 环境判断逻辑

**文件路径**：`galaxy-client/src/msg-center/core/application-config/index.js`

```javascript
// 环境判断优先级
let application;
if (process.env.ELECTRON_NODE_ENV === 'test' 
    || process.env.ELECTRON_NODE_ENV === 'dev') {
    application = applicationQa;       // 测试/开发环境
} else if (process.env.ELECTRON_NODE_ENV === 'prod') {
    application = applicationProd;     // 生产环境
} else {
    // 根据 urls.type 动态判断
    application = urls.type === 'prod' ? applicationProd : applicationQa;
}
```

### 6.3 构建命令与环境的映射

```json
{
  "app-build:test": "electron-builder -w -c config/weixinzhushou/build.yml -p never",
  "app-build:prod": "electron-builder -w -c config/weixinzhushou/build.yml -p never"
}
```

两者使用相同的 `build.yml`，区别在于 `scripts/build.js` 中 `changePkgVersion()` 写入的版本号不同：
- `release` 版本 → prod 构建
- 非 `release` 版本 → test 构建

### 6.4 版本号与环境的关系

**文件路径**：`galaxy-client/changePkgVer.js`

```javascript
// test → prod: 去掉 -test.N 后缀
if (version.indexOf('test') !== -1 && preMode === 'prod') {
    json.version = version.replace(/(.*)-test\.\d+$/, '$1');
}

// prod → test: 添加 -test.1 后缀
if (version.indexOf('test') === -1 && preMode === 'test') {
    json.version = version + '-test.1';
}
```

版本号示例：
| 原始版本 | 目标环境 | 转换结果 |
|---------|---------|---------|
| `5.5.0-release01` | prod | `5.5.0-release01`（不变） |
| `5.5.0-release01` | test | `5.5.0-release01-test.1` |
| `5.5.0-test.3` | prod | `5.5.0` |

### 6.5 URLs 的环境适配

**文件路径**：`galaxy-client/src/common/urls.js`

```javascript
const type = (() => {
    if (process.env.ENV_TYPE) return process.env.ENV_TYPE;
    // 根据版本号自动判断
    const version = app.getVersion();
    if (version.includes('release')) return 'prod';
    if (version.includes('test')) return 'test';
    return 'test'; // 默认测试
})();
```

### 7.5 条件编译（conditional-loader）

**文件路径**：`galaxy/conditional-loader.js`

通过文件名后缀实现条件编译：

```
src/common/ipc.js           ← Electron 模式使用
src/common/ipc.browser.js   ← 浏览器模式使用（BUILD_PATH=browser）
src/common/ipc.browserDev.js ← 开发浏览器模式使用（BUILD_PATH=browserDev）
```

工作原理：
1. Webpack loader 拦截所有 `.js`/`.ts` 文件
2. 检测 `BUILD_PATH` 环境变量
3. 如果为 `browser`，查找同名的 `.browser.js` 文件
4. 存在则替换文件内容
5. 不存在则使用原始文件

### 7.6 Web 构建模式

```json
{
  "web-build:test": "cross-env REACT_APP_ENV=test BUILD_PATH=web react-app-rewired build && node upload.js",
  "browser-build:test": "cross-env REACT_APP_ENV=test BUILD_PATH=browser react-app-rewired build && node uploadBrowser.js"
}
```

| 模式 | BUILD_PATH | 用途 | IPC | electron |
|------|-----------|------|-----|----------|
| Electron | `build` | 桌面客户端 | `electron-better-ipc` | `window.require('electron')` |
| Browser | `browser` | 纯浏览器 H5 | Proxy 空实现 | `{}` 空对象 |
| Web | `web` | Web 部署 | Proxy 空实现 | `{}` 空对象 |

## 8. 构建产物上传 OSS

### 8.1 uploadFile.js

**文件路径**：`galaxy-client/scripts/uploadFile.js`

```javascript
async function uploadFile(version) {
    const uploadEnv = version.includes("release") ? "prod" : "test";
    const fileName = `Weixinzhushou-win-${version}.exe`;
    const uploadToPath = `qz-wxzs/${uploadEnv}/${fileName}`;

    // 获取阿里云 OSS STS Token
    const tokenInfo = await fetchOssToken();

    const client = new OSS({
        region: "oss-cn-beijing",
        accessKeyId: tokenInfo.accessKey,
        accessKeySecret: tokenInfo.secretKey,
        stsToken: tokenInfo.securityToken,
        bucket: "gh-fe",
    });

    // 分片上传
    await client.multipartUpload(uploadToPath, filePath, {
        progress(p) {
            console.log(`上传进度：${(p * 100).toFixed(2)}%`);
        },
    });
}
```

```javascript
const ossTokenUrl = "https://qun-yunying.umeng100.com/uqun/token/aly/internal/open/ossToken";
```

- `5.5.0-release01` — 生产发布版本

| Habo 埋点 | `urls.clientVersion` | 行为统计按版本分析 |

### 9.3 changePkgVer.js 的作用

```javascript
// scripts/build.js
const currentVersionConfig =
```
