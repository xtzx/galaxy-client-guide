## 1. 安全架构总览

Galaxy Client 的安全设计分为两个层次：主窗口（低安全、高权限）和 webview（相对安全）：

```
┌───── BrowserWindow (主窗口) ─────────────────────────────────┐
│                                                               │
│  webPreferences:                                              │
│    contextIsolation: false                                    │
│    nodeIntegration: true                                      │
│    sandbox: false                                             │
│                                                               │
│  preload: inject.js                                           │
│    → window.require = require                                 │
│    → window.eleRemote = @electron/remote                      │
│    → window.preloadWebviewPath = webviewPreload.js 路径       │
│    → CAS 登录页自动填充                                        │
│                                                               │
│  渲染进程可直接调用：                                            │
│    require('electron')                                        │
│    require('fs')                                              │
│    require('@electron/remote')                                │
│    process.env / __dirname / ...                              │
│                                                               │
│  ┌──── <webview> ─────────────────────────────────────────┐  │
│  │                                                         │  │
│  │  preload: webviewPreload.js                             │  │
│  │  contextIsolation: true (Electron 默认)                  │  │
│  │  nodeIntegration: false (webview 默认)                   │  │
│  │                                                         │  │
│  │  只暴露一个安全 API：                                    │  │
│  │    window.electronAPI.readFileAsBase64(filePath)         │  │
│  │    → ipcRenderer.invoke('readFileAsBase64', filePath)    │  │
│  │                                                         │  │
│  └─────────────────────────────────────────────────────────┘  │
└───────────────────────────────────────────────────────────────┘
```

## 2. 主窗口 webPreferences 配置

**文件路径**：`galaxy-client/src/init/window.js`

```javascript
const mainWindow = createStateWindow({
    webPreferences: {
        sandbox: false,
        preload: path.resolve(__dirname, '../../extraResources/load/inject.js'),
        nodeIntegration: true,
        webviewTag: true,
        allowRunningInsecureContent: true,
        enableRemoteModule: true,
        contextIsolation: false,
    },
});
```

### 2.1 配置项详解

| 配置项 | 值 | Electron 默认 | 说明 |
|--------|-----|--------------|------|
| `sandbox` | `false` | `true` (Electron 20+) | 关闭沙箱，preload 可完全访问 Node.js |
| `nodeIntegration` | `true` | `false` | 渲染进程可直接使用 `require` |
| `contextIsolation` | `false` | `true` (Electron 12+) | 渲染进程与 preload 共享同一 JS 上下文 |
| `webviewTag` | `true` | `false` | 允许使用 `<webview>` 标签 |
| `allowRunningInsecureContent` | `true` | `false` | 允许加载 HTTP 内容（混合内容） |
| `enableRemoteModule` | `true` | `false` (Electron 14+) | 启用 `@electron/remote` |

### 2.2 为什么不遵循安全最佳实践

项目运行在**受控的企业环境**中：
- 不加载不可信的外部内容
- 加载的页面都是自己的前端应用或 CAS 登录页
- 前端需要大量使用 Node.js API（`fs`、`path`、`electron` 等）
- 使用 `@electron/remote` 访问主进程模块

## 3. inject.js — 主窗口 preload 脚本

**文件路径**：`galaxy-client/extraResources/load/inject.js`

### 3.1 完整源码

```javascript
const path = require('path');

window.__injected = true;
window.require = require;
window.preloadPath = path.join('file://', path.join(__dirname, './inject.js'));
window.eleRemote = require('@electron/remote');
window.eleRemoteMain = require('@electron/remote/main');
window.isDev = !eleRemote.app.isPackaged;
window.preloadWebviewPath = path.join('file://', path.join(__dirname, './webviewPreload.js'));

(function () {
    const DELAY = 1000;
    const ele = window.require('electron');
    const ipc = ele.ipcRenderer;
    let username;
    let timer;

    function setUsername() {
        const input = document.getElementById('username');
        if (username && input) {
            const nativeInputValueSetter =
                Object.getOwnPropertyDescriptor(window.HTMLInputElement.prototype, 'value').set;
            nativeInputValueSetter.call(input, username);
            const event = new Event('input', { bubbles: true });
            input.dispatchEvent(event);
            return username;
        }
    }

    function go() {
        if (!setUsername()) {
            timer = setTimeout(go, DELAY);
        }
    }

    window.addEventListener('DOMContentLoaded', function () {
        // CAS 自动填充逻辑
        if (~location.href.indexOf('cas')) {
            ipc.on('cas-auto-complete-info', (event, data) => {
                if (username !== data.username) {
                    username = data.username;
                    clearTimeout(timer);
                    timer = setTimeout(go, DELAY / 4);
                }
            });
            setInterval(() => { ipc.send('cas-auto-complete-info'); }, DELAY);
        }
        // 登录错误检测
        if (~location.href.indexOf('userLoginInfo')) {
            // ...检测错误页面，触发 userLogout
        }
    });
})();
```

### 3.2 注入的全局变量

| 变量 | 类型 | 用途 |
|------|------|------|
| `window.__injected` | `boolean` | 标记 preload 已注入 |
| `window.require` | `function` | Node.js require 函数 |
| `window.preloadPath` | `string` | inject.js 自身路径（file:// 协议） |
| `window.eleRemote` | `object` | `@electron/remote` 模块 |
| `window.eleRemoteMain` | `object` | `@electron/remote/main` 模块 |
| `window.isDev` | `boolean` | 是否开发环境 |
| `window.preloadWebviewPath` | `string` | webviewPreload.js 路径 |

### 3.3 前端如何使用这些全局变量

```javascript
// galaxy/src/alias/electron.js
module.exports = window.require('electron');

// galaxy 前端代码中
const { ipcRenderer } = window.require('electron');
const app = window.eleRemote.app;
```

## 4. webviewPreload.js — webview preload 脚本

**文件路径**：`galaxy-client/extraResources/load/webviewPreload.js`

```javascript
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('electronAPI', {
    readFileAsBase64: (filePath) => {
        return ipcRenderer.invoke('readFileAsBase64', filePath);
    }
});
```

### 4.1 安全设计

与主窗口的 `inject.js` 形成鲜明对比：

| 维度 | inject.js (主窗口) | webviewPreload.js (webview) |
|------|-------------------|-----------------------------|
| `contextIsolation` | `false` | `true`（默认） |
| `nodeIntegration` | `true` | `false`（webview 默认） |
| 暴露方式 | `window.require = require` | `contextBridge.exposeInMainWorld` |
| 暴露范围 | 完整 Node.js + Electron API | 仅 `readFileAsBase64` 一个函数 |
| 安全性 | 低（完全信任渲染进程） | 高（最小权限原则） |

### 4.2 为什么 webview 更安全

webview 加载的内容可能是第三方页面（如嵌入的 H5），需要更严格的安全控制。通过 `contextBridge` 只暴露必要的 API，防止恶意脚本访问 Node.js。

## 6. Cookie 注入机制

**文件路径**：`galaxy-client/src/event/ipc.js`

### 6.1 writeUqunCookie — U群 Cookie

```javascript
ipc.on('writeUqunCookie', async (event, message) => {
    await session.defaultSession.cookies.set({
        url: message.uqunUrl + '/',
        name: 'isFromTongBao',
        value: 'true'
    });
    await session.defaultSession.cookies.set({
        url: message.uqunUrl + '/',
        name: 'tongBaoUserId',
        value: message.userId
    });
});
```

前端调用方：`galaxy/src/entries/menu/App.js`

```javascript
ipc.send('writeUqunCookie', { userId, uqunUrl: config.uqunUrl });
```

| Cookie 名 | 值 | 用途 |
|-----------|-----|------|
| `isFromTongBao` | `'true'` | 标识来源为同跑平台 |
| `tongBaoUserId` | 用户 ID | 传递用户身份 |

### 6.2 writeGidCookie — GID Cookie

```javascript
ipc.on('writeGidCookie', async (event, message) => {
    await session.defaultSession.cookies.set({
        url: message.url,
        name: 'GID',
        value: getStoreGid()
    });
});
```

前端调用方：`galaxy/src/entries/sub/index.js`

```javascript
ipc.send('writeGidCookie', { url: window.location.origin });
```

GID 是设备唯一标识，写入 Cookie 后供前端 HTTP 请求自动携带。

### 6.3 Cookie 注入原理

使用 `session.defaultSession.cookies.set()` 直接操作 Chromium 的 Cookie 存储。这是一个**主进程特权操作**，渲染进程无法直接操作 Cookie（跨域限制），因此通过 IPC 委托主进程完成。

## 7. readFileAsBase64 — IPC invoke 安全通道

**文件路径**：`galaxy-client/src/event/ipc.js`

```javascript
ipc.handle('readFileAsBase64', async (event, filePath) => {
    return new Promise((resolve, reject) => {
        const formatPath = filePath.replace(/%20/g, ' ');
        fs.readFile(formatPath, (err, bitmap) => {
            if (err) { reject(err); return; }
            const base64Data = `data:image/png;base64,${
                Buffer.from(bitmap, 'binary').toString('base64')
            }`;
            resolve(base64Data);
        });
    });
});
```

这是 webview 中唯一可调用的 Node.js 能力，通过 `ipcRenderer.invoke` → `ipc.handle` 的安全通道实现。

**数据流**：

```
webview 页面代码
    │
    └──▶ window.electronAPI.readFileAsBase64(filePath)
          │ (contextBridge 暴露)
          │
          └──▶ ipcRenderer.invoke('readFileAsBase64', filePath)
                │ (IPC 通道)
                │
                └──▶ 主进程 ipc.handle('readFileAsBase64')
                      │
                      └──▶ fs.readFile → base64 编码 → 返回
```

## 8. conditional-loader — 条件编译

**文件路径**：`galaxy/conditional-loader.js`

```javascript
module.exports = function (source) {
    const callback = this.async();
    const isTargetBrowserPlatform = process.env.BUILD_PATH === 'browser';

    if (!isTargetBrowserPlatform) {
        return callback(null, source);
    }

    const extensionMatch = this.resourcePath.match(/\.(jsx?|tsx?)$/);
    const fileExtension = extensionMatch ? extensionMatch[0] : '';
    const browserFileExtension = fileExtension.replace(
        /^(\.jsx?|\.tsx?)$/,
        '.browser$1'
    );

    const browserFilePath = this.resourcePath.replace(
        fileExtension,
        browserFileExtension
    );

    if (fs.existsSync(browserFilePath)) {
        this.addDependency(browserFilePath);
        fs.readFile(browserFilePath, 'utf8', (err, data) => {
            if (err) return callback(err);
            callback(null, data);
        });
    } else {
        callback(null, source);
    }
};
```

### 8.1 工作原理

这是一个 Webpack loader，实现**条件编译**：

1. 检查 `BUILD_PATH` 环境变量是否为 `browser`
2. 如果是浏览器构建，查找对应的 `.browser.js` 文件
3. 如果 `.browser.js` 存在，用它替换原始文件
4. 如果不存在，使用原始文件

### 8.2 文件替换规则

| 原始文件 | 浏览器替换文件 |
|----------|---------------|
| `electron.js` | `electron.browser.js` |
| `utils.ts` | `utils.browser.ts` |
| `component.jsx` | `component.browser.jsx` |

### 8.3 使用场景

galaxy 前端需要同时支持：
- **Electron 模式**：运行在 Electron 渲染进程中，可以使用 IPC
- **浏览器模式**：运行在普通浏览器中（开发调试或 SaaS 部署），需要剔除 Electron 依赖

## 9. Electron 别名系统

### 9.1 Electron 模式

**文件路径**：`galaxy/src/alias/electron.js`

```javascript
module.exports = window.require('electron');
```

依赖 `inject.js` 注入的 `window.require`。在 Webpack 中配置别名：

```javascript
// webpack config
resolve: {
    alias: {
        'electron': path.resolve('src/alias/electron.js')
    }
}
```

### 9.2 浏览器模式

**文件路径**：`galaxy/src/alias/electron.browser.js`

```javascript
module.exports = {};
```

在浏览器构建时，通过 `conditional-loader`，`electron.js` 被替换为 `electron.browser.js`，返回空对象。所有对 `electron` 的 `require` 都得到一个空对象，避免运行时错误。

### 9.3 别名依赖链

```
前端代码: import ipc from '@/common/ipc'
    │
    └──▶ ipc.js: const { ipcRenderer } = require('electron')
          │
          ├── [Electron 模式] → alias/electron.js
          │     → window.require('electron')
          │     → 真实的 electron 模块
          │
          └── [浏览器模式] → alias/electron.browser.js
                → {}
                → ipcRenderer = undefined
                → IPC 调用静默跳过
```

## 10. CSP 配置现状

**文件路径**：`galaxy-client/index.html`

```html
<!-- https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP -->
<!-- <meta
    http-equiv="Content-Security-Policy"
    content="default-src 'self'; script-src 'self'"
  />
  <meta
    http-equiv="X-Content-Security-Policy"
    content="default-src 'self'; script-src 'self'"
  /> -->
```

CSP（Content Security Policy）已被注释，**未启用**。

### 10.1 未启用 CSP 的原因

- 应用需要加载多个外部域名的资源（CAS 服务器、Habo 上报、API 服务等）
- 使用了 `allowRunningInsecureContent: true` 加载 HTTP 内容
- 主窗口使用 `nodeIntegration: true`，CSP 对主进程的脚本限制意义有限

## 11. 安全最佳实践对比

| Electron 安全建议 | 推荐值 | 本项目选择 | 原因 |
|-------------------|--------|------------|------|
| `nodeIntegration` | `false` | `true` | 前端需要 Node.js API |
| `contextIsolation` | `true` | `false`（主窗口）/ `true`（webview） | 主窗口需要共享上下文 |
| `sandbox` | `true` | `false` | preload 需完整 Node.js |
| CSP | 启用 | 注释未启用 | 多域名资源加载 |
| `allowRunningInsecureContent` | `false` | `true` | 内部 HTTP 服务 |
| `webviewTag` | 避免使用 | `true` | 多页面架构需要 |
| `@electron/remote` | 避免使用 | 启用 | 渲染进程需要主进程模块 |

### 11.1 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| XSS 攻击可访问 Node.js | 高 | 不加载不可信内容 |
| 远程代码执行 | 高 | 企业内网环境 |
| 混合内容降级 | 中 | HTTP 仅用于内部服务 |
| webview 逃逸 | 低 | webview 使用 contextBridge |

## 12. CAS 自动填充安全分析

`inject.js` 中的 CAS 自动填充功能：

```javascript
// 绕过 React 的合成事件系统，直接操作原生 input
const nativeInputValueSetter =
    Object.getOwnPropertyDescriptor(window.HTMLInputElement.prototype, 'value').set;
nativeInputValueSetter.call(input, username);
const event = new Event('input', { bubbles: true });
input.dispatchEvent(event);
```

### 12.1 为什么需要 nativeInputValueSetter

CAS 登录页是第三方 React 应用，直接设置 `input.value` 不会触发 React 的状态更新。通过访问原生 `HTMLInputElement.prototype.value` 的 setter 并手动派发 `input` 事件，可以正确触发 React 的 onChange。

### 12.2 数据流

```
主进程 store                 inject.js (preload)
    │                            │
    │ cas-auto-complete-info     │ 每秒轮询
    │◀─── IPC send ────────────│
    │                            │
    │──── IPC reply ──────────▶│
    │ { username: '...' }       │
    │                            │ setUsername()
    │                            │ → nativeInputValueSetter
    │                            │ → dispatchEvent('input')
    │                            │ → React onChange 触发
```
