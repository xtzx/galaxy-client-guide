## 1. 窗口架构全景

### 1.1 窗口结构

Galaxy Client 采用**单 BrowserWindow + 多 webview** 的架构：

```
┌─── Electron 进程架构 ───────────────────────────────────────┐
│                                                             │
│  主进程 (Main Process)                                       │
│    ├── electron.js 入口                                     │
│    ├── window.js → 创建主窗口                                │
│    ├── app.js → 应用生命周期                                 │
│    └── ipc.js → IPC 事件处理                                │
│                                                             │
│  ┌─── BrowserWindow (主窗口) ─────────────────────────────┐  │
│  │  preload: inject.js                                    │  │
│  │  nodeIntegration: true                                 │  │
│  │  webviewTag: true                                      │  │
│  │                                                        │  │
│  │  加载 URL:                                              │  │
│  │    ├── loginUrl (CAS 登录页)                            │  │
│  │    └── successUrl (menu.html 主界面)                     │  │
│  │                                                        │  │
│  │  ┌─── <webview> (sub.html?channelId=0) ──────────┐    │  │
│  │  │  preload: webviewPreload.js                    │    │  │
│  │  │  nodeIntegration: true                         │    │  │
│  │  │  独立渲染进程                                   │    │  │
│  │  └────────────────────────────────────────────────┘    │  │
│  │                                                        │  │
│  │  ┌─── <webview> (sub.html?channelId=N) ──────────┐    │  │
│  │  │  更多 sub 实例...                               │    │  │
│  │  └────────────────────────────────────────────────┘    │  │
│  │                                                        │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 与传统多窗口设计的区别

| 方面 | 传统多 BrowserWindow | Galaxy Client |
|------|---------------------|---------------|
| 窗口数量 | 每个页面一个 BrowserWindow | 1 个 BrowserWindow |
| 子页面 | 独立窗口 | `<webview>` 标签 |
| 窗口管理 | 需管理多窗口位置/大小 | 只需管理 1 个主窗口 |
| 进程架构 | 每窗口一个渲染进程 | 主窗口 + 每个 webview 各一个渲染进程 |

```javascript
const isDev = !app.isPackaged;

    const mainWindow = app.mainWindow = createStateWindow({
        width: 1060,
        height: 680,
        minWidth: 1060,
        minHeight: 680,

        mainWindow.loadURL(url);

```

### 2.2 窗口配置详解

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `width` / `height` | 1060 × 680 | 默认窗口尺寸 |
| `minWidth` / `minHeight` | 1060 × 680 | 最小窗口尺寸（不可缩小） |
| `show` | `false` | 创建时不立即显示（等待 `ready-to-show`） |

```javascript

```

启用 `@electron/remote` 模块，允许渲染进程通过 `require('electron').remote` 访问主进程模块（如 `app`、`dialog` 等）。

### 2.4 移除默认菜单

```javascript

```

Electron 默认创建的菜单栏（File、Edit、View 等）被移除，应用使用自定义 UI。

## 3. 状态持久化窗口 createStateWindow.js

**文件路径**：`galaxy-client/src/common/createStateWindow.js`

```javascript
module.exports = options => {
    let state = {};
    let win;
    const defaultSize = {
        width: options.width,
        height: options.height
    };

    const restore = () => {
        const restoredState = store.getWindowState();
        return { ...defaultSize, ...restoredState };
    };

    const getCurrentPosition = () => {
        const position = win.getPosition();
        const size = win.getSize();
        return {
            x: position[0],
            y: position[1],
            width: size[0],
            height: size[1]
        };
    };

    const windowWithinBounds = (windowState, bounds) => (
        windowState.x >= bounds.x
        && windowState.y >= bounds.y
        && windowState.x + windowState.width <= bounds.x + bounds.width
        && windowState.y + windowState.height <= bounds.y + bounds.height
    );

    const resetToDefaults = () => {
        const { bounds } = screen.getPrimaryDisplay();
        return {
            ...defaultSize,
            x: (bounds.width - defaultSize.width) / 2,
            y: (bounds.height - defaultSize.height) / 2
        };
    };

    const ensureVisibleOnSomeDisplay = windowState => {
        const visible = screen.getAllDisplays().some(
            display => windowWithinBounds(windowState, display.bounds)
        );
        if (!visible) {
            return resetToDefaults();
        }
        return windowState;
    };

    const saveState = () => {
        if (!win.isMinimized() && !win.isMaximized()) {
            Object.assign(state, getCurrentPosition(), { max: 0 });
        } else if (win.isMaximized()) {
            Object.assign(state, { max: 1 });
        }
        store.setWindowState(state);
    };

    state = ensureVisibleOnSomeDisplay(restore());
    win = new BrowserWindow({ ...options, ...state });

    // HTTP 请求头注入环境标记...

    if (state.max) {
        win.maximize();
    }

    win.on('close', saveState);
    return win;
};
```

### 3.1 窗口状态持久化

| 操作 | 说明 |
|------|------|
| `restore()` | 从 `electron-store` 读取上次保存的窗口状态 |
| `saveState()` | 窗口关闭时保存当前位置和大小 |
| `ensureVisibleOnSomeDisplay()` | 确保窗口在某个显示器内可见 |
| `resetToDefaults()` | 窗口不可见时重置到主显示器中央 |

### 3.2 windowState 数据结构

```javascript
{
    x: 100,       // 窗口左上角 X 坐标
    y: 200,       // 窗口左上角 Y 坐标
    width: 1060,  // 窗口宽度
    height: 680,  // 窗口高度
    max: 0        // 是否最大化（0=否，1=是）
}
```

### 3.3 多显示器支持

```javascript
const ensureVisibleOnSomeDisplay = windowState => {
    const visible = screen.getAllDisplays().some(
        display => windowWithinBounds(windowState, display.bounds)
    );
    if (!visible) {
        return resetToDefaults();
    }
    return windowState;
};
```

遍历所有显示器，检查窗口是否在某个显示器的可视范围内。如果不在（例如外接显示器已断开），自动重置到主显示器中央。

### 3.4 最大化状态恢复

```javascript
if (state.max) {
    win.maximize();
}
```

如果上次关闭时是最大化状态，恢复时自动最大化。

## 4. URL 加载策略 loadUrl.js

**文件路径**：`galaxy-client/src/common/loadUrl.js`

```javascript
const { loginUrl, successUrl, offlineUrl } = require('./urls');
const store = require('./store');

function getLoadUrl() {
    let userId = store.getUserId();
    if (!userId) {
        userId = randomString(16);
        store.setUserId(userId);
        return `${loginUrl}?userId=${userId}`;
    }
    return `${successUrl}?userId=${userId}`;
}

async function getLoadUrlAsync() {
    try {
        const result = await net.check();
        if (result) {
            return getLoadUrl();
        }
        return offlineUrl;
    } catch (error) {
        log.error('LOAD URL ERROR', error.message);
    }
}
```

### 4.1 URL 决策逻辑

```
getLoadUrlAsync()
    │
    ├── 网络检测 net.check()
    │     │
    │     ├── 成功 → getLoadUrl()
    │     │          │
    │     │          ├── 无 userId → 生成随机 ID → loginUrl
    │     │          │                （CAS 登录页）
    │     │          │
    │     │          └── 有 userId → successUrl
    │     │                        （主界面 menu.html）
    │     │
    │     └── 失败 → offlineUrl（离线提示页）
    │
    └── 异常 → 记录错误日志
```

### 4.2 三种 URL

| URL | 场景 | 示例 |
|-----|------|------|
| `loginUrl` | 首次启动，未登录 | `http://cas.example.com/login?userId=xxx` |
| `successUrl` | 已登录，进入主界面 | `http://localhost:3000/menu.html?userId=xxx` |
| `offlineUrl` | 无网络 | 本地离线提示页面 |

### 4.3 userId 的作用

`userId` 是一个 16 位随机字符串，在以下场景中使用：

1. CAS 登录时作为会话标识
2. 加载 `successUrl` 时携带，用于关联前端和主进程的用户身份
3. 持久化在 `electron-store` 中，避免每次启动都需要重新登录

### 5.2 各事件详解

| 事件 | 触发时机 | 处理逻辑 |
|------|----------|----------|
| `ready` | 应用初始化完成 | 创建窗口、注册协议、初始化注册表 |

| `browser-window-focus` | 窗口获焦 | 重新注册快捷键 |
| `render-process-gone` | 渲染进程崩溃 | 上报崩溃事件 |
| `open-url` | URL 协议唤醒 | 通知渲染进程处理 |

将 V8 引擎的堆内存上限设置为 6GB（默认约 1.7GB），防止在处理大量微信消息时出现 OOM。

## 6. 单实例锁与协议唤醒

### 6.1 单实例锁

```javascript
const gotTheLock = app.requestSingleInstanceLock();
if (gotTheLock) {
    app.on('second-instance', (event, argv) => {
        const { mainWindow } = app;
        if (mainWindow) {
            if (mainWindow.isMinimized()) mainWindow.restore();
            mainWindow.focus();
            mainWindow.show();
        }
        // 处理协议链接唤醒
        if (isWin) {
            const activeUrl = decodeURI(argv.slice().pop());
            sendToRenderMsg('web-awaken', activeUrl);
        }
    });
} else {
    app.quit();
    return true;
}
```

**作用**：防止用户打开多个客户端实例。第二个实例启动时：
1. 第一个实例接收到 `second-instance` 事件
2. 聚焦已有的主窗口
3. 如果通过协议链接唤醒，将 URL 传递给渲染进程处理

注册自定义协议（如 `wxzs://`），使得通过浏览器点击特定链接可以唤醒客户端。

## 7. webPreferences 安全配置

### 7.1 配置项详解

```javascript
webPreferences: {
    sandbox: false,
    preload: path.resolve(__dirname, '../../extraResources/load/inject.js'),
    nodeIntegration: true,
    webviewTag: true,
    allowRunningInsecureContent: true,
    enableRemoteModule: true,
    contextIsolation: false,
}
```

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `sandbox` | `false` | 关闭沙箱，允许 preload 脚本完全访问 Node.js |
| `preload` | `inject.js` | 预加载脚本路径 |
| `nodeIntegration` | `true` | 渲染进程可直接使用 Node.js API |
| `webviewTag` | `true` | 允许使用 `<webview>` 标签 |
| `allowRunningInsecureContent` | `true` | 允许加载 HTTP 内容（混合内容） |
| `enableRemoteModule` | `true` | 启用 `@electron/remote` |
| `contextIsolation` | `false` | 关闭上下文隔离 |

### 7.2 安全性考量

这些配置相比 Electron 安全最佳实践有较大偏差：

| 最佳实践 | 本项目选择 | 原因 |
|----------|------------|------|
| `nodeIntegration: false` | `true` | 前端需要直接调用 `require('electron')` |
| `contextIsolation: true` | `false` | 需要渲染进程与 preload 共享同一上下文 |
| `sandbox: true` | `false` | preload 需要完整 Node.js 能力 |
| 仅 HTTPS | 允许 HTTP | 内部服务使用 HTTP |

这是一个**功能优先于安全**的选择，因为客户端运行在受控的企业环境中，不加载不可信的外部内容。

## 8. 窗口关闭与退出流程

### 8.1 用户点击关闭按钮

```javascript
mainWindow.on('close', e => {
    e.preventDefault();

    if (updateFlag) {
        // 自动更新导致的关闭
        setTimeout(() => {
            app.mainWindow = null;
            app.quit();
        }, 1000);
    } else {
        // 用户手动关闭 → 弹出确认对话框
        dialog.showMessageBox(app.mainWindow, {
            type: 'warning',
            title: '关闭',
            message: '是否确认关闭？',
            buttons: ['取消', '确定'],
        }).then(index => {
            if (index.response === 1) {
                store.clearAutoCompleteInfo();
                sendToRenderMsg('ws-kill-java-only');
                sendToRenderMsg('logout-notify');
                logUtil.customLog('AppQuitConfirm');
                setRecordInfo(1);  // 标记正常退出
                setTimeout(() => {
                    app.mainWindow = null;
                    app.quit();
                }, 2000);
            }
        });
    }
});
```

### 8.2 完整退出流程

```
用户点击关闭
    │
    ├── [更新模式] → 1秒后直接退出
    │
    └── [正常模式] → 弹出确认对话框
          │
          ├── [取消] → 不退出
          │
          └── [确定]
               │
               ├── 1. store.clearAutoCompleteInfo()   // 清理自动填充
               ├── 2. sendToRenderMsg('ws-kill-java')  // 通知前端关闭WS
               ├── 3. sendToRenderMsg('logout-notify') // 通知前端登出
               ├── 4. setRecordInfo(1)                 // 写入正常退出标记
               └── 5. (2秒后) app.quit()
                        │
                        └──▶ before-quit 事件
                              ├── stopBasicService()
                              ├── stopBasicService64()
                              ├── RegistryConfig.removeAll()
                              └── app.exit()
```

### 8.3 before-quit 清理

```javascript
app.on('before-quit', event => {
    event.preventDefault();
    stopBasicService(() => {
        RegistryConfig.removeAll();
        app.exit();
    });
    stopBasicService64(() => {
        RegistryConfig.removeAll();
        app.exit();
    });
});
```

退出前确保：
1. 停止所有逆向 DLL 服务（32位 + 64位）
2. 清理所有注册表条目（关闭 IPC + MQTT 连接）
3. 调用 `app.exit()` 强制退出

## 9. 屏幕自适应 screenAdapter.js

**文件路径**：`galaxy-client/src/common/screenAdapter.js`

```javascript
function getScreenScaleFactor() {
    log.info('调整屏幕缩放中');
    const browserAllWindow = BrowserWindow.getAllWindows();
    if (Array.isArray(browserAllWindow)) {
        const browserWindow = browserAllWindow[0];
        if (browserWindow) {
            browserWindow.webContents.setZoomFactor(1);
        }
    }
}
```

### 9.1 触发时机

```javascript
// window.js
mainWindow.on('move', debounce(getScreenScaleFactor, 1000));
```

当窗口移动时（可能从一个显示器拖到另一个），使用防抖延迟 1 秒后重置缩放因子为 1。

### 9.2 设计目的

Windows 系统中不同显示器可能有不同的 DPI 缩放比例。当窗口在不同 DPI 的显示器间移动时，Electron 会自动调整缩放，可能导致 UI 变形。通过强制设置 `zoomFactor = 1` 确保 UI 一致性。

## 10. CAS 登录重定向机制

### 10.1 登录流程

```javascript
getLoadUrlAsync().then(url => {
    mainWindow.loadURL(url);

    session.defaultSession.webRequest.onBeforeRedirect({ urls: [] }, details => {
        const { url, redirectURL } = details;
        const redirectHost = getHostName(redirectURL);

        if (['www.baijiahulian.com', 'www.baijia.com'].includes(redirectHost)) {
            const nextUrl = getLoadUrl();
            app.mainWindow.loadURL(nextUrl);
        }
    });
});
```

### 10.2 登录重定向时序

```
1. mainWindow.loadURL(loginUrl)
   → 加载 CAS 登录页面

2. 用户输入凭据并提交

3. CAS 服务器验证通过，302 重定向到 www.baijia.com

4. onBeforeRedirect 拦截重定向
   → 检测到 redirectHost 是 www.baijia.com
   → 调用 getLoadUrl() 获取 successUrl
   → mainWindow.loadURL(successUrl)
   → 加载 menu.html 主界面
```

const initUpdater = require('../event/updater');

// 在窗口 ready-to-show 之后初始化
initUpdater(() => {
    updateFlag = true;

`initUpdater` 接收一个回调，当检测到更新并下载完成后调用。`updateFlag = true` 会改变窗口关闭行为（跳过确认对话框直接退出）。

## 12. 环境流量标记注入

**文件路径**：`galaxy-client/src/common/createStateWindow.js`

```javascript
ses.webRequest.onBeforeSendHeaders((details, callback) => {
    const envSettings = store.getEnvSettings();
    if (envSettings && envSettings.ingressTrafficEnv) {
        details.requestHeaders['ingress-traffic-env'] = envSettings.ingressTrafficEnv;
    }
    if (envSettings && envSettings.trafficEnv) {
        details.requestHeaders['traffic-env'] = envSettings.trafficEnv;
    }
    callback({ requestHeaders: details.requestHeaders });
});
```

### 12.1 用途

在所有 HTTP 请求的请求头中注入环境流量标记：

| Header | 说明 |
|--------|------|
| `ingress-traffic-env` | Ingress 流量环境标识 |
| `traffic-env` | 通用流量环境标识 |

这些标记用于服务端的流量路由（灰度发布、测试环境隔离等），通过 `envSettings` 从 `electron-store` 读取。

## 13. 窗口间通信机制

### 13.1 主进程 → 渲染进程

通过 `sendToRenderMsg` 工具函数：

```javascript
// galaxy-client/src/utils.js
function sendToRenderMsg(channel, data) {
    const { mainWindow } = app;
    if (mainWindow && mainWindow.webContents) {
        mainWindow.webContents.send(channel, data);
    }
}
```

常用事件：

| Channel | 方向 | 说明 |
|---------|------|------|
| `ws-kill-java-only` | 主→渲染 | 通知关闭 WebSocket |
| `logout-notify` | 主→渲染 | 通知登出 |
| `web-awaken` | 主→渲染 | 协议唤醒传递 URL |
| `set-dev-buttons-visible` | 主→渲染 | DevTools 开关状态 |

### 13.2 渲染进程 → 主进程

通过 `electron-better-ipc` 的 `ipc.callMain()` 或 `ipc.send()`：

```javascript
// galaxy 前端
const port = await ipc.callMain('get-ws-port');
ipc.send('set-modules', moduleData);
```

### 13.3 主窗口 → webview

通过 webview 的 `send` 方法：

```javascript
// galaxy/src/entries/menu/App.js
webview.send('parent-message', data);
```

### 13.4 webview → 主窗口

通过 `ipc-message` 事件或渲染进程 IPC：

```javascript
// webview 内部
const { ipcRenderer } = require('electron');
ipcRenderer.sendToHost('message', data);

// 主窗口监听
webview.addEventListener('ipc-message', event => { ... });
```

## 14. 开发者工具与右键菜单

### 14.1 开发者工具

```javascript
if (isDev || type === 'vt' || type === 'test') {
    mainWindow.webContents.openDevTools({
        mode: 'undocked',
        activate: false,
    });
}
```

- 开发环境（`isDev`）和测试环境（`vt`、`test`）自动打开 DevTools
- `mode: 'undocked'`：独立窗口打开
- `activate: false`：不抢占焦点

### 14.2 DevTools 可见状态通知

```javascript
mainWindow.webContents.on('devtools-opened', () => {
    sendToRenderMsg('set-dev-buttons-visible', true);
});
mainWindow.webContents.on('devtools-closed', () => {
    sendToRenderMsg('set-dev-buttons-visible', false);
});
```

当 DevTools 打开/关闭时通知渲染进程，用于显示/隐藏调试按钮。

### 14.3 右键菜单

```javascript
const ctxMenu = new Menu();
ctxMenu.append(new MenuItem({
    label: '粘贴',
    role: 'paste',
}));
mainWindow.webContents.on('context-menu', (e, params) => {
    ctxMenu.popup(mainWindow, params.x, params.y);
});
```

自定义的右键菜单只包含「粘贴」功能。
