## 推荐学习顺序

### 📚 第一阶段：理解项目启动流程（1-2 天）

按以下顺序阅读，理解应用如何启动：

```
1. src/electron.js                          # 入口文件
2. src/init/window.js                       # 窗口创建
3. src/common/urls.js                       # 环境配置
4. src/msg-center/start/appStart.js         # 消息中心启动
5. src/msg-center/start/frontStart.js       # WebSocket启动
6. src/msg-center/start/reverseStart.js     # IPC连接启动
```

### 📚 第二阶段：理解核心通信机制（2-3 天）

```
【IPC 逆向通信】
7. src/msg-center/core/reverse/initIpcTask.js     # IPC初始化
8. src/msg-center/core/reverse/asyncSelectTask.js # IPC消息轮询
9. src/msg-center/core/reverse/dll/clibrary.js    # DLL调用

【WebSocket 前端通信】
10. src/msg-center/core/front/frontServer.js      # WS服务端

【MQTT 云端通信】
11. src/msg-center/core/mq/mqttClientBase.js      # MQTT客户端
```

### 📚 第三阶段：理解消息分发机制（3-4 天）

```
【消息分发中心】
12. src/msg-center/dispatch-center/dispatchInBound.js   # 入站
13. src/msg-center/dispatch-center/dispatchOutBound.js  # 出站

【消息处理器】
14. src/msg-center/dispatch-center/handle/msgHandleBase.js  # 三段式
```

### 📚 第四阶段：理解业务逻辑（持续）

```
【登录流程】
15. src/msg-center/business/convert-service/loginService.js

【消息任务】
16. src/msg-center/business/task-mqtt/mqttChatService.js
```

## 调试技巧

### 日志关键字搜索

```bash
# 查看 MQTT 接收的任务
grep "[接收mqtt任务]" app.log

# 查看下发给逆向的消息
grep "[dispatchInBound]" app.log

# 查看逆向返回的消息
grep "[DispatchCenter]:接收逆向" app.log

# 查看三段式消息处理
grep "[msgConvert]" app.log

# 查看特定用户
grep "[wxid_xxx]" app.log
```

```
Windows: %USERPROFILE%\AppData\Roaming\galaxy-client\logs\
```
